Deep Dive into Trie Data Structure (Python Implementation)

Introduction

A Trie (or prefix tree) is a tree-based data structure that stores a set (or associative array) of keys—typically strings—by decomposing them into individual characters. Each node represents a prefix; the root is an empty string, and each edge adds a character.

With operations like insertion, search, and deletion running in O(m) time (where m is the length of the key), Tries excel at tasks involving prefix-based queries. They are particularly useful in autocomplete systems, spell checkers, and network routing (e.g., IP routing via longest prefix match).

Real-World Applications

  • Autocomplete: Quickly suggest words based on user input by traversing to the prefix node and enumerating completions.
  • Spell Checking: Validate whether a word exists in a large dictionary efficiently.
  • IP Routing: Routers use tries to match IP address prefixes and determine the appropriate routing path.
  • Dictionary Validation: Used in text editors and word games to verify word existence.

Python Implementation of Trie

Below is a simple and clear Python implementation of the Trie data structure. Each method is commented to explain the functionality. 


# Define the Trie Node
class TrieNode:
    def __init__(self):
        # Dictionary to hold children nodes: key is character, value is TrieNode
        self.children = {}
        # Flag to indicate the end of a word
        self.is_end_of_word = False

# Define the Trie Data Structure
class Trie:
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, word: str) -> None:
        """
        Insert a word into the trie.
        """
        node = self.root
        for char in word:
            # If the character is not already a child, create a new node
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        # Mark the final node as the end of the word
        node.is_end_of_word = True

    def search(self, word: str) -> bool:
        """
        Search for a word in the trie.
        Returns True if the word exists, otherwise False.
        """
        node = self.root
        for char in word:
            if char not in node.children:
                return False  # Character path doesn't exist, word not found
            node = node.children[char]
        return node.is_end_of_word  # Ensure the word is complete

    def starts_with(self, prefix: str) -> bool:
        """
        Check if any word in the trie starts with the given prefix.
        """
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False  # Prefix path doesn't exist
            node = node.children[char]
        return True

    def delete(self, word: str) -> bool:
        """
        Delete a word from the trie.
        Returns True if deletion was successful, otherwise False.
        """
        def _delete(node: TrieNode, word: str, index: int) -> bool:
            # Base case: reached the end of the word
            if index == len(word):
                if not node.is_end_of_word:
                    return False  # Word doesn't exist
                node.is_end_of_word = False
                # If node has no children, it can be removed
                return len(node.children) == 0
            char = word[index]
            if char not in node.children:
                return False  # Word doesn't exist
            should_delete_child = _delete(node.children[char], word, index + 1)
            if should_delete_child:
                # Remove the mapping for the character
                del node.children[char]
                # Return True if node is not the end of another word and has no other children
                return len(node.children) == 0 and not node.is_end_of_word
            return False

        return _delete(self.root, word, 0)

    print("Search for 'hello':", trie.search("hello"))      # Expected True

    print("Starts with 'hel':", trie.starts_with("hel"))    # Expected True

Advanced DSA Problems Using Trie in Python

Tries are a popular topic in coding interviews and algorithm challenges. Below are several advanced problems solved using the Trie.

1. Longest Common Prefix

Problem: Given an array of strings, determine the longest common prefix shared among them.

Approach: Insert all words into the trie. Then, traverse from the root while there is only one child and the node is not the end of a word. 


def longest_common_prefix(words):
    if not words:
        return ""
    
    # Build the trie with all words
    trie = Trie()
    for word in words:
        trie.insert(word)
    
    prefix = ""
    node = trie.root
    # Continue down the trie until branching occurs or end of word is reached
    while node and not node.is_end_of_word and len(node.children) == 1:
        char = next(iter(node.children))
        prefix += char
        node = node.children[char]
    return prefix

# Example usage:
words_list = ["flower", "flow", "flight"]
print("Longest common prefix:", longest_common_prefix(words_list))  # Output: "fl"

2. Autocomplete System

Problem: Given a dictionary of words and a prefix, return all words starting with that prefix.

Approach: Traverse the trie to the node corresponding to the prefix and then perform a DFS to collect all descendant words. 


def autocomplete(trie, prefix):
    # Helper function to perform DFS and collect words
    def dfs(node, current_word, words_found):
        if node.is_end_of_word:
            words_found.append(current_word)
        for char, child in node.children.items():
            dfs(child, current_word + char, words_found)
    
    node = trie.root
    for char in prefix:
        if char not in node.children:
            return []  # No words with the given prefix
        node = node.children[char]
    
    words_found = []
    dfs(node, prefix, words_found)
    return words_found

# Example usage:
words_dict = ["dog", "deer", "deal", "dove", "dodge"]
trie_auto = Trie()
for word in words_dict:
    trie_auto.insert(word)

print("Autocomplete suggestions for 'do':", autocomplete(trie_auto, "do"))

3. Replacing Words Using a Prefix Dictionary

Problem: Given a dictionary of root words and a sentence, replace each word in the sentence with the shortest root word that is a prefix of that word.

Approach: Build a trie with the root words. For each word in the sentence, traverse the trie until a node marked as the end of a word is found; if none is found, keep the original word. 


def replace_words(dictionary, sentence):
    # Build the trie from the dictionary of roots
    trie = Trie()
    for root in dictionary:
        trie.insert(root)
    
    def replace_word(word):
        node = trie.root
        replacement = ""
        for char in word:
            if char not in node.children:
                break
            replacement += char
            node = node.children[char]
            if node.is_end_of_word:
                return replacement  # Found the shortest root
        return word  # No matching root found
    
    # Replace each word in the sentence
    replaced_words = [replace_word(word) for word in sentence.split()]
    return " ".join(replaced_words)

# Example usage:
roots = ["cat", "bat", "rat"]
sentence = "the cattle was rattled by the battery"
print("Replaced sentence:", replace_words(roots, sentence))

4. Stream Checker (Trie with a Sliding Window)

Problem: Given a continuous stream of characters, determine if any suffix of the stream forms a word in the given dictionary.

Approach: Insert the reversed words into the trie. As characters stream in, maintain a running sequence (in reverse order) and traverse the trie to check for any matching word. 


class StreamChecker:
    def __init__(self, words):
        # Build a trie of reversed words for efficient suffix checking
        self.trie = Trie()
        for word in words:
            self.trie.insert(word[::-1])
        self.stream = ""
    
    def query(self, letter: str) -> bool:
        # Prepend the new letter to the stream (to keep the order reversed)
        self.stream = letter + self.stream
        node = self.trie.root
        for char in self.stream:
            if char not in node.children:
                return False
            node = node.children[char]
            if node.is_end_of_word:
                return True
        return False

# Example usage:
stream_checker = StreamChecker(["cd", "f", "kl"])
print("Query 'a':", stream_checker.query("a"))  # False
print("Query 'b':", stream_checker.query("b"))  # False
print("Query 'c':", stream_checker.query("c"))  # False
print("Query 'd':", stream_checker.query("d"))  # True, since "cd" is found

Trie in Parallel with Other Data Structures

While Tries are excellent for prefix-based searches, combining them with other data structures can unlock even more functionality:

  • Trie + Hash Map: Enhance each trie node by storing extra information (e.g., frequency counts, metadata, or links to full records) using a hash map. This combination is useful in ranking autocomplete suggestions based on frequency.
  • Trie + Heap (Priority Queue): In an autocomplete system, each trie node could maintain a heap (or priority queue) of the top K suggestions, allowing quick retrieval of the most relevant words.
  • Trie vs. Hash Tables: Although hash tables offer O(1) lookup for complete words, they do not support prefix queries. A combined approach can use a hash table for exact matches and a trie for prefix-based queries.
  • Trie with Graph Algorithms: For problems like word ladders or transformation sequences, a trie can be used alongside BFS/DFS to prune search spaces by quickly discarding non-existent prefixes.

The choice to combine a trie with another data structure depends on the problem's requirements and can lead to significant performance improvements.

A. Trie + Hash Map Example

In this example, each trie node stores a frequency count (using a hash map) to rank suggestions. 


class TrieNodeWithFreq:
    def __init__(self):
        self.children = {}
        self.is_end_of_word = False
        self.frequency = 0  # Frequency count for the word ending here

class TrieWithFreq:
    def __init__(self):
        self.root = TrieNodeWithFreq()
    
    def insert(self, word: str) -> None:
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNodeWithFreq()
            node = node.children[char]
        node.is_end_of_word = True
        node.frequency += 1  # Increase frequency for this word
    
    def autocomplete(self, prefix: str):
        """Return words starting with prefix along with their frequencies."""
        def dfs(node, current_word, results):
            if node.is_end_of_word:
                results.append((current_word, node.frequency))
            for char, child in node.children.items():
                dfs(child, current_word + char, results)
        
        node = self.root
        for char in prefix:
            if char not in node.children:
                return []  # No words with this prefix
            node = node.children[char]
        
        suggestions = []
        dfs(node, prefix, suggestions)
        # Sort suggestions by frequency in descending order
        return sorted(suggestions, key=lambda x: -x[1])

# Example usage:

B. Trie + Heap (Priority Queue) Example

Here, each node maintains a heap of top suggestions. This simplified example uses Python’s heapq to store suggestions. 


import heapq

class TrieNodeWithHeap:
    def __init__(self):
        self.children = {}
        self.is_end_of_word = False
        # Heap to store suggestions as (-frequency, word) for max-heap behavior
        self.top_suggestions = []

class TrieWithHeap:
    def __init__(self, k=3):
        self.root = TrieNodeWithHeap()
        self.k = k  # Number of top suggestions to maintain
    
    def insert(self, word: str, frequency: int = 1) -> None:
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNodeWithHeap()
            node = node.children[char]
            # Maintain a heap of top suggestions at this node
            heapq.heappush(node.top_suggestions, (-frequency, word))
            # Keep only top K elements in the heap
            if len(node.top_suggestions) > self.k:
                heapq.heappop(node.top_suggestions)
        node.is_end_of_word = True
    
    def get_top_suggestions(self, prefix: str):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return []
            node = node.children[char]
        # Return the words sorted by frequency (highest first)
        return [word for freq, word in sorted(node.top_suggestions, reverse=True)]
    
# Example usage:
trie_heap = TrieWithHeap(k=3)
trie_heap.insert("hello", 5)
trie_heap.insert("helium", 3)
trie_heap.insert("help", 8)
trie_heap.insert("helicopter", 2)
print("Top suggestions for 'hel':", trie_heap.get_top_suggestions("hel"))

C. Trie vs. Hash Tables

While a hash table provides O(1) lookup for exact keys, it does not allow for prefix queries. A common approach is to use:

  • A hash table for quickly checking exact matches.
  • A trie for retrieving all keys that share a given prefix.

Combining these can be useful in applications like auto-correction or when both exact and prefix searches are needed.

D. Trie with Graph Algorithms

In problems such as word ladders or transformation sequences, a trie can be used to quickly prune search spaces. For example, when searching for valid intermediate words, a trie can discard paths that do not lead to any dictionary word. Combined with BFS or DFS, this can dramatically improve performance.
