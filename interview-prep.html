Mindset First

The Right Mindset

Patterns > Problems. Understanding > Memorization. Solving 300 random LeetCode problems is far less effective than deeply understanding 15 patterns. The interview will throw you an unseen problem — your pattern recognition is what will save you.

❌ Wrong Approach

  • Grind 200+ problems in random order
  • Look at the solution after 5 minutes
  • Skip the "why" — just memorize code
  • Only practice Easy and ignore Medium/Hard
  • Never do timed practice

✅ Right Approach

  • Master one pattern at a time (use this guide)
  • Struggle for 25–30 min before looking
  • Articulate the pattern in plain English first
  • Do 3–4 Mediums per pattern to solidify it
  • Final week: timed mock interviews

How to Use This Guide

  1. Follow the 8-week plan — one topic cluster per week, roughly 2 problems per day.
  2. Read the KnowAlgo deep dive for each topic before attempting problems.
  3. Attempt every problem yourself for at least 25 minutes before reading solutions.
  4. After solving, write a 1-sentence summary of the pattern used. This builds retrieval memory.
  5. Revisit weak spots in weeks 7–8 and do timed mock interviews.

Quick Reference

Pattern Recognition Cheatsheet

For each problem, ask: "What pattern does this fit?" Use these trigger phrases to match quickly.

👉 Two Pointers

Sorted array, pair with target sum, partitioning, palindrome check, 3Sum family.

🪟 Sliding Window

Subarray / substring with constraint. "Longest/shortest … that satisfies …"

➕ Prefix Sum

Subarray sum queries, "count subarrays with sum = k", 2D range sums.

🔍 Binary Search

Sorted input, "minimum / maximum … that satisfies …", search in rotated array.

📚 Stack

Matching brackets, next greater/smaller element, monotonic stack, expression eval.

🔗 Linked List

Detect cycle (Floyd), reverse, merge sorted lists, find kth from end (two runners).

🌳 Tree DFS

Path sums, subtree problems, level-by-level BFS, serialize/deserialize, LCA.

📊 Heap

Top-K elements, kth largest/smallest, merge K sorted, scheduling problems.

🗺 Graph BFS/DFS

Shortest path (unweighted), connected components, islands, cycle detection.

💡 Dynamic Programming

"Count ways", "min/max cost", overlapping subproblems with optimal substructure.

🔙 Backtracking

Generate all combinations/permutations/subsets, constraint satisfaction (Sudoku, N-queens).

💚 Greedy

Locally optimal choices lead to global optimum. Interval scheduling, jump game, task scheduling.

📅 Intervals

Merge overlapping, count meetings, minimum rooms. Always sort by start time first.

⚡ Bit Manipulation

Missing/duplicate, single non-duplicate, bit flags, XOR cancellation, efficient power-of-2 ops.

📝 Trie

Prefix matching, word dictionary, autocomplete, phone book. Always O(m) per insert/search.

Final Advice

Interview Strategy

During the Problem (45-minute interview structure)

TimeWhat to Do
0–5 minRead aloud, clarify constraints (input size, duplicates, edge cases). Ask: "Can I assume…?"
5–10 minIdentify the pattern. State it out loud: "This looks like a sliding window problem because…" Start with brute force verbally.
10–30 minCode the optimal solution. Talk as you code. Handle edge cases explicitly.
30–40 minWalk through your code with the example input. Catch bugs before the interviewer does.
40–45 minState time and space complexity. Discuss trade-offs and possible optimizations.

When You're Stuck

The Pattern Ladder — climb until unstuck:
  1. Restate the problem in your own words
  2. Work through a concrete small example by hand
  3. Ask: "What pattern does this remind me of?"
  4. Think: "What if I sort the input? What structure becomes apparent?"
  5. Think: "What if I start from the end? Or from both ends?"
  6. Think: "Can I define a subproblem? (→ DP)"
  7. Say out loud: "I'm going to try [approach] — here's why I think it might work"

Complexity Rules of Thumb

Input Size nExpected Time ComplexityTypical Approach
n ≤ 20O(2ⁿ) or O(n!)Backtracking, bitmask DP
n ≤ 100O(n³)3 nested loops, Floyd-Warshall
n ≤ 1,000O(n²)2D DP, brute-force 2 pointers
n ≤ 100,000O(n log n)Sort + binary search, heap, merge sort DP
n ≤ 10,000,000O(n)Sliding window, hash map, prefix sum
n ≤ 10¹⁸O(log n)Binary search, fast exponentiation, math
You're Ready When…
  • You can solve any Easy in < 10 minutes
  • You can solve 80% of Mediums in 20–25 minutes
  • You can talk through your approach before writing a single line of code
  • You immediately recognize which pattern applies within 2 minutes of reading a problem
