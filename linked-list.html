Introduction

What is a Linked List?

A linked list is a linear data structure where each element (node) contains a value and a pointer to the next node. Unlike arrays, linked list elements are not stored in contiguous memory — each node is independently allocated and connected via pointers.

Singly Linked List: head ↓ [1 | →] ──→ [2 | →] ──→ [3 | →] ──→ [4 | None] ↑ tail Node structure: { val: 1, next: <reference to next node> }
Array vs Linked List — When to use which
  • Array: O(1) random access by index, O(n) insert/delete at middle (must shift)
  • Linked List: O(n) access by index (must traverse), O(1) insert/delete at known position
  • Use linked list when: frequent insertions/deletions at head or when iterating sequentially
  • Use array when: need random access or cache performance matters
Variants

Types: Singly, Doubly, Circular

Singly Linked List

A ──→ B ──→ C ──→ None (can only traverse forward)

Doubly Linked List

None ←── A ⇄ B ⇄ C ──→ None (can traverse both directions) Node: { val, prev, next }

Circular Linked List

A ──→ B ──→ C ──→ (back to A) (no None; tail.next = head) Useful for: round-robin scheduling, music playlists
Python's collections.deque is a doubly linked list Python's built-in deque (double-ended queue) is implemented as a doubly linked list internally, providing O(1) append and popleft — perfect for BFS queues and sliding windows.

Core Operations

Python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

# BUILD a linked list from array: O(n)
def build(arr):
    dummy = ListNode(0)
    curr = dummy
    for val in arr:
        curr.next = ListNode(val)
        curr = curr.next
    return dummy.next  # 1 -> 2 -> 3 -> None

# TRAVERSE: O(n)
def traverse(head):
    result = []
    curr = head
    while curr:
        result.append(curr.val)
        curr = curr.next
    return result

# INSERT at HEAD: O(1)
def insert_head(head, val):
    node = ListNode(val, head)
    return node  # new head

# INSERT at TAIL: O(n) without tail pointer
def insert_tail(head, val):
    node = ListNode(val)
    if not head:
        return node
    curr = head
    while curr.next:
        curr = curr.next
    curr.next = node
    return head

# INSERT at position k (0-indexed): O(k)
def insert_at(head, k, val):
    dummy = ListNode(0, head)
    curr = dummy
    for _ in range(k):
        if not curr.next:
            break
        curr = curr.next
    curr.next = ListNode(val, curr.next)
    return dummy.next

# DELETE by value: O(n)
def delete_val(head, val):
    dummy = ListNode(0, head)
    curr = dummy
    while curr.next:
        if curr.next.val == val:
            curr.next = curr.next.next
            break
        curr = curr.next
    return dummy.next  # dummy.next is new head

# LENGTH: O(n)
def length(head):
    count = 0
    while head:
        count += 1
        head = head.next
    return count
From Scratch

Python Doubly Linked List

A production-quality doubly linked list using sentinel (dummy) head and tail nodes to eliminate all edge cases. This is the exact pattern used inside Python's collections.OrderedDict.

Python
class Node:
    def __init__(self, val=0):
        self.val = val
        self.prev = None
        self.next = None

class DoublyLinkedList:
    def __init__(self):
        # Sentinel nodes eliminate edge cases (no null checks needed)
        self.head = Node()  # dummy head
        self.tail = Node()  # dummy tail
        self.head.next = self.tail
        self.tail.prev = self.head
        self._size = 0

    def _insert_after(self, node: Node, new_node: Node):
        """Insert new_node right after node. O(1)"""
        new_node.prev = node
        new_node.next = node.next
        node.next.prev = new_node
        node.next = new_node
        self._size += 1

    def _remove(self, node: Node):
        """Unlink node from list. O(1)"""
        node.prev.next = node.next
        node.next.prev = node.prev
        self._size -= 1

    def append(self, val) -> Node:
        """Add to tail (before sentinel tail). O(1)"""
        node = Node(val)
        self._insert_after(self.tail.prev, node)
        return node

    def prepend(self, val) -> Node:
        """Add to head (after sentinel head). O(1)"""
        node = Node(val)
        self._insert_after(self.head, node)
        return node

    def pop(self):
        """Remove from tail. O(1)"""
        if self._size == 0:
            raise IndexError("pop from empty list")
        node = self.tail.prev
        self._remove(node)
        return node.val

    def popleft(self):
        """Remove from head. O(1)"""
        if self._size == 0:
            raise IndexError("pop from empty list")
        node = self.head.next
        self._remove(node)
        return node.val

    def move_to_end(self, node: Node):
        """Move existing node to tail. O(1). Used in LRU Cache."""
        self._remove(node)
        self._insert_after(self.tail.prev, node)

    def __len__(self):
        return self._size

    def __iter__(self):
        curr = self.head.next
        while curr is not self.tail:
            yield curr.val
            curr = curr.next


# Usage
dll = DoublyLinkedList()
dll.append(1)
dll.append(2)
node3 = dll.append(3)
dll.prepend(0)
print(list(dll))      # [0, 1, 2, 3]
dll.move_to_end(node3)  # move 3 to end (it's already there)
dll.pop()             # removes 3
print(list(dll))      # [0, 1, 2]
Interview Toolkit

Key Techniques

1. Dummy Head Pattern

Add a dummy node before the real head. This eliminates special cases for head deletion and simplifies list building.

Python
# WITHOUT dummy: must handle head deletion separately
def delete_no_dummy(head, val):
    if head and head.val == val:
        return head.next  # special case!
    curr = head
    while curr and curr.next:
        if curr.next.val == val:
            curr.next = curr.next.next
            return head
        curr = curr.next
    return head

# WITH dummy: no special cases needed
def delete_with_dummy(head, val):
    dummy = ListNode(0, head)  # dummy.next = head
    curr = dummy
    while curr.next:           # treat dummy as "previous node"
        if curr.next.val == val:
            curr.next = curr.next.next
            return dummy.next  # might return new head
        curr = curr.next
    return dummy.next

2. Two Pointers (Fast & Slow / Runner Technique)

Use two pointers at different speeds or offsets to solve problems in a single pass.

Python
# Find MIDDLE of linked list (slow advances 1, fast advances 2)
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next        # 1 step
        fast = fast.next.next   # 2 steps
    return slow  # at middle when fast reaches end

# Find Kth from END in single pass
def kth_from_end(head, k):
    fast = slow = head
    # Advance fast k steps ahead
    for _ in range(k):
        fast = fast.next
    # Move both until fast reaches end
    while fast:
        fast = fast.next
        slow = slow.next
    return slow  # slow is now k steps from end

# Find INTERSECTION of two linked lists
def get_intersection(headA, headB):
    a, b = headA, headB
    while a is not b:
        # Switch to other list's head when reaching end
        a = a.next if a else headB
        b = b.next if b else headA
    return a  # either the intersection node or None

3. Floyd's Cycle Detection

Two pointers: slow moves 1 step, fast moves 2 steps. If they meet, there's a cycle. Mathematical proof: the meeting point is always the same distance from the cycle start as the head.

Linked list with cycle: 1 → 2 → 3 → 4 → 5 ↑ ↓ 8 ← 7 ← 6 Slow: 1,2,3,4,5,6,7,8,3... Fast: 1,3,5,7,3,5,7,3... They meet at node 3! Phase 2: reset slow to head, advance both 1 step at a time slow: 1,2,3 → meets fast at 3 = cycle start ✓
Python
def hasCycle(head) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False

def detectCycle(head):
    """Find the node where the cycle begins."""
    slow = fast = head
    # Phase 1: detect cycle
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            break
    else:
        return None  # no cycle

    # Phase 2: find cycle start
    # Reset one pointer to head; advance both 1 step at a time
    slow = head
    while slow is not fast:
        slow = slow.next
        fast = fast.next
    return slow  # cycle start node

4. In-Place Reversal

Python
def reverse(head):
    """Reverse entire linked list. O(n) time, O(1) space."""
    prev = None
    curr = head
    while curr:
        next_node = curr.next  # save next
        curr.next = prev       # reverse pointer
        prev = curr            # advance prev
        curr = next_node       # advance curr
    return prev  # new head

def reverseRange(head, left, right):
    """Reverse sublist from position left to right (1-indexed)."""
    dummy = ListNode(0, head)
    prev = dummy
    # Move prev to node just before 'left'
    for _ in range(left - 1):
        prev = prev.next

    curr = prev.next
    for _ in range(right - left):
        next_node = curr.next
        curr.next = next_node.next
        next_node.next = prev.next
        prev.next = next_node
    return dummy.next

LeetCode Problems

LC #206 Reverse Linked List Easy

Reverse a singly linked list. Classic iterative and recursive solutions.

Python
def reverseList(self, head):
    prev, curr = None, head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev

# Recursive (elegant but O(n) stack space)
def reverseList_rec(self, head):
    if not head or not head.next:
        return head
    new_head = self.reverseList_rec(head.next)
    head.next.next = head
    head.next = None
    return new_head
⏱ Time: O(n)💾 Space: O(1) iterative
LC #21 Merge Two Sorted Lists Easy

Merge two sorted linked lists into one sorted list. Use dummy head to simplify.

Python
def mergeTwoLists(self, l1, l2):
    dummy = ListNode(0)
    curr = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2  # attach remaining nodes
    return dummy.next
⏱ Time: O(n+m)💾 Space: O(1)
LC #141 Linked List Cycle Easy

Detect if a linked list has a cycle. Must use O(1) extra space (no sets allowed).

def hasCycle(self, head) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:  # same node → cycle exists
            return True
    return False
⏱ Time: O(n)💾 Space: O(1)
LC #19 Remove Nth Node From End of List Medium

Remove the nth node from the end of the list in one pass.

Key insight: Two pointers n+1 apart. When fast reaches the end, slow is at the node before the one to remove.

Python
def removeNthFromEnd(self, head, n: int):
    dummy = ListNode(0, head)
    fast = slow = dummy
    # Move fast n+1 steps ahead (so slow stops at node BEFORE the target)
    for _ in range(n + 1):
        fast = fast.next
    while fast:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next  # skip the nth node from end
    return dummy.next
⏱ Time: O(n)💾 Space: O(1)
LC #143 Reorder List Medium

Reorder L0→L1→...→Ln to L0→Ln→L1→Ln-1→L2→Ln-2→... in place.

Key insight: Three steps — find middle, reverse second half, interleave.

Python
def reorderList(self, head) -> None:
    # Step 1: Find middle
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next

    # Step 2: Reverse second half
    prev, curr = None, slow.next
    slow.next = None  # cut list in half
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # Step 3: Interleave first and reversed second half
    first, second = head, prev
    while second:
        tmp1, tmp2 = first.next, second.next
        first.next = second
        second.next = tmp1
        first, second = tmp1, tmp2
⏱ Time: O(n)💾 Space: O(1)
LC #23 Merge k Sorted Lists Hard

Merge k sorted linked lists into one sorted list.

Key insight: Use a min-heap of size k. Always extract the minimum, then add that node's next to the heap.

Python
import heapq

def mergeKLists(self, lists):
    # heap items: (val, tie-breaker index, node)
    heap = []
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    dummy = curr = ListNode(0)
    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
⏱ Time: O(n log k) where n=total nodes, k=number of lists💾 Space: O(k)
Real World

Real-World Applications

  • Python collections.deque: doubly linked list, used as stack/queue/deque in standard library
  • Browser history: forward/back navigation (doubly linked list)
  • Undo/redo: text editors, Photoshop layers (doubly linked list)
  • Music playlists: circular linked list — "repeat all" wraps to beginning
  • LRU Cache: doubly linked list + hash map = O(1) access + O(1) recency updates
  • OS process scheduling: run queue as circular linked list for round-robin
  • Hash table chaining: each bucket is a linked list of colliding key-value pairs
  • Memory allocators: free list tracking available memory blocks
