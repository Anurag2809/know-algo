Weighted Graph Primer

In a weighted graph, each edge has a numerical cost (distance, time, price). The goal is often to find the cheapest path from one node to another.

Shortest Path

Dijkstra's Algorithm

Greedy algorithm: always expand the unvisited vertex with the smallest known distance. Uses a min-heap for efficiency. Only works with non-negative edge weights.

Graph: A→B(4), A→C(1), C→B(2), B→D(1), C→D(5) Find shortest path from A to all nodes. Init: dist={A:0, B:∞, C:∞, D:∞}, heap=[(0,A)] Pop (0,A): relax A's neighbors B: 0+4=4 < ∞ → dist[B]=4, heap push (4,B) C: 0+1=1 < ∞ → dist[C]=1, heap push (1,C) heap=[(1,C),(4,B)] Pop (1,C): relax C's neighbors B: 1+2=3 < 4 → dist[B]=3, heap push (3,B) D: 1+5=6 < ∞ → dist[D]=6, heap push (6,D) heap=[(3,B),(4,B),(6,D)] Pop (3,B): relax B's neighbors D: 3+1=4 < 6 → dist[D]=4, heap push (4,D) heap=[(4,B),(4,D),(6,D)] Pop (4,B): dist[B]=3 < 4, skip (stale entry) Pop (4,D): done. Final: {A:0, C:1, B:3, D:4}
Python
import heapq
from collections import defaultdict

def dijkstra(graph, start):
    """
    graph: {node: [(neighbor, weight), ...]}  (weighted adjacency list)
    Returns: {node: min_distance_from_start}
    """
    dist = {start: 0}
    heap = [(0, start)]   # (distance, node)

    while heap:
        d, node = heapq.heappop(heap)

        # Skip stale heap entries (node already processed with shorter distance)
        if d > dist.get(node, float('inf')):
            continue

        for neighbor, weight in graph.get(node, []):
            new_dist = d + weight
            if new_dist < dist.get(neighbor, float('inf')):
                dist[neighbor] = new_dist
                heapq.heappush(heap, (new_dist, neighbor))

    return dist

# Dijkstra with PATH RECONSTRUCTION
def dijkstra_with_path(graph, start, end):
    dist = {start: 0}
    parent = {start: None}
    heap = [(0, start)]
    while heap:
        d, node = heapq.heappop(heap)
        if node == end:
            break
        if d > dist.get(node, float('inf')):
            continue
        for neighbor, weight in graph.get(node, []):
            new_dist = d + weight
            if new_dist < dist.get(neighbor, float('inf')):
                dist[neighbor] = new_dist
                parent[neighbor] = node
                heapq.heappush(heap, (new_dist, neighbor))
    # Reconstruct path
    path, curr = [], end
    while curr is not None:
        path.append(curr)
        curr = parent.get(curr)
    return dist.get(end, float('inf')), path[::-1]
Why Dijkstra fails with negative edges Dijkstra assumes that once a node is popped from the heap, its distance is final (greedy). With negative edges, a later path could be shorter even after we've "finalized" a node. Use Bellman-Ford when negative weights are possible.
Negative Weights

Bellman-Ford Algorithm

Relax all edges V-1 times. Any shortest path has at most V-1 edges (assuming no negative cycles). A V-th relaxation that still improves distances indicates a negative cycle.

Python
def bellman_ford(n, edges, start):
    """
    n: number of vertices (0 to n-1)
    edges: list of (u, v, weight)
    Returns: (dist_dict, has_negative_cycle)
    """
    dist = {i: float('inf') for i in range(n)}
    dist[start] = 0

    # Relax all edges V-1 times
    for _ in range(n - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:
            break  # early termination: no improvement

    # V-th pass: detect negative cycle
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return dist, True  # negative cycle exists!

    return dist, False

# SPFA (Shortest Path Faster Algorithm) — practical BFS-based optimization
# Average O(E), worst O(VE) — often faster than Bellman-Ford in practice
from collections import deque

def spfa(n, adj, start):
    """adj: {node: [(neighbor, weight), ...]}"""
    dist = [float('inf')] * n
    dist[start] = 0
    in_queue = [False] * n
    in_queue[start] = True
    queue = deque([start])
    count = [0] * n  # how many times each node enqueued

    while queue:
        node = queue.popleft()
        in_queue[node] = False
        for neighbor, weight in adj.get(node, []):
            if dist[node] + weight < dist[neighbor]:
                dist[neighbor] = dist[node] + weight
                count[neighbor] += 1
                if count[neighbor] >= n:
                    return None  # negative cycle
                if not in_queue[neighbor]:
                    queue.append(neighbor)
                    in_queue[neighbor] = True
    return dist
All-Pairs

Floyd-Warshall Algorithm

Find shortest paths between all pairs of vertices in O(V³). Uses dynamic programming: for each intermediate vertex k, check if going through k improves the path from i to j.

Python
def floyd_warshall(n, edges):
    """
    n: number of vertices
    edges: list of (u, v, weight)
    Returns: dist[i][j] = shortest path from i to j
    """
    INF = float('inf')
    dist = [[INF] * n for _ in range(n)]
    for i in range(n):
        dist[i][i] = 0
    for u, v, w in edges:
        dist[u][v] = min(dist[u][v], w)

    # Core: try every vertex k as intermediate
    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    # Check for negative cycles: if dist[i][i] < 0, negative cycle exists
    for i in range(n):
        if dist[i][i] < 0:
            return None  # negative cycle

    return dist
When to use Floyd-Warshall vs running Dijkstra from each node
  • Floyd-Warshall: O(V³), simple code, handles negative edges (not negative cycles)
  • Running Dijkstra V times: O(V·(V+E)·log V) — better for sparse graphs with many vertices
  • Rule of thumb: Floyd-Warshall for dense graphs or when V is small (<500); repeated Dijkstra for large sparse graphs
DAG Ordering

Topological Sort

A topological ordering of a DAG lists vertices such that for every edge u→v, u comes before v. Requires: directed and acyclic. Two approaches: Kahn's BFS (in-degree based) and DFS post-order.

DAG: course prerequisites 0→2, 0→3, 1→3, 1→4, 2→5, 3→5, 4→5 Valid topological orderings: [0,1,2,3,4,5] or [1,0,3,2,4,5] etc. (many valid orderings exist; just need all deps before dependents)
Python
from collections import deque, defaultdict

# KAHN'S ALGORITHM (BFS + in-degree)
def topo_sort_kahn(n, edges):
    """
    Returns topological order, or [] if cycle exists.
    edges: list of (u, v) meaning u must come before v.
    """
    graph = defaultdict(list)
    in_degree = [0] * n
    for u, v in edges:
        graph[u].append(v)
        in_degree[v] += 1

    # Start with all nodes having no incoming edges
    queue = deque([i for i in range(n) if in_degree[i] == 0])
    result = []

    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    return result if len(result) == n else []  # [] = cycle exists

# DFS POST-ORDER (add to result AFTER exploring all successors)
def topo_sort_dfs(n, edges):
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)

    visited = set()
    order = []

    def dfs(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)
        order.append(node)  # add AFTER all neighbors explored

    for node in range(n):
        if node not in visited:
            dfs(node)

    return order[::-1]  # reverse post-order = topological order
Connectivity

Union-Find (Disjoint Set Union)

Efficiently maintains a partition of elements into disjoint sets. Supports two operations: find (which set does element belong to?) and union (merge two sets). With path compression + union by rank, both operations are nearly O(1) — specifically O(α(n)) where α is the inverse Ackermann function (practically a constant ≤ 4 for any real input).

Python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))  # initially each element is its own root
        self.rank = [0] * n           # tree height (for union by rank)
        self.components = n           # number of connected components

    def find(self, x):
        """Find root with PATH COMPRESSION (flattens tree). O(α(n))."""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # compress path
        return self.parent[x]

    def union(self, x, y) -> bool:
        """Merge sets containing x and y. Returns False if already same set."""
        px, py = self.find(x), self.find(y)
        if px == py:
            return False  # already connected — adding this edge would create a cycle
        # UNION BY RANK: attach smaller tree under larger tree
        if self.rank[px] < self.rank[py]:
            px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        self.components -= 1
        return True

    def connected(self, x, y) -> bool:
        return self.find(x) == self.find(y)


# Example uses
uf = UnionFind(5)
uf.union(0, 1)     # merge {0} and {1} → {0,1}
uf.union(1, 2)     # merge {0,1} and {2} → {0,1,2}
print(uf.connected(0, 2))  # True
print(uf.connected(0, 3))  # False
print(uf.components)       # 3  ({0,1,2}, {3}, {4})

# Cycle detection: if union returns False, edge creates a cycle
def has_cycle(n, edges):
    uf = UnionFind(n)
    for u, v in edges:
        if not uf.union(u, v):
            return True  # u and v already connected → cycle!
    return False
Spanning Trees

Minimum Spanning Tree

A spanning tree of a connected undirected graph connects all vertices with exactly V-1 edges and minimum total weight. Two classic algorithms:

Python
# KRUSKAL'S ALGORITHM: sort edges by weight, add if doesn't create cycle
# Time: O(E log E) for sorting, O(E·α(V)) for Union-Find
def kruskal(n, edges):
    """edges: list of (weight, u, v)"""
    edges.sort()  # sort by weight
    uf = UnionFind(n)
    mst_cost = 0
    mst_edges = []

    for weight, u, v in edges:
        if uf.union(u, v):          # no cycle created
            mst_cost += weight
            mst_edges.append((u, v, weight))
            if len(mst_edges) == n - 1:
                break               # MST complete (V-1 edges)

    return mst_cost, mst_edges


# PRIM'S ALGORITHM: greedily add cheapest edge connecting MST to non-MST vertex
# Time: O((V+E) log V) with min-heap
def prim(n, adj):
    """adj: {node: [(neighbor, weight), ...]}"""
    visited = set()
    heap = [(0, 0)]   # (cost, start_node)
    mst_cost = 0

    while heap and len(visited) < n:
        cost, node = heapq.heappop(heap)
        if node in visited:
            continue
        visited.add(node)
        mst_cost += cost
        for neighbor, weight in adj.get(node, []):
            if neighbor not in visited:
                heapq.heappush(heap, (weight, neighbor))

    return mst_cost if len(visited) == n else -1  # -1 if disconnected

LeetCode Problems

LC #743Network Delay TimeMedium

Find the time until all nodes receive a signal. = Dijkstra single-source shortest path; answer is the max distance to any node.

Python
def networkDelayTime(self, times, n, k):
    graph = defaultdict(list)
    for u, v, w in times:
        graph[u].append((v, w))
    dist = dijkstra(graph, k)  # from source k
    if len(dist) == n:
        return max(dist.values())
    return -1  # some node unreachable
⏱ Time: O((V+E) log V)💾 Space: O(V+E)
LC #787Cheapest Flights Within K StopsMedium

Find cheapest price from src to dst with at most k stops. Modified Bellman-Ford: relax exactly k+1 times, tracking prices after each iteration.

Python
def findCheapestPrice(self, n, flights, src, dst, k):
    prices = [float('inf')] * n
    prices[src] = 0

    for _ in range(k + 1):  # k stops = k+1 edges
        temp = prices[:]    # snapshot: only use prices from PREVIOUS round
        for u, v, w in flights:
            if prices[u] != float('inf') and prices[u] + w < temp[v]:
                temp[v] = prices[u] + w
        prices = temp

    return prices[dst] if prices[dst] != float('inf') else -1
⏱ Time: O(k·E)💾 Space: O(V)
LC #210Course Schedule IIMedium

Return a valid course ordering respecting prerequisites, or [] if impossible (cycle exists). = Topological sort.

Python
def findOrder(self, numCourses, prerequisites):
    graph = defaultdict(list)
    in_degree = [0] * numCourses
    for a, b in prerequisites:
        graph[b].append(a)
        in_degree[a] += 1

    queue = deque([i for i in range(numCourses) if in_degree[i] == 0])
    result = []
    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    return result if len(result) == numCourses else []
⏱ Time: O(V+E)💾 Space: O(V+E)
LC #547Number of ProvincesMedium

Count connected components in an undirected graph given as adjacency matrix. Union-Find is perfectly suited here.

Python
def findCircleNum(self, isConnected):
    n = len(isConnected)
    uf = UnionFind(n)
    for i in range(n):
        for j in range(i + 1, n):
            if isConnected[i][j] == 1:
                uf.union(i, j)
    return uf.components
⏱ Time: O(n²·α(n))💾 Space: O(n)
LC #1584Min Cost to Connect All PointsMedium

Find the minimum cost to connect all points (Manhattan distance as edge weight). = Minimum Spanning Tree with Prim's algorithm.

Python
def minCostConnectPoints(self, points):
    n = len(points)
    visited = set()
    heap = [(0, 0)]  # (cost, point_index)
    total = 0

    while len(visited) < n:
        cost, i = heapq.heappop(heap)
        if i in visited:
            continue
        visited.add(i)
        total += cost
        for j in range(n):
            if j not in visited:
                dist = abs(points[i][0]-points[j][0]) + abs(points[i][1]-points[j][1])
                heapq.heappush(heap, (dist, j))
    return total
⏱ Time: O(n² log n)💾 Space: O(n²)
LC #332Reconstruct ItineraryHard

Find Eulerian path in a directed graph (use all edges exactly once, lexicographically smallest). Hierholzer's algorithm: DFS, add to result only when no more edges to explore.

Python
def findItinerary(self, tickets):
    graph = defaultdict(list)
    for src, dst in sorted(tickets, reverse=True):
        graph[src].append(dst)
    # Sort reverse so smallest is at back (pop() is O(1))

    result = []
    stack = ["JFK"]
    while stack:
        while graph[stack[-1]]:
            stack.append(graph[stack[-1]].pop())
        result.append(stack.pop())  # dead end = add to result
    return result[::-1]
⏱ Time: O(E log E)💾 Space: O(V+E)
