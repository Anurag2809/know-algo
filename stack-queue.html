Stack & Queue

LIFO vs FIFO. Two deceptively simple structures that unlock a huge class of problems. The monotonic stack is one of the most powerful — and underrated — patterns in competitive programming and interviews.

📚 LIFO / FIFO 🐍 Python deque 🎯 6 LeetCode problems 📉 Monotonic stack
Core Structure

Stack (LIFO)

A stack follows Last In, First Out — like a stack of plates. You can only add (push) or remove (pop) from the top. Python's list provides an efficient stack with append (push) and pop — both O(1) amortized.

Push 1, Push 2, Push 3, Pop: [] → [1] → [1,2] → [1,2,3] → [1,2] ↑ top=3, popped Think: stack of plates — always add/remove from the top
Python
# Stack using Python list — the standard approach
stack = []

# PUSH: O(1) amortized
stack.append(1)
stack.append(2)
stack.append(3)

# PEEK (look at top without removing): O(1)
top = stack[-1]        # 3

# POP (remove from top): O(1)
val = stack.pop()      # returns 3, stack = [1, 2]

# IS EMPTY: O(1)
if not stack:
    print("empty")

# SIZE: O(1)
print(len(stack))      # 2

# Common pattern: process until empty
while stack:
    item = stack.pop()
    # process item
When to reach for a stack
  • You need to reverse something (process in LIFO order)
  • You need to match pairs (brackets, tags, parentheses)
  • You need to remember context while going deeper (DFS, recursive call stack)
  • You see the phrase "nearest/previous/next greater/smaller element"
Core Structure

Queue (FIFO)

A queue follows First In, First Out — like a line at a store. You add (enqueue) to the back and remove (dequeue) from the front. Always use collections.deque, never list for queue operations.

Enqueue 1, Enqueue 2, Enqueue 3, Dequeue: [] → [1] → [1,2] → [1,2,3] → [2,3] front front=1 dequeued WHY NOT list.pop(0)? list.pop(0) = O(n) — shifts ALL elements left deque.popleft() = O(1) — doubly linked list, no shifting
Python
from collections import deque

# Queue using deque — ALWAYS use this, never list for queues
queue = deque()

# ENQUEUE (add to back): O(1)
queue.append(1)
queue.append(2)
queue.append(3)

# PEEK front: O(1)
front = queue[0]       # 1

# DEQUEUE (remove from front): O(1)
val = queue.popleft()  # returns 1, queue = deque([2, 3])

# BFS template (most common queue use)
def bfs(start, graph):
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        # process node
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
Double-Ended

Deque (Double-Ended Queue)

A deque supports O(1) insertions and deletions at both ends. It subsumes both stack and queue. Python's collections.deque is implemented as a doubly linked list and is the go-to for sliding window algorithms.

Python
from collections import deque

d = deque([1, 2, 3])

# Add/remove from BOTH ends: all O(1)
d.append(4)       # [1, 2, 3, 4]
d.appendleft(0)   # [0, 1, 2, 3, 4]
d.pop()           # [0, 1, 2, 3] → returns 4
d.popleft()       # [1, 2, 3] → returns 0

# Fixed-size deque (automatically evicts oldest element)
window = deque(maxlen=3)
for x in [1, 2, 3, 4, 5]:
    window.append(x)
# After loop: deque([3, 4, 5], maxlen=3)
# → maxlen evicts leftmost when full

# Access both ends: O(1)
print(d[0], d[-1])    # front and back

# Rotate (move elements from one end to other)
d.rotate(1)   # rightward: last element moves to front
d.rotate(-1)  # leftward: first element moves to back
Advanced Pattern

Monotonic Stack

A monotonic stack maintains a sorted invariant — elements are always increasing or decreasing from bottom to top. It's the key to solving "next greater/smaller element" problems in O(n) instead of O(n²).

Monotonic Increasing Stack → Next Greater Element

Array: [2, 1, 5, 3, 4] Goal: for each element, find the next element that is GREATER i=0, val=2: stack empty → push 2 → stack=[2] i=1, val=1: 1 < 2 → push 1 → stack=[2,1] i=2, val=5: 5 > 1 → pop 1, NGE[1]=5; 5>2 → pop 2, NGE[2]=5 → push 5 → stack=[5] i=3, val=3: 3 < 5 → push 3 → stack=[5,3] i=4, val=4: 4 > 3 → pop 3, NGE[3]=4 → push 4 → stack=[5,4] Remaining [5,4] never found greater → NGE = -1 Result: [5, 5, -1, 4, -1] KEY: Pop when current > top → the current IS the "next greater" for popped elements
Python
def next_greater_element(nums):
    """For each element, find the next larger element. O(n)."""
    n = len(nums)
    result = [-1] * n
    stack = []  # stores INDICES (not values)

    for i in range(n):
        # Pop all elements that are SMALLER than current
        while stack and nums[stack[-1]] < nums[i]:
            idx = stack.pop()
            result[idx] = nums[i]  # current is the "next greater" for idx
        stack.append(i)
    # Remaining elements in stack have no next greater → stay -1
    return result

def next_smaller_element(nums):
    """Monotonic decreasing stack."""
    n = len(nums)
    result = [-1] * n
    stack = []
    for i in range(n):
        while stack and nums[stack[-1]] > nums[i]:  # flip comparison
            idx = stack.pop()
            result[idx] = nums[i]
        stack.append(i)
    return result

# CIRCULAR array variant (traverse twice)
def next_greater_circular(nums):
    n = len(nums)
    result = [-1] * n
    stack = []
    for i in range(2 * n):  # go around twice
        while stack and nums[stack[-1]] < nums[i % n]:
            idx = stack.pop()
            result[idx] = nums[i % n]
        if i < n:
            stack.append(i)
    return result

Monotonic Deque → Sliding Window Maximum

nums=[3,1,2,5,4], k=3 (window size 3) Find max in each window of size k. Monotonic DECREASING deque (front = max of current window): i=0, val=3: deque=[] → push 0 → deque=[0(3)] i=1, val=1: 1<3 → push 1 → deque=[0(3),1(1)] i=2, val=2: 2>1 → pop 1; 2<3 → push 2 → deque=[0(3),2(2)] window [0,2] complete → max=nums[front]=nums[0]=3 i=3, val=5: 5>2 → pop 2; 5>3 → pop 0; push 3 → deque=[3(5)] evict indices outside window → deque=[3(5)] window [1,3] → max=5 i=4, val=4: 4<5 → push 4 → deque=[3(5),4(4)] window [2,4] → max=5 Result: [3, 5, 5]
Python
from collections import deque

def sliding_window_maximum(nums, k):
    """
    Max of each sliding window of size k. O(n) time.
    Maintain a monotonic decreasing deque of indices.
    Front is always the index of the current window's maximum.
    """
    result = []
    window = deque()  # stores indices; values are decreasing

    for i, num in enumerate(nums):
        # Remove indices that are OUT of the current window [i-k+1, i]
        while window and window[0] < i - k + 1:
            window.popleft()

        # Remove from back any index whose value is LESS than current
        # (they can never be the max while current is in the window)
        while window and nums[window[-1]] < num:
            window.pop()

        window.append(i)

        # Window is full: record the maximum (front of deque)
        if i >= k - 1:
            result.append(nums[window[0]])

    return result
Classic Designs

Classic Stack/Queue Implementations

Python
# Min Stack: O(1) getMin at all times
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []  # parallel stack tracking minimums

    def push(self, val: int) -> None:
        self.stack.append(val)
        # Push new minimum (either val or current min)
        current_min = min(val, self.min_stack[-1] if self.min_stack else val)
        self.min_stack.append(current_min)

    def pop(self) -> None:
        self.stack.pop()
        self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]  # O(1)!


# Implement Queue using Two Stacks (LC 232)
class MyQueue:
    """
    in_stack: receives all pushes
    out_stack: provides pops/peeks (lazily transferred from in_stack)
    Transfer happens only when out_stack is empty — amortized O(1).
    """
    def __init__(self):
        self.in_stack = []
        self.out_stack = []

    def push(self, x: int) -> None:
        self.in_stack.append(x)

    def _transfer(self):
        if not self.out_stack:
            while self.in_stack:
                self.out_stack.append(self.in_stack.pop())

    def pop(self) -> int:
        self._transfer()
        return self.out_stack.pop()

    def peek(self) -> int:
        self._transfer()
        return self.out_stack[-1]

    def empty(self) -> bool:
        return not self.in_stack and not self.out_stack


# Implement Stack using Two Queues (LC 225)
class MyStack:
    def __init__(self):
        self.queue = deque()

    def push(self, x: int) -> None:
        self.queue.append(x)
        # Rotate so new element is at front
        for _ in range(len(self.queue) - 1):
            self.queue.append(self.queue.popleft())

    def pop(self) -> int:
        return self.queue.popleft()

    def top(self) -> int:
        return self.queue[0]

    def empty(self) -> bool:
        return not self.queue
Interview Patterns

Pattern Recognition

Python
# Pattern 1: Matching brackets/pairs
def is_valid(s: str) -> bool:
    stack = []
    pairs = {')': '(', '}': '{', ']': '['}
    for ch in s:
        if ch in '([{':
            stack.append(ch)
        elif ch in ')]}':
            if not stack or stack[-1] != pairs[ch]:
                return False
            stack.pop()
    return not stack

# Pattern 2: Previous greater element (think: stock span)
def stock_span(prices):
    span = []
    stack = []  # (price, accumulated_span)
    for p in prices:
        curr_span = 1
        while stack and stack[-1][0] <= p:
            curr_span += stack.pop()[1]
        stack.append((p, curr_span))
        span.append(curr_span)
    return span

# Pattern 3: Expression evaluation (Reverse Polish Notation)
def eval_rpn(tokens):
    stack = []
    ops = {'+': lambda a,b: a+b, '-': lambda a,b: a-b,
           '*': lambda a,b: a*b, '/': lambda a,b: int(a/b)}
    for token in tokens:
        if token in ops:
            b, a = stack.pop(), stack.pop()
            stack.append(ops[token](a, b))
        else:
            stack.append(int(token))
    return stack[0]

Why monotonic stack is O(n), not O(n²) Each element is pushed exactly once and popped at most once. Total operations = 2n → amortized O(1) per element. Even though the inner while loop can pop many elements, those pops don't "repeat" — each element is only ever popped once across the entire traversal.

LC #20Valid ParenthesesEasy

Given a string with (, ), {, }, [, ], determine if it is valid (properly closed and nested).

Python
def isValid(self, s: str) -> bool:
    stack = []
    mapping = {')': '(', '}': '{', ']': '['}
    for char in s:
        if char in mapping:
            top = stack.pop() if stack else '#'
            if mapping[char] != top:
                return False
        else:
            stack.append(char)
    return not stack
⏱ Time: O(n)💾 Space: O(n)

LC #739Daily TemperaturesMedium

For each day, find how many days until a warmer temperature. If none, return 0.

Key insight: Classic "next greater element" using monotonic stack on indices.

Python
def dailyTemperatures(self, temperatures: list[int]) -> list[int]:
    n = len(temperatures)
    result = [0] * n
    stack = []  # indices of days waiting for a warmer day

    for i, temp in enumerate(temperatures):
        while stack and temperatures[stack[-1]] < temp:
            idx = stack.pop()
            result[idx] = i - idx  # distance to warmer day
        stack.append(i)

    return result
⏱ Time: O(n)💾 Space: O(n)

LC #84Largest Rectangle in HistogramHard

Find the largest rectangle in a histogram. For each bar, find how far left and right it can extend (until a shorter bar is encountered).

Key insight: Monotonic increasing stack. When we pop a bar, the current bar is the right boundary and the new stack top is the left boundary.

Python
def largestRectangleArea(self, heights: list[int]) -> int:
    # Add sentinel 0s at both ends to flush remaining stack elements
    heights = [0] + heights + [0]
    stack = [0]  # indices; monotonic increasing
    max_area = 0

    for i in range(1, len(heights)):
        while heights[stack[-1]] > heights[i]:
            height = heights[stack.pop()]
            width = i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)

    return max_area

# Example walkthrough for [2,1,5,6,2,3]:
# When we pop height 6 (idx=4), current i=5(h=2):
#   height=6, left_boundary=stack[-1]=3(h=5), right=5
#   width = 5-3-1=1, area=6
# When we pop height 5 (idx=3):
#   height=5, left_boundary=stack[-1]=2(h=1), right=5
#   width=5-2-1=2, area=10  ← answer!
⏱ Time: O(n)💾 Space: O(n)
Real World

Real-World Applications

  • Stack — call stack: every function call pushes a frame; return pops it. Stack overflow = too deep recursion
  • Stack — undo/redo: text editors maintain an undo stack; Ctrl+Z pops the last action
  • Stack — compiler: expression parsing, syntax checking, code generation
  • Queue — BFS: web crawlers, social network friend-of-friend search, shortest path in mazes
  • Queue — task scheduling: printer queue, OS process scheduler (round-robin = circular queue)
  • Queue — event loops: Node.js event queue, JavaScript's setTimeout/Promise callbacks
  • Deque — sliding window: real-time analytics (max/min over last N data points)
  • Monotonic stack: stock span problems, largest rectangle, trapping rain water, next greater element
