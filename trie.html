Trie (Prefix Tree)

A specialized tree that stores strings character-by-character, enabling O(m) search, insert, and prefix queries — where m is the word length, regardless of how many words are stored.

⏱ 25 min read 🎯 6 LeetCode solutions 🐍 Python code 📊 Complexity analysis

🌳What is a Trie?

A Trie (pronounced "try", from the word retrieval) is a tree-like data structure used to store strings. Unlike a binary search tree where entire strings are stored at nodes, a trie stores strings by decomposing them into individual characters — each character occupies its own node.

Every path from the root to a marked leaf spells out a word. The key insight: words with shared prefixes share the same nodes. This is what makes Tries space-efficient and blazing fast for prefix-based queries.

The Name The term "trie" was coined by Edward Fredkin in 1960. It is an infix from the word "retrieval". Despite being spelled "trie", it was originally meant to be pronounced like "tree" — though today most people say "try" to distinguish it from a regular tree.

Why not just use a Hash Table?

A hash table gives you O(1) exact lookups — but it can't answer prefix queries. Given the words ["apple", "app", "application"], a hash table cannot efficiently answer: "What words start with 'app'?" A Trie answers this in O(prefix_length) and then enumerates all matches in O(total_matches).

Core Strengths of a Trie

⚡

O(m) Operations

Insert, search, and prefix check all run in O(m) where m is the word/prefix length. Independent of dictionary size.

🔗

Shared Prefixes

"apple" and "application" share nodes for 'a','p','p','l'. Memory grows with unique characters, not total characters.

🔍

Prefix Enumeration

Find all words starting with a given prefix in O(prefix_len + results). No other data structure does this as naturally.

✂️

Early Pruning

In search problems (like Word Search II), a trie prunes dead-end paths before they are fully explored — huge speedup.

🔬Anatomy of a Trie

The TrieNode

Every trie is built from TrieNode objects. Each node has exactly two things:

  1. children — a dictionary (or array of size 26 for lowercase letters) mapping a character to the next TrieNode.
  2. is_end_of_word — a boolean flag that marks whether this node is the final character of a complete word.
python 
class TrieNode:
    def __init__(self):
        # Maps each character to the child TrieNode that follows it.
        # Using a dict is flexible (supports any alphabet).
        # Using list[26] is faster but limited to lowercase a-z.
        self.children: dict[str, TrieNode] = {}

        # True only if this node is the LAST character of a stored word.
        # This distinguishes "app" (complete word) from the prefix "app"
        # on the way to "apple" or "application".
        self.is_end_of_word: bool = False
Design Choice: dict vs. list[26] Use dict when your alphabet is large or unknown (unicode, lowercase + uppercase, etc.). Use list[26] when you only handle lowercase a–z and need maximum speed. The list approach uses ord(char) - ord('a') for indexing.

Visual Diagram

Let's store five words: "do", "dog", "dot", "go", "got". Each circle is one node. Blue = prefix-only node. Green + ✓ = a complete word ends here (is_end_of_word = True).

Notice that "do", "dog", and "dot" all share the same two nodes for d and o — only 7 nodes total store all 5 words (not 13 individual characters). The o node under d has is_end_of_word = True AND has two children — it is simultaneously the end of "do" and a stepping stone to "dog" and "dot". This dual role is the key subtlety of tries.

Common Mistake Many beginners confuse "no children" with "end of word". A node can be both an end-of-word AND have children (like the 'p' node for "app" above). Always check the is_end_of_word flag, not the absence of children.

⚙️Core Operations

1. Insert

To insert a word, walk the trie character by character. At each step, if the current character has no child node, create one. After processing all characters, mark the final node as is_end_of_word = True.

python – insert 
def insert(self, word: str) -> None:
    node = self.root           # Start at the root every time

    for char in word:
        if char not in node.children:
            # Character not yet seen from this node: create a fresh TrieNode
            node.children[char] = TrieNode()

        node = node.children[char]  # Walk down one level

    # After the loop, 'node' sits at the last character of 'word'.
    # Flag it so we know a complete word ends here.
    node.is_end_of_word = True

# Time Complexity:  O(m)  where m = len(word)
# Space Complexity: O(m)  in the worst case (all new nodes)

2. Search

Walk the trie character by character. If at any point the current character is not a child of the current node, the word is definitely not in the trie — return False. If you reach the end of the word successfully, return node.is_end_of_word. This last check is crucial: we must ensure we landed on a complete word, not just a prefix.

python – search 
def search(self, word: str) -> bool:
    node = self.root

    for char in word:
        if char not in node.children:
            return False          # Dead end — word cannot exist
        node = node.children[char]

    # We successfully walked the entire word.
    # But is it a COMPLETE word or just a prefix?
    # Example: trie contains "apple". Searching "app" walks 3 nodes
    # successfully, but "app" is NOT stored → return False.
    return node.is_end_of_word   # True only if a word ends exactly here

# Time Complexity: O(m)
# Space Complexity: O(1)  (no extra space, just pointer movement)

3. StartsWith (Prefix Check)

Identical to search, except we don't check is_end_of_word at the end. If we can walk all the characters of the prefix without hitting a dead end, the prefix exists.

python – starts_with 
def starts_with(self, prefix: str) -> bool:
    node = self.root

    for char in prefix:
        if char not in node.children:
            return False          # Prefix doesn't exist in the trie
        node = node.children[char]

    # We reached the node at the end of 'prefix'.
    # We don't care if it's a complete word — any extension counts.
    return True                   # Prefix exists

# Time Complexity: O(m) where m = len(prefix)
# Space Complexity: O(1)

4. Delete

Delete is the trickiest operation because we need to clean up nodes that are no longer needed. The rule: after un-marking a word's end node, if that node has no children (no other words branch through it), we can delete it. We recurse from the bottom up, deleting nodes only when safe.

1

Recurse to the end of the word

Use a recursive helper that walks to the target word's final node.

2

Un-mark the end node

Set is_end_of_word = False on the final character's node.

3

Check if the node can be deleted

A node is safe to delete if it has no children AND is not the end of another word. Return this as a signal to the parent.

4

Propagate deletion upward

If the parent receives a "delete me" signal and the parent itself has only that one child and is not an end-of-word, delete the parent too — and so on up the chain.

python – delete 
def delete(self, word: str) -> bool:
    """
    Delete 'word' from the trie.
    Returns True if the deletion was successful, False if word not found.
    """
    def _delete(node: TrieNode, word: str, depth: int) -> bool:
        """
        Returns True if the current node should be deleted by its parent.
        """
        # Base case: we've consumed all characters of 'word'
        if depth == len(word):
            if not node.is_end_of_word:
                return False          # Word doesn't exist in trie
            node.is_end_of_word = False
            # Delete this node only if it has NO children
            # (otherwise other words still pass through it)
            return len(node.children) == 0

        char = word[depth]
        if char not in node.children:
            return False              # Word doesn't exist in trie

        # Recurse into the child for the next character
        should_delete_child = _delete(node.children[char], word, depth + 1)

        if should_delete_child:
            # Child said "please delete me"
            del node.children[char]
            # Now check if WE should also be deleted:
            # only if we have no remaining children AND we aren't an end-of-word
            return len(node.children) == 0 and not node.is_end_of_word

        return False

    return _delete(self.root, word, 0)

# Time Complexity:  O(m)
# Space Complexity: O(m)  due to recursion call stack
Delete Example Trie contains: "app", "apple", "application". Delete "app": Walk to the third 'p'. Set is_end_of_word = False. Does it have children? Yes ('l'). So we do NOT delete the node — we just un-mark it. "apple" and "application" are unaffected.

💻Full Python Implementation

Here is the complete, production-ready Trie implementation with all operations. Every line is commented to explain the reasoning.

python – complete trie 
class TrieNode:
    """
    A single node in the Trie.
    Each node represents one character on a path from root to leaf.
    """
    def __init__(self):
        self.children: dict[str, 'TrieNode'] = {}
        self.is_end_of_word: bool = False


class Trie:
    """
    Trie (Prefix Tree) data structure.

    Supports:
      - insert(word)       → O(m)
      - search(word)       → O(m), returns bool
      - starts_with(pfx)   → O(m), returns bool
      - get_words(pfx)     → O(m + output), returns list[str]
      - delete(word)       → O(m), returns bool
      - count_words()      → O(n * m) total words stored

    where m = word/prefix length, n = number of words.
    """

    def __init__(self):
        self.root = TrieNode()
        self._word_count = 0

    # ──────────────────────────────────────────────
    # INSERT
    # ──────────────────────────────────────────────
    def insert(self, word: str) -> None:
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]

        if not node.is_end_of_word:       # Avoid double-counting duplicates
            node.is_end_of_word = True
            self._word_count += 1

    # ──────────────────────────────────────────────
    # SEARCH (exact match)
    # ──────────────────────────────────────────────
    def search(self, word: str) -> bool:
        node = self._traverse(word)
        return node is not None and node.is_end_of_word

    # ──────────────────────────────────────────────
    # PREFIX CHECK
    # ──────────────────────────────────────────────
    def starts_with(self, prefix: str) -> bool:
        return self._traverse(prefix) is not None

    # ──────────────────────────────────────────────
    # GET ALL WORDS WITH PREFIX  (autocomplete)
    # ──────────────────────────────────────────────
    def get_words(self, prefix: str = "") -> list[str]:
        """Return all words in the trie that start with 'prefix'."""
        node = self._traverse(prefix)
        if node is None:
            return []

        results = []
        self._dfs(node, prefix, results)
        return results

    def _dfs(self, node: TrieNode, current: str, results: list[str]) -> None:
        if node.is_end_of_word:
            results.append(current)
        for char, child in node.children.items():
            self._dfs(child, current + char, results)

    # ──────────────────────────────────────────────
    # DELETE
    # ──────────────────────────────────────────────
    def delete(self, word: str) -> bool:
        deleted = self._delete(self.root, word, 0)
        if deleted is not False:
            self._word_count -= 1
        return deleted is not False

    def _delete(self, node: TrieNode, word: str, depth: int):
        if depth == len(word):
            if not node.is_end_of_word:
                return False
            node.is_end_of_word = False
            return len(node.children) == 0   # Signal: safe to delete this node?

        char = word[depth]
        if char not in node.children:
            return False

        should_delete = self._delete(node.children[char], word, depth + 1)
        if should_delete is False:
            return False

        if should_delete:
            del node.children[char]
            return len(node.children) == 0 and not node.is_end_of_word

        return False

    # ──────────────────────────────────────────────
    # HELPERS
    # ──────────────────────────────────────────────
    def _traverse(self, prefix: str):
        """Walk the trie for 'prefix'. Return the final node or None."""
        node = self.root
        for char in prefix:
            if char not in node.children:
                return None
            node = node.children[char]
        return node

    def __len__(self) -> int:
        return self._word_count


# ──────────────────────────────────────────────────────
# USAGE EXAMPLE
# ──────────────────────────────────────────────────────
if __name__ == "__main__":
    trie = Trie()

    words = ["apple", "app", "application", "apply", "apt", "bat", "ball"]
    for w in words:
        trie.insert(w)

    print(f"Total words: {len(trie)}")           # 7
    print(trie.search("app"))                    # True
    print(trie.search("ap"))                     # False (prefix only)
    print(trie.starts_with("ap"))               # True
    print(trie.get_words("app"))                # ['app', 'apple', 'application', 'apply']

    trie.delete("app")
    print(trie.search("app"))                    # False
    print(trie.search("apple"))                  # True  (unaffected)

Operation Trie Hash Table Sorted Array (BST) Notes

Exact Search O(m) O(m) O(m log n) Hash table wins in practice (lower constant)
Prefix Search O(m) ❌ Not supported O(m log n) Trie's biggest advantage
Autocomplete O(m + k) ❌ O(n·m) scan O(m log n + k) k = number of results
Delete O(m) O(m) O(m log n)

Space Complexity

MetricValueExplanation
Worst case O(N · M) N words, each length M, all completely unique (no shared prefixes)
Best case O(M) All words share the same prefix — only one path exists
Typical case Much better than O(N·M) Real dictionaries have massive prefix overlap
Memory Overhead Caveat Each TrieNode object has overhead from Python's dict. For large alphabets or sparse tries, a Radix Tree (Compressed Trie) can dramatically reduce node count. The raw O() analysis doesn't capture Python's per-object overhead.

LC #208
Implement Trie (Prefix Tree)
Medium Trie Design Hash Table

Problem

Implement a trie with insert, search, and startsWith methods. This is the foundational problem — building the data structure from scratch.

Key Insight

The trie class itself can act as the root node. Each instance has children and is_end. This makes the implementation cleaner — no separate root attribute needed.

python – LC 208 
class Trie:
    def __init__(self):
        # The Trie node IS the trie root.
        # Each instance holds children and a flag.
        self.children: dict = {}
        self.is_end: bool = False

    def insert(self, word: str) -> None:
        node = self
        for ch in word:
            if ch not in node.children:
                node.children[ch] = Trie()  # Create child Trie node
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end   # Must be a COMPLETE word

    def startsWith(self, prefix: str) -> bool:
        node = self
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True           # Prefix exists — don't check is_end

# ── Complexity ──────────────────────────────────────
# Time:  O(m) per operation  (m = length of word/prefix)
# Space: O(N * M) total      (N = number of words, M = avg length)
LC #211
Design Add and Search Words Data Structure
Medium Trie DFS Backtracking

Problem

Design a data structure that supports addWord(word) and search(word). The search method supports the wildcard '.' which matches any single character.

Key Insight

Normal trie search is iterative. Once you add '.' wildcards, a single position can branch into ALL children. This forces a recursive DFS. When we hit '.', we try every child and return True if any path succeeds.

python – LC 211 
class WordDictionary:
    def __init__(self):
        self.children: dict = {}
        self.is_end: bool = False

    def addWord(self, word: str) -> None:
        node = self
        for ch in word:
            if ch not in node.children:
                node.children[ch] = WordDictionary()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        # We must use recursion because '.' can branch in multiple directions.
        node = self
        for i, ch in enumerate(word):
            if ch == '.':
                # '.' matches any character → try all existing children
                # For each child, recursively search the rest of the word
                return any(
                    child.search(word[i + 1:])
                    for child in node.children.values()
                )
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

# ── Example ──────────────────────────────────────────
# wd = WordDictionary()
# wd.addWord("bad")
# wd.addWord("dad")
# wd.addWord("mad")
# wd.search("pad")   → False
# wd.search("bad")   → True
# wd.search(".ad")   → True  ('.' matches 'b', 'd', or 'm')
# wd.search("b..")   → True  ('.' matches 'a', '..' matches 'ad')

# addWord: O(m)

LC #212
Word Search II
Hard Trie DFS Backtracking Matrix

Problem

Given an m × n board of characters and a list of words, find all words that can be constructed from sequentially adjacent cells (horizontally/vertically). Each cell may be used at most once per word.

Key Insight — Why Trie beats Brute Force

Naively searching for each word independently is O(words × m × n × 4^L). The Trie approach builds the trie from all words first, then does a single DFS from every cell. At each DFS step, we check if the current character exists in the current trie node — if not, we prune the entire subtree instantly. This is the "early termination" that makes Trie so powerful here.

Optimization: Delete found words from the trie After finding a word, remove it from the trie. This prevents finding duplicates AND prunes trie branches that are fully exhausted, making subsequent DFS calls faster.
python – LC 212 (optimized) 
from typing import List

class Solution:
    def findWords(self, board: List[List[str]], words: List[str]) -> List[str]:
        ROWS, COLS = len(board), len(board[0])

        # ── Build the Trie from all words ──────────────
        # We use a plain dict-based trie for speed (no class overhead).
        # '#' key marks end of word; its value stores the word itself.
        root = {}
        for word in words:
            node = root
            for ch in word:
                node = node.setdefault(ch, {})
            node['#'] = word          # Store the full word at the leaf

        result = []

        # ── DFS from every cell ────────────────────────
        def dfs(node: dict, r: int, c: int) -> None:
            ch = board[r][c]

            # This character is not a child of the current trie node → prune
            if ch not in node:
                return

            next_node = node[ch]

            # Found a complete word!
            if '#' in next_node:
                result.append(next_node.pop('#'))  # pop = remove to avoid duplicates

            # Mark cell as visited by temporarily replacing it
            board[r][c] = '*'

            # Explore all 4 adjacent cells
            for dr, dc in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
                nr, nc = r + dr, c + dc
                if 0 <= nr < ROWS and 0 <= nc < COLS and board[nr][nc] != '*':
                    dfs(next_node, nr, nc)

            # Restore the cell (backtrack)
            board[r][c] = ch

            # ── Pruning optimization ──────────────────
            # If next_node is now empty (no more words branch through it),
            # remove it from the parent. This shrinks the trie over time,
            # making future DFS calls faster.
            if not next_node:
                del node[ch]

        for r in range(ROWS):
            for c in range(COLS):
                dfs(root, r, c)

        return result

# ── Complexity ───────────────────────────────────────────
# Build trie:  O(W * L)  where W = num words, L = max word length
# DFS:         O(M * N * 4^L)  worst case — Trie pruning reduces this heavily
# Space:       O(W * L)  for the trie
LC #648
Replace Words
Medium Trie String Hash Table

Problem

Given a dictionary of root words and a sentence, replace each word in the sentence with the shortest root that is a prefix of that word. If multiple roots are prefixes, use the shortest one.

Example: roots = ["cat", "bat", "rat"], sentence = "the cattle was rattled by the battery"
Output: "the cat was rat by the bat"

Key Insight

Insert all roots into a Trie. For each word in the sentence, walk the trie character by character. The moment you hit a node marked is_end_of_word = True, you've found the shortest root — return it immediately. No need to walk further.

python – LC 648 
from typing import List

class Solution:
    def replaceWords(self, dictionary: List[str], sentence: str) -> str:
        # Build trie from roots (the dictionary)
        root = {}
        for word in dictionary:
            node = root
            for ch in word:
                node = node.setdefault(ch, {})
            node['#'] = True          # Mark end of root word

        def replace(word: str) -> str:
            node = root
            for i, ch in enumerate(word):
                if ch not in node:
                    # No root shares this prefix → keep original word
                    return word
                node = node[ch]
                if '#' in node:
                    # Found the shortest root that prefixes 'word'
                    # Return only the prefix portion (word[:i+1])
                    return word[:i + 1]
            return word               # word itself is not prefixed by any root

        # Process each word in the sentence
        return ' '.join(replace(word) for word in sentence.split())

# ── Complexity ──────────────────────────────────────
# Build trie: O(R * L)  where R = num roots, L = max root length
# Replace:    O(N * W)  where N = sentence words, W = avg word length
# Total:      O(R*L + N*W)
LC #1268
Search Suggestions System
Medium Trie Binary Search String

Problem

Given a list of products and a search word, return a list of lists — after each character of the search word is typed, return the top 3 products (lexicographically smallest) that start with the typed prefix.

Key Insight

Sort products first. Build a Trie where each node stores a list of up to 3 matching words. Since products are sorted before insertion, the first 3 stored at each node are always the 3 lexicographically smallest. After building, answer each prefix query in O(prefix_length).

python – LC 1268 
from typing import List

class Solution:
    def suggestedProducts(
        self, products: List[str], searchWord: str
    ) -> List[List[str]]:

        # Sort so that when we insert, the first 3 we see are lexicographically smallest
        products.sort()

        # Build Trie — each node stores at most 3 suggestions
        root = {}
        for product in products:
            node = root
            for ch in product:
                if ch not in node:
                    # Each node: { char: child_node, '$': [top_3_suggestions] }
                    node[ch] = {'$': []}
                node = node[ch]
                # Only keep the 3 smallest (list is already sorted by insertion order)
                if len(node['$']) < 3:
                    node['$'].append(product)

        # Answer each prefix query
        result = []
        node = root
        prefix_broken = False       # Once we leave the trie, all answers are []

        for ch in searchWord:
            if prefix_broken or ch not in node:
                result.append([])
                prefix_broken = True
            else:
                node = node[ch]
                result.append(node['$'])    # Already has top 3

        return result

# ── Complexity ──────────────────────────────────────
# Sort:       O(P * log P * L)   P = products, L = avg length
# Build:      O(P * L)
# Query:      O(|searchWord|)    — just walk the trie
# Total:      O(P*L*logP + |searchWord|)
LC #421
Maximum XOR of Two Numbers in an Array
Medium Bit Manipulation Binary Trie Array

Problem

Given an integer array nums, return the maximum result of nums[i] XOR nums[j] where 0 ≤ i ≤ j < n.

Key Insight — Binary Trie

This is a totally different kind of Trie — a Binary Trie that stores the binary representations of numbers (bit by bit, from MSB to LSB). To maximize XOR with a number x, for each bit of x, we want the opposite bit in the other number (since 1 XOR 0 = 1 and 0 XOR 1 = 1). The Binary Trie lets us greedily navigate to the number that maximizes XOR in O(32) time.

python – LC 421 (Binary Trie) 
from typing import List

class Solution:
    def findMaximumXOR(self, nums: List[int]) -> int:
        # Binary Trie stored as nested dicts
        # Keys at each node are 0 or 1 (bits)
        root = {}

        def insert(num: int) -> None:
            node = root
            # Process bits from MSB (bit 31) to LSB (bit 0)
            for i in range(31, -1, -1):
                bit = (num >> i) & 1      # Extract the i-th bit
                if bit not in node:
                    node[bit] = {}
                node = node[bit]

        def max_xor_with(num: int) -> int:
            """
            Find the number in the trie that maximizes XOR with 'num'.
            Greedy: at each bit position, prefer the OPPOSITE bit.
            """
            node = root
            xor_val = 0
            for i in range(31, -1, -1):
                bit = (num >> i) & 1
                want = 1 - bit            # Opposite bit maximizes XOR

                if want in node:
                    xor_val |= (1 << i)   # This bit contributes to XOR
                    node = node[want]
                else:
                    node = node[bit]      # Forced to take same bit (XOR = 0 here)

            return xor_val

        # Insert all numbers into the binary trie
        for num in nums:
            insert(num)

        # For each number, find its maximum XOR partner in the trie
        return max(max_xor_with(num) for num in nums)

# ── Complexity ──────────────────────────────────────
# Build:  O(N * 32) = O(N)
# Query:  O(N * 32) = O(N)
# Total:  O(N)  — linear!
# Space:  O(N * 32) = O(N)  for the binary trie

🌍Real-World Applications

🔍

Search Autocomplete

Google, YouTube, and IDEs use trie-based structures to suggest completions as you type. Each keystroke walks one level deeper in the trie.

📝

Spell Checkers

Word processors use tries to validate words and suggest corrections. A dictionary of 200,000 words can be stored and queried in microseconds.

🌐

IP Routing

Network routers use binary tries for longest-prefix matching. An IP address is a 32-bit binary string — the trie finds the best route in O(32).

🎮

Word Games

Scrabble solvers and Boggle solvers use tries to quickly validate words and prune invalid board paths (like in Word Search II).

🔒

T9 / Phone Keyboards

Old-school T9 and modern phone keyboards use tries to predict which word you're typing from a sequence of digit presses.

📦

Package Namespaces

Package managers (npm, pip) use trie-like structures for namespace resolution — quickly finding packages that match a given prefix.

🔀Trie Variations

1. Compressed Trie (Radix Tree / Patricia Tree)

A regular trie can waste space on long chains of nodes with only one child (e.g., storing "uniqueword" creates 10 nodes, each with one child). A Compressed Trie merges these chains into a single edge labeled with the entire substring.

Regular Trie
  • Simple implementation
  • Easy to understand
  • One node per character
  • Works with any alphabet
Compressed Trie (Radix Tree)
  • Fewer nodes — less memory
  • Complex split/merge on insert
  • Used in Linux kernel (mount points)
  • Used in route tables

2. Binary Trie

Used for bit manipulation problems. Instead of 26 character children, each node has only 2 children: 0 and 1. Numbers are stored bit-by-bit from MSB to LSB. Enables O(32) maximum XOR, minimum XOR, and range-based bit queries (as shown in LC 421).

3. Ternary Search Tree

A hybrid of a Trie and a Binary Search Tree. Each node has three children: less than, equal to, and greater than. More space-efficient than a regular trie for sparse data. Used in some spell-checker implementations.

4. Trie + Frequency Map (for Autocomplete Ranking)

Each trie node stores a frequency count (or a heap of the top-K words). When a word is searched/typed, its frequency increments. Autocomplete returns completions sorted by frequency rather than alphabetically — this is how most real search engines work.

python – frequency trie (autocomplete with ranking) 
import heapq
from collections import defaultdict

class FrequencyTrieNode:
    def __init__(self):
        self.children: dict = {}
        self.is_end: bool = False
        self.frequency: int = 0        # How often this word was inserted/searched

class FrequencyTrie:
    """
    A trie where each complete word stores a frequency.
    get_top_k(prefix, k) returns the k most-frequent words starting with prefix.
    """
    def __init__(self):
        self.root = FrequencyTrieNode()

    def insert(self, word: str, freq: int = 1) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = FrequencyTrieNode()
            node = node.children[ch]
        node.is_end = True
        node.frequency += freq         # Accumulate frequency on repeated inserts

    def _collect(self, node, prefix, results):
        if node.is_end:
            results.append((-node.frequency, prefix))  # Negate for min-heap trick
        for ch, child in node.children.items():
            self._collect(child, prefix + ch, results)

    def get_top_k(self, prefix: str, k: int = 3) -> list[tuple[str, int]]:
        """Return top-k words by frequency for a given prefix."""
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return []
            node = node.children[ch]

        results = []
        self._collect(node, prefix, results)

        # Use partial sort (heapq.nsmallest) — O(n + k log n)
        # Negated frequency, so smallest neg = highest freq
        top_k = heapq.nsmallest(k, results)
        return [(word, -freq) for freq, word in top_k]


# ── Example ──────────────────────────────────────────
ft = FrequencyTrie()
data = [("apple", 5), ("app", 3), ("application", 8),
        ("apply", 2), ("apt", 1), ("appetite", 6)]

for word, freq in data:
    ft.insert(word, freq)

print(ft.get_top_k("app", k=3))
# → [('application', 8), ('apple', 5), ('appetite', 6)]
# Note: 'appetite' starts with 'app'? No — it starts with 'app', 'e'...
# Let's trace: 'a','p','p','e','t','i','t','e' — yes 'app' is a prefix of 'appetite'
# Correct output: [('application', 8), ('appetite', 6), ('apple', 5)]
Summary: When to Use a Trie Use a Trie when: (1) you need prefix-based queries, (2) you're building autocomplete or spell-check, (3) you're solving board/grid problems where early path pruning is critical, (4) you're doing bit manipulation problems requiring maximum/minimum XOR. Prefer a Hash Table for simple exact-match lookups with no prefix requirements.
