Foundation

Binary Fundamentals

Every integer in memory is stored as binary — a sequence of 0s and 1s. Bit manipulation works directly on these bits, bypassing expensive arithmetic operations.

Decimal 13 → Binary 1101 Bit position: 7 6 5 4 3 2 1 0 Value: 0 0 0 0 1 1 0 1 Weight: 128 64 32 16 8 4 2 1 13 = 8 + 4 + 1 = 2³ + 2² + 2⁰ In Python: bin(13) → '0b1101' 13 & 0xFF → keep lowest 8 bits (mask) ~13 → -14 (two's complement: flip all bits, add 1) -13 in 8-bit two's complement = 11110011
Python's Integers Are Unbounded In Python, integers have arbitrary precision — there's no 32-bit overflow. This differs from C/Java where ~x = -(x+1) still holds but you must mask with & 0xFFFFFFFF to simulate 32-bit behavior for problems that specify it.

Two's Complement (Signed Integers)

Most languages use two's complement for negative numbers. The key property: to negate x, flip all bits and add 1.

Positive 5: 0000 0101 Negate: Step 1 — flip bits: 1111 1010 Step 2 — add 1: 1111 1011 = -5 This means: -x = ~x + 1 And: x + ~x = -1 (all bits set = -1 in two's complement)

Most Powerful Trick

XOR Magic

XOR is the most powerful bit manipulation tool for interviews. Memorize these four properties cold:

XOR Properties: ──────────────────────────────────── a ^ 0 = a (identity) a ^ a = 0 (self-inverse) a ^ b = b ^ a (commutative) (a ^ b) ^ c = a ^ (b ^ c) (associative) Consequence: if you XOR all numbers in a list, every number that appears twice cancels to 0, leaving only the number that appears once. Example: [2, 3, 2, 5, 3] 0 ^ 2 ^ 3 ^ 2 ^ 5 ^ 3 = (2^2) ^ (3^3) ^ 5 = 0 ^ 0 ^ 5 = 5 ✓

XOR Swap (No Temp Variable)

Python
# Swap two integers without a temp variable
a, b = 5, 9
a ^= b   # a = 5^9 = 12
b ^= a   # b = 9^12 = 5
a ^= b   # a = 12^5 = 9
# Now a=9, b=5  (swapped!)

# But in Python, prefer: a, b = b, a  (cleaner, same speed)

XOR to Find Unique Elements

Python
from functools import reduce

# Find the one element that appears once (all others appear twice)
def single_number(nums):
    return reduce(lambda a, b: a ^ b, nums)
    # Equivalently: result = 0; for n in nums: result ^= n

# Two unique elements (all others appear twice)
# Key insight: a^b != 0, so some bit differs between a and b
# Use that bit to separate nums into two groups, XOR each group
def single_number_iii(nums):
    xor = reduce(lambda a, b: a ^ b, nums)  # xor = a ^ b
    # Isolate the rightmost differing bit
    diff_bit = xor & (-xor)                 # lowest set bit trick
    a = 0
    for n in nums:
        if n & diff_bit:   # one group has bit set
            a ^= n
    b = xor ^ a
    return [a, b]
Interview Tip: XOR Canonical Uses
  • Find the single non-duplicate in an array of pairs
  • Detect if two values are different (a ^ b != 0)
  • Toggle a specific bit (x ^ (1 << i))
  • Compute parity of a number (XOR all bits together)
  • Check if two strings are anagrams or differ by one char

Classic Algorithm

Counting Set Bits

Brian Kernighan's Algorithm — O(k) where k = number of set bits

Instead of checking every bit, jump directly to set bits using n & (n-1) which clears the lowest set bit each iteration.

Count set bits in n = 13 (1101) Naive: check all 32 bits → O(32) = O(1) but slow constant Kernighan: only iterate over SET bits Iteration 1: n = 1101 → n & (n-1) = 1101 & 1100 = 1100 count=1 Iteration 2: n = 1100 → n & (n-1) = 1100 & 1011 = 1000 count=2 Iteration 3: n = 1000 → n & (n-1) = 1000 & 0111 = 0000 count=3 n = 0, done! popcount(13) = 3 ✓
Python — Three Ways to Count Bits
# Method 1: Brian Kernighan — O(k) per number
def count_bits_kernighan(n):
    count = 0
    while n:
        n &= n - 1   # clear lowest set bit
        count += 1
    return count

# Method 2: Built-in — O(1) in practice
def count_bits_builtin(n):
    return bin(n).count('1')
    # Or: return n.bit_count()  (Python 3.10+)

# Method 3: DP for range [0, n] — O(n) total using recurrence
# dp[i] = dp[i >> 1] + (i & 1)
# "the count of i equals count of i//2 plus whether i is odd"
def count_bits_range(n):
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        dp[i] = dp[i >> 1] + (i & 1)
    return dp

# Example: count_bits_range(5) → [0, 1, 1, 2, 1, 2]
DP Recurrence Intuition dp[i] = dp[i >> 1] + (i & 1)
Right-shifting drops the last bit. So i >> 1 has already been computed. The remaining question is whether i's last bit is 1 — checked by i & 1. This makes the DP O(n) with no inner loop.

Parity of a Number

Python
# Parity = 1 if odd number of set bits, 0 if even
# Use XOR: each set bit flips parity
def parity(n):
    result = 0
    while n:
        result ^= 1    # toggle parity
        n &= n - 1     # clear lowest set bit
    return result

# Faster O(log W) using XOR folding:
def parity_fast(n):
    n ^= n >> 16
    n ^= n >> 8
    n ^= n >> 4
    n ^= n >> 2
    n ^= n >> 1
    return n & 1
Advanced Application

Bitmask Patterns

A bitmask uses an integer to represent a set of boolean flags. Bit i being set means "element i is in the set." This enables O(1) set operations and is essential for DP on subsets.

Elements: [A, B, C, D] → 4 bits = 16 possible subsets {A, C} → 0101 = 5 {B, D} → 1010 = 10 {A,B,C,D}→ 1111 = 15 {} → 0000 = 0 Add D to {A,C}: 0101 | 1000 = 1101 = {A,C,D} Remove A from {A,C}: 0101 & ~0001 = 0101 & 1110 = 0100 = {C} Check if B in {A,C}: 0101 & 0010 = 0000 → False Union: {A,C} | {B,D} = 0101 | 1010 = 1111 Intersect: {A,C} & {B,D} = 0101 & 1010 = 0000

Enumerate All Subsets

Python — Bitmask Subset Enumeration
nums = ['A', 'B', 'C']
n = len(nums)

# Enumerate all 2^n subsets
for mask in range(1 << n):   # 0 to 2^n - 1
    subset = [nums[i] for i in range(n) if mask & (1 << i)]
    print(f"mask={bin(mask):<6} → {subset}")
# mask=0b0   → []
# mask=0b1   → ['A']
# mask=0b10  → ['B']
# mask=0b11  → ['A', 'B']
# mask=0b100 → ['C']  ...etc.

# Enumerate all subsets of a given mask (useful in DP)
def iter_subsets(mask):
    sub = mask
    while sub:
        yield sub
        sub = (sub - 1) & mask   # next smaller subset
    yield 0  # include empty set

Bitmask DP Example — Traveling Salesman (TSP)

Python
# Minimum cost to visit all cities exactly once
# dp[mask][i] = min cost to visit cities in 'mask', ending at city i
# Time: O(2^n * n^2)  Space: O(2^n * n)
def tsp(dist):
    n = len(dist)
    INF = float('inf')
    # Start at city 0
    dp = [[INF] * n for _ in range(1 << n)]
    dp[1][0] = 0  # mask=1 means only city 0 visited, at city 0

    for mask in range(1 << n):
        for u in range(n):
            if not (mask & (1 << u)) or dp[mask][u] == INF:
                continue
            for v in range(n):
                if mask & (1 << v):  # already visited
                    continue
                new_mask = mask | (1 << v)
                dp[new_mask][v] = min(dp[new_mask][v],
                                     dp[mask][u] + dist[u][v])

    full = (1 << n) - 1
    return min(dp[full][i] + dist[i][0] for i in range(n))
When to Use Bitmask DP Use bitmask DP when n ≤ 20 and you need to track which subset of elements has been used. Classic problems: TSP, minimum cost to assign tasks, word break with all visited words, Hamiltonian path.

Bitwise AND of a Range

A classic trick: to find m & (m+1) & ... & n, note that all differing lower bits get zeroed out. The answer is the common prefix of m and n in binary.

Python
# Bitwise AND of all numbers from m to n
def range_bitwise_and(m: int, n: int) -> int:
    shift = 0
    # Shift both right until they're equal (finding common prefix)
    while m != n:
        m >>= 1
        n >>= 1
        shift += 1
    return m << shift  # shift common prefix back

# Why it works:
# [5..7] = 101, 110, 111
# m=5=101, n=7=111  →  shift until equal
# 5>>1=2, 7>>1=3  → not equal
# 2>>1=1, 3>>1=1  → equal! shift=2
# return 1 << 2 = 4 = 100  ✓ (101 & 110 & 111 = 100 = 4)

#136 Single Number Easy

Given a non-empty array where every element appears twice except for one, find that single one. Must be O(n) time and O(1) space.

Pattern: XOR Self-Inverse XOR every number together. Pairs cancel (a^a=0). The survivor is the answer.
Python
def singleNumber(nums):
    # XOR all numbers together; pairs cancel out
    result = 0
    for n in nums:
        result ^= n
    return result

# One-liner using functools.reduce
from functools import reduce
def singleNumber(nums):
    return reduce(lambda a, b: a ^ b, nums)

# Walkthrough: [4, 1, 2, 1, 2]
# 0^4=4, 4^1=5, 5^2=7, 7^1=6, 6^2=4 → 4 ✓
⏱ O(n) time💾 O(1) space🔑 XOR cancellation
#191 Number of 1 Bits Easy

Return the number of '1' bits (popcount / Hamming weight) in the binary representation of a positive integer.

Pattern: Brian Kernighan — Clear Lowest Set Bit n & (n-1) clears the rightmost 1-bit. Count how many times you do this before n=0.
Python
def hammingWeight(n: int) -> int:
    # Brian Kernighan's algorithm
    count = 0
    while n:
        n &= n - 1   # clears the lowest set bit
        count += 1
    return count

# Alternative: Python built-ins
def hammingWeight_v2(n: int) -> int:
    return bin(n).count('1')
    # or: return n.bit_count()  (Python 3.10+)

# Trace for n = 13 (1101):
# 1101 & 1100 = 1100 → count=1
# 1100 & 1011 = 1000 → count=2
# 1000 & 0111 = 0000 → count=3  ✓
⏱ O(k) time, k=set bits💾 O(1) space🔑 n&(n-1) trick
#338 Counting Bits Easy

Return an array ans of length n+1 such that ans[i] is the number of 1's in the binary representation of i, for i in [0, n].

Pattern: DP with Right-Shift Recurrence dp[i] = dp[i >> 1] + (i & 1) — shift removes last bit (already computed), add 1 if last bit was set.
Python
def countBits(n: int) -> list[int]:
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        dp[i] = dp[i >> 1] + (i & 1)
        # dp[i>>1]: bit count of i with last bit removed
        # (i&1): whether last bit is set
    return dp

# Example: n = 5
# i=0: 0
# i=1: dp[0]+1 = 1    (1 → 1 set bit)
# i=2: dp[1]+0 = 1    (10 → 1 set bit)
# i=3: dp[1]+1 = 2    (11 → 2 set bits)
# i=4: dp[2]+0 = 1    (100 → 1 set bit)
# i=5: dp[2]+1 = 2    (101 → 2 set bits)
# → [0, 1, 1, 2, 1, 2]
⏱ O(n) time💾 O(n) space🔑 DP recurrence
#190 Reverse Bits Easy

Reverse the bits of a given 32-bit unsigned integer.

Pattern: Extract & Place Extract each bit from the right of n, place it in the corresponding position from the left of the result.
Python
def reverseBits(n: int) -> int:
    result = 0
    for _ in range(32):
        result = (result << 1) | (n & 1)  # append LSB of n to result
        n >>= 1                             # shift n right to process next bit
    return result

# Trace for n = 0b00000010100101000001111010011100
# Each iteration: grab LSB of n → push into result

# If called many times, cache with lookup table (divide 32-bit into
# 4 groups of 8 bits, reverse each byte, concatenate reversed)
def reverseBits_cached(n: int) -> int:
    # Reverse 8-bit chunks and recombine
    def reverse_byte(b):
        result = 0
        for _ in range(8):
            result = (result << 1) | (b & 1)
            b >>= 1
        return result
    return (reverse_byte(n & 0xFF) << 24 |
            reverse_byte((n >> 8) & 0xFF) << 16 |
            reverse_byte((n >> 16) & 0xFF) << 8 |
            reverse_byte((n >> 24) & 0xFF))
⏱ O(1) — 32 iterations💾 O(1) space🔑 Extract LSB, build result
#268 Missing Number Easy

Given an array nums containing n distinct numbers in range [0, n], return the one missing number.

Two Patterns (1) XOR: XOR all indices 0..n and all values; missing number survives. (2) Math: expected sum - actual sum.
Python
# Method 1: XOR
def missingNumber_xor(nums):
    n = len(nums)
    result = n  # start with n since we loop 0..n-1 below
    for i, num in enumerate(nums):
        result ^= i ^ num
        # Each present number cancels with its index
        # Missing index survives
    return result

# Method 2: Gauss sum formula (simpler to remember)
def missingNumber_math(nums):
    n = len(nums)
    expected = n * (n + 1) // 2
    return expected - sum(nums)

# Both are O(n) time, O(1) space
# XOR is overflow-safe (better for very large n in languages with fixed int size)
⏱ O(n) time💾 O(1) space🔑 XOR index vs value
#371 Sum of Two Integers Medium

Calculate the sum of two integers without using + or -.

Pattern: Half Adder Simulation XOR gives sum without carry. AND+shift gives the carry. Repeat until no carry.
Half adder truth table: 0+0=0 carry 0 | a^b=0, a&b=0 0+1=1 carry 0 | a^b=1, a&b=0 1+0=1 carry 0 | a^b=1, a&b=0 1+1=0 carry 1 | a^b=0, a&b=1 → carry shifts left by 1 add(3, 5): 011 ^ 101 = 110 (sum bits), 011 & 101 = 001 → 010 (carry) add(6, 2): 110 ^ 010 = 100 (sum bits), 110 & 010 = 010 → 100 (carry) add(4, 4): 100 ^ 100 = 000 (sum bits), 100 & 100 = 100 → 1000 (carry... wait) add(0, 8): 000 ^ 1000 = 1000, no carry → 8 ✓
Python
def getSum(a: int, b: int) -> int:
    # Python has arbitrary precision integers, so we must mask to 32 bits
    MASK = 0xFFFFFFFF   # 32-bit mask
    MAX  = 0x7FFFFFFF   # max positive 32-bit int

    while b & MASK:
        carry = (a & b) << 1   # carry bits
        a = a ^ b               # sum without carry
        b = carry
        a &= MASK               # keep only 32 bits
        b &= MASK

    # If a > MAX, it's a negative number in two's complement
    return a if a <= MAX else ~(a ^ MASK)

# Key insight:
# XOR  = addition without carry
# AND  = carry positions
# Shift carry left by 1 (carry propagates one bit left)
# Repeat until carry is 0
⏱ O(1) — at most 32 iterations💾 O(1) space🔑 XOR=sum, AND+shift=carry
#201 Bitwise AND of Numbers Range Medium

Return the bitwise AND of all numbers in range [left, right] inclusive.

Pattern: Common Prefix Any bit that differs between left and right will hit both 0 and 1 in the range, so ANDing zeros it out. The answer is the common binary prefix of left and right.
Python
def rangeBitwiseAnd(left: int, right: int) -> int:
    shift = 0
    # Shift both right until they share the same prefix (become equal)
    while left != right:
        left  >>= 1
        right >>= 1
        shift += 1
    # Shift common prefix back to the original position
    return left << shift

# Example: left=5 (101), right=7 (111)
# 101 != 111 → shift: 10, 11 → shift: 1, 1 → equal! shift=2
# return 1 << 2 = 4 = 100
# Verify: 5&6&7 = 101&110&111 = 100 = 4 ✓

# Alternative: use Brian Kernighan to strip lower bits of right
def rangeBitwiseAnd_v2(left: int, right: int) -> int:
    while right > left:
        right &= right - 1  # strip lowest set bit of right
    return right
⏱ O(log n)💾 O(1) space🔑 Common binary prefix
#137 Single Number II Medium

Every element appears exactly three times except for one element which appears exactly once. Find that element. O(1) space required.

Pattern: Bit Count Modulo 3 For each bit position, count how many numbers have that bit set. The unique number contributes an odd count to each of its set bits mod 3.
Python
def singleNumber(nums) -> int:
    # Method 1: Count bits mod 3 at each bit position
    result = 0
    for bit in range(32):
        total = sum((n >> bit) & 1 for n in nums)
        if total % 3:              # this bit belongs to the unique number
            result |= (1 << bit)

    # Handle negative numbers (Python arbitrary precision)
    if result >= (1 << 31):
        result -= (1 << 32)
    return result

# Method 2: State machine (O(1) space, no 32-bit loop)
# Track 'ones' (bits seen 1 mod 3 times) and 'twos' (bits seen 2 mod 3 times)
def singleNumber_v2(nums) -> int:
    ones, twos = 0, 0
    for n in nums:
        ones = (ones ^ n) & ~twos   # add to 'seen once', remove if in 'twos'
        twos = (twos ^ n) & ~ones   # add to 'seen twice', remove if in 'ones'
    return ones  # bits seen exactly once remain in 'ones'

# Why it works: a 2-bit counter per bit position:
# count 0 → ones=0, twos=0
# count 1 → ones=1, twos=0
# count 2 → ones=0, twos=1
# count 3 → ones=0, twos=0 (resets)
⏱ O(n) time💾 O(1) space🔑 Bit count mod 3
