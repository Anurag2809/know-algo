Intervals

Interval problems are deceptively uniform — nearly all of them follow the same recipe: sort by start time, then apply a greedy strategy. Master the 4 core patterns and you can solve the entire Blind 75 + NeetCode 150 interval category on autopilot.

📅 Sort → Greedy 🏢 Meeting Rooms ✂️ Non-Overlapping 🎯 7 LeetCode problems
Foundation

The Sorting Prerequisite

Almost every interval problem starts with sorting by start time. Once sorted, you only need to check adjacent intervals — any non-adjacent intervals either don't overlap (they're already separated) or their overlap is captured by transitivity through the middle one.

Unsorted: [[1,3],[6,9],[2,6],[8,10],[15,18]] Sort by start: [1,3] [2,6] [6,9] [8,10] [15,18] |------| ← check: 2 ≤ 3 → overlap with [1,3] |------| ← check: 6 ≤ 6 → overlap with merged [1,6] |-------| ← check: 8 ≤ 9 → overlap with [6,9] |--------| ← check: 15 > 10 → no overlap Result: [[1,10],[15,18]] Key: once sorted, we only compare with the LAST merged interval. We never need to look further back.
Two sorting strategies — choose based on the problem:
  • Sort by start time → Merge intervals, Insert interval, Meeting Rooms II
  • Sort by end time → Non-Overlapping (maximize kept), Interval scheduling (greedy)
Pattern 1

Merge Overlapping Intervals

Given a list of intervals, merge all overlapping ones. Two intervals [a,b] and [c,d] overlap when c ≤ b (the next starts before the current ends). When they overlap, the merged interval is [a, max(b,d)].

Python
def merge(intervals):          # LC #56
    intervals.sort(key=lambda x: x[0])   # sort by start time
    merged = [intervals[0]]

    for start, end in intervals[1:]:
        last_end = merged[-1][1]
        if start <= last_end:
            # Overlap: extend the last interval's end if necessary
            merged[-1][1] = max(last_end, end)
        else:
            # Gap: start a new interval
            merged.append([start, end])

    return merged

# [[1,3],[2,6],[8,10],[15,18]]
# → sort → [[1,3],[2,6],[8,10],[15,18]] (already sorted)
# Step 1: last=[1,3], cur=[2,6] → 2<=3 → merge → [1,6]
# Step 2: last=[1,6], cur=[8,10] → 8>6  → new  → [1,6],[8,10]
# Step 3: last=[8,10], cur=[15,18] → 15>10 → new → [1,6],[8,10],[15,18] ✓
Why max(last_end, end) and not just end? One interval might be completely contained within another: [1,10] followed by [2,5]. Here 2 ≤ 10, so they overlap, but the merged end should remain 10, not shrink to 5. The max handles containment.
Pattern 2

Insert Interval

Given a sorted list of non-overlapping intervals and a new interval, insert it in the right place and merge. The key insight: divide the existing intervals into three groups relative to the new interval.

existing = [[1,2],[3,5],[6,7],[8,10],[12,16]] newInterval = [4,8] Group 1: intervals that END before new starts (no overlap, keep as-is) [1,2]: 2 < 4 ✓ → keep [3,5]: 5 < 4? NO → stop Group 2: intervals that OVERLAP with new (merge all into new) [3,5]: start(3) ≤ end(8) AND end(5) ≥ start(4) → merge → new=[3,8] [6,7]: start(6) ≤ end(8) → merge → new=[3,8] [8,10]: start(8) ≤ end(8) → merge → new=[3,10] [12,16]: start(12) > end(10) → stop Group 3: intervals that START after new ends (no overlap, keep as-is) [12,16] → keep Result: [[1,2],[3,10],[12,16]]
Python
def insert(intervals, newInterval):    # LC #57
    result = []
    i = 0
    n = len(intervals)
    ns, ne = newInterval

    # Phase 1: add all intervals that end before new starts
    while i < n and intervals[i][1] < ns:
        result.append(intervals[i])
        i += 1

    # Phase 2: merge all overlapping intervals into newInterval
    while i < n and intervals[i][0] <= ne:
        ns = min(ns, intervals[i][0])
        ne = max(ne, intervals[i][1])
        i += 1
    result.append([ns, ne])

    # Phase 3: add remaining intervals
    while i < n:
        result.append(intervals[i])
        i += 1

    return result
Overlap condition: Interval [a,b] and [c,d] do NOT overlap if b < c (b ends before c starts) or d < a (d ends before a starts). They overlap in all other cases. For Phase 2, we continue while intervals[i][0] ≤ ne — the next interval's start is within our current merged range.
Pattern 3

Meeting Rooms: Minimum Resources

How many conference rooms (or resources) do you need to schedule all meetings? This equals the maximum number of meetings occurring simultaneously at any point.

Approach 1: Min-Heap (O(n log n))

Sort by start time. Use a min-heap tracking end times of ongoing meetings. When a new meeting starts, check if any room has freed up (heap's min ≤ current start). If yes, reuse it; otherwise allocate a new room.

Python
import heapq

def minMeetingRooms(intervals):      # LC #253
    if not intervals:
        return 0
    intervals.sort(key=lambda x: x[0])
    # Min-heap of end times (each entry = end time of a room in use)
    heap = []

    for start, end in intervals:
        if heap and heap[0] <= start:
            # A room ended before this meeting starts → reuse it
            heapq.heapreplace(heap, end)   # pop old, push new end time
        else:
            # No room available → allocate new room
            heapq.heappush(heap, end)

    return len(heap)   # heap size = rooms in use at peak

# [[0,30],[5,10],[15,20]]
# sort → same
# [0,30]: heap=[] → push → heap=[30]
# [5,10]: heap[0]=30 > 5 → new → heap=[10,30]
# [15,20]: heap[0]=10 ≤ 15 → reuse → heap=[20,30]
# return 2 ✓

Approach 2: Sweep Line / Events (O(n log n), more general)

Create "start" and "end" events, sort them, sweep through counting active meetings. The maximum count reached is the answer. This generalizes to any resource counting problem.

Python
def minMeetingRoomsSweep(intervals):
    events = []
    for start, end in intervals:
        events.append((start, 1))    # +1 at start
        events.append((end, -1))     # -1 at end

    # Sort: on tie, process end (-1) before start (+1) so [0,5],[5,10]
    # doesn't count as two overlapping → sort by (time, type)
    events.sort(key=lambda x: (x[0], x[1]))

    active = max_rooms = 0
    for time, delta in events:
        active += delta
        max_rooms = max(max_rooms, active)
    return max_rooms
Pattern 4

Non-Overlapping: Maximize Kept Intervals

Given intervals, find the minimum number to remove so no two overlap. Equivalently: maximize the number of intervals you can keep. This is the classic Activity Selection problem.

Greedy insight: Sort by end time. Always keep the interval that ends earliest — it leaves the most room for future intervals. Greedily pick non-overlapping intervals, counting removals.

intervals = [[1,2],[2,3],[3,4],[1,3]] Sort by END time: [1,2] [2,3] [1,3] [3,4] ↑ keep first (ends earliest) Step 1: keep [1,2], last_end = 2 Step 2: [2,3] start=2 >= last_end=2 → keep, last_end=3 Step 3: [1,3] start=1 < last_end=3 → REMOVE (overlaps, ends same/later) Step 4: [3,4] start=3 >= last_end=3 → keep, last_end=4 Removed = 1 ✓ Why sort by end, not start? Sorting by start might pick a very long interval first, blocking many future ones. Ending earliest leaves the most time open.
Python
def eraseOverlapIntervals(intervals):  # LC #435
    intervals.sort(key=lambda x: x[1])  # sort by END time
    removed = 0
    last_end = float('-inf')

    for start, end in intervals:
        if start >= last_end:
            # No overlap with last kept → keep this interval
            last_end = end
        else:
            # Overlap → remove (count it, keep the one ending earlier
            # which is already tracked via last_end)
            removed += 1

    return removed

Minimum Arrows to Burst Balloons (same pattern)

Balloons are intervals on a number line. An arrow shot at position x bursts all balloons whose range contains x. Minimum arrows = minimum number of non-overlapping groups = same greedy by end time.

Python
def findMinArrowShots(points):         # LC #452
    points.sort(key=lambda x: x[1])   # sort by end
    arrows = 1
    arrow_pos = points[0][1]           # shoot first arrow at earliest end

    for start, end in points[1:]:
        if start > arrow_pos:          # balloon not hit → new arrow
            arrows += 1
            arrow_pos = end            # shoot at this balloon's end
        # else: this balloon is hit by the current arrow (start <= arrow_pos)

    return arrows

# Key difference from eraseOverlapIntervals:
# [1,2],[2,3] — arrow at 2 bursts BOTH (touching counts as burst)
# so we use > not >= in the condition
Pattern 5

Interval List Intersections

Given two sorted lists of non-overlapping intervals, find all intersections. Use two pointers — advance the pointer of the interval that ends first, since it can't intersect any future intervals from the other list.

Python
def intervalIntersection(A, B):       # LC #986
    i = j = 0
    result = []

    while i < len(A) and j < len(B):
        # Intersection of A[i]=[a,b] and B[j]=[c,d] exists iff max(a,c) <= min(b,d)
        lo = max(A[i][0], B[j][0])
        hi = min(A[i][1], B[j][1])

        if lo <= hi:
            result.append([lo, hi])   # valid intersection

        # Advance the pointer with the smaller endpoint
        # (that interval is "done" — it can't intersect anything beyond it)
        if A[i][1] < B[j][1]:
            i += 1
        else:
            j += 1

    return result

# A=[[0,2],[5,10],[13,23],[24,25]]
# B=[[1,5],[8,12],[15,24],[25,26]]
# → [[1,2],[5,5],[8,10],[15,23],[24,24],[25,25]]
Advanced

Sweep Line: General Framework

The sweep line generalizes all interval problems. Model each interval as two events (start, +1) and (end, -1), sort by time, and sweep. The "height" (running count) at any point tells you the depth of overlap.

Python
def sweepLine(intervals):
    """Find the time of maximum overlap and its depth."""
    events = []
    for start, end in intervals:
        events.append((start, 1))    # +1: interval opens
        events.append((end, -1))     # -1: interval closes

    events.sort()                    # sort by time; -1 before +1 on ties (alphabetically)

    depth = max_depth = 0
    for time, delta in events:
        depth += delta
        max_depth = max(max_depth, depth)
    return max_depth

# Generalizes to:
# - Count active rentals at peak time
# - Maximum concurrent users
# - Minimum platforms needed at a train station
# - Employee availability / scheduling

# For queries: "for each query point, how many intervals contain it?"
# Sort queries + events together, process in order.

Python
def merge(intervals):
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:

#57Insert IntervalMedium

Insert a new interval into a sorted non-overlapping list, merging as necessary.

Python
def insert(intervals, newInterval):
    res, i, n = [], 0, len(intervals)
    ns, ne = newInterval
    while i < n and intervals[i][1] < ns:   # before new
        res.append(intervals[i]); i += 1
    while i < n and intervals[i][0] <= ne:   # overlapping: merge
        ns = min(ns, intervals[i][0])
        ne = max(ne, intervals[i][1]); i += 1
    res.append([ns, ne])
    while i < n:                             # after new
        res.append(intervals[i]); i += 1
    return res
⏱ O(n)💾 O(n)Pattern: 3-phase scan

def minMeetingRooms(intervals):

#1851Minimum Interval to Include Each QueryHard

For each query point, find the size of the smallest interval that contains it. Return -1 if none exists.

Insight: Sort intervals by start, sort queries. Process queries in ascending order — use a min-heap of (size, end) for intervals whose start ≤ query. Before answering, remove expired intervals (end < query) from the heap.

Python
import heapq
def minInterval(intervals, queries):
    intervals.sort()                        # by start
    sorted_q = sorted(enumerate(queries), key=lambda x: x[1])  # sorted queries with original index
    ans = [-1] * len(queries)
    heap = []   # (size, end) — min by size
    i = 0

    for idx, q in sorted_q:
        # Add all intervals starting at or before q
        while i < len(intervals) and intervals[i][0] <= q:
            s, e = intervals[i]
            heapq.heappush(heap, (e - s + 1, e))
            i += 1
        # Remove intervals that have ended before q
        while heap and heap[0][1] < q:
            heapq.heappop(heap)
        # Smallest remaining interval (if any) is the answer
        if heap:
            ans[idx] = heap[0][0]

    return ans
⏱ O((n+q) log n)💾 O(n+q)Pattern: Sort intervals + queries + min-heap sweep
