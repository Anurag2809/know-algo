Algorithms

Backtracking

The systematic "try everything" strategy. Backtracking builds a solution incrementally and abandons a candidate the moment it can't lead to a valid solution. It's brute force — but smart brute force, guided by constraints.

🌳 Decision Tree ✂️ Pruning 🔄 Choose / Explore / Unchoose 🎯 8 LeetCode problems
Foundation

The Backtracking Template

Every backtracking solution has the same three steps, repeated recursively. Think of it as navigating a decision tree: at each node, you make a choice, recurse into the subtree, then undo your choice (backtrack) to try the next option.

The three-step loop:
  1. Choose: Pick the next candidate to try
  2. Explore: Recurse — make the choice and continue building
  3. Unchoose: Undo the choice to restore state for the next candidate
Python
def backtrack(state, choices):
    if is_solution(state):
        record(state)              # base case: valid complete solution found
        return

    for choice in choices:
        if is_valid(state, choice):
            make_choice(state, choice)       # CHOOSE
            backtrack(state, next_choices)   # EXPLORE
            undo_choice(state, choice)       # UNCHOOSE (backtrack)

The Decision Tree

Every backtracking problem maps to a tree where each node is a partial solution and each edge is a choice. The algorithm does a DFS, collecting all paths that reach a leaf satisfying the constraints.

Example: Subsets of [1, 2, 3] [] / | \ [1] [2] [3] / \ | [1,2] [1,3] [2,3] | [1,2,3] Each node = partial solution Leaf nodes (when start_index > n) = collected as results → Total 8 subsets (2^3 = 2^n) Pruning: cut branches that can't lead to valid solutions (e.g., in combination sum, stop if running total > target)
Backtracking vs Dynamic Programming: Both explore subproblems, but DP stores subproblem results to avoid recomputation (overlapping subproblems). Backtracking doesn't — it explores all branches but prunes invalid ones. Use DP when there's optimal substructure and overlapping subproblems; use backtracking when you need all solutions or the problem has constraints that vary per path.
Pattern 1

Subsets (Power Set)

Generate all subsets of a set. The key insight: for each element, you have a binary choice — include it or exclude it. This gives 2ⁿ subsets total.

Approach 1: Include/Exclude at Each Index

Python
def subsets(nums):            # LC #78
    result = []

    def backtrack(start, current):
        result.append(current[:])    # snapshot: every node is a valid subset

        for i in range(start, len(nums)):
            current.append(nums[i])          # CHOOSE: include nums[i]
            backtrack(i + 1, current)        # EXPLORE: recurse with next elements
            current.pop()                    # UNCHOOSE: remove nums[i]

    backtrack(0, [])
    return result

# nums = [1, 2, 3] → [[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]]
# Note: start=i+1 ensures we only look forward → no duplicates

Subsets with Duplicates

When the input has duplicates (LC #90), sort first, then skip duplicate elements at the same recursion level:

Python
def subsetsWithDup(nums):      # LC #90
    nums.sort()                # sort so duplicates are adjacent
    result = []

    def backtrack(start, current):
        result.append(current[:])
        for i in range(start, len(nums)):
            # Skip duplicate at the same level: if nums[i] == nums[i-1]
            # and i > start, we already explored this branch
            if i > start and nums[i] == nums[i - 1]:
                continue
            current.append(nums[i])
            backtrack(i + 1, current)
            current.pop()

    backtrack(0, [])
    return result

# nums = [1, 2, 2] → [[], [1], [1,2], [1,2,2], [2], [2,2]]
# Without skip: [1,2] would appear twice
The duplicate-skipping rule: After sorting, identical elements appear consecutively. At any given recursion depth (same start position), if we see nums[i] == nums[i-1] with i > start, we've already explored the branch with nums[i-1] at this position — skipping avoids generating duplicate subsets.
Pattern 2

Permutations

Generate all orderings of a list. Unlike subsets, every element appears exactly once and order matters. A "used" boolean array tracks which elements are already in the current permutation.

All Permutations (No Duplicates)

Python
def permute(nums):             # LC #46
    result = []
    used = [False] * len(nums)

    def backtrack(current):
        if len(current) == len(nums):
            result.append(current[:])  # complete permutation
            return

        for i in range(len(nums)):
            if used[i]:
                continue               # already placed in current permutation
            used[i] = True
            current.append(nums[i])    # CHOOSE
            backtrack(current)         # EXPLORE
            current.pop()              # UNCHOOSE
            used[i] = False

    backtrack([])
    return result

# Alternative: swap-based (modifies nums in-place, no "used" array)
def permuteSwap(nums):
    result = []
    def backtrack(start):
        if start == len(nums):
            result.append(nums[:])
            return
        for i in range(start, len(nums)):
            nums[start], nums[i] = nums[i], nums[start]  # CHOOSE: swap
            backtrack(start + 1)                          # EXPLORE
            nums[start], nums[i] = nums[i], nums[start]  # UNCHOOSE: swap back
    backtrack(0)
    return result

Permutations with Duplicates

Python
def permuteUnique(nums):       # LC #47
    nums.sort()
    result = []
    used = [False] * len(nums)

    def backtrack(current):
        if len(current) == len(nums):
            result.append(current[:])
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            # Skip duplicate: if nums[i] == nums[i-1] and nums[i-1] is NOT used,
            # then we're at the same level and about to generate the same permutation
            if i > 0 and nums[i] == nums[i - 1] and not used[i - 1]:
                continue
            used[i] = True
            current.append(nums[i])
            backtrack(current)
            current.pop()
            used[i] = False

    backtrack([])
    return result

# nums=[1,1,2]:  only [1,1,2], [1,2,1], [2,1,1] — not 6 permutations
Why not used[i-1]? We want to ensure that among duplicate values, we always place the leftmost occurrence first. If nums[i-1] is not currently used, it means we're starting a new branch at the same level — placing nums[i] (a duplicate) here would create an identical arrangement to one we already generated via nums[i-1].
Pattern 3

Combinations & Combination Sum

Combinations choose k elements from n where order doesn't matter. Combination Sum extends this to allow repeated use of elements to reach a target sum.

Basic Combinations (LC #77)

Python
def combine(n, k):             # LC #77: choose k numbers from [1..n]
    result = []

    def backtrack(start, current):
        if len(current) == k:
            result.append(current[:])
            return
        # Pruning: need at least (k - len(current)) more elements
        # so start can't exceed n - (k - len(current)) + 1
        remaining = k - len(current)
        for i in range(start, n - remaining + 2):
            current.append(i)
            backtrack(i + 1, current)
            current.pop()

    backtrack(1, [])
    return result

Combination Sum (Unlimited Reuse)

Python
def combinationSum(candidates, target):   # LC #39
    result = []

    def backtrack(start, current, remaining):
        if remaining == 0:
            result.append(current[:])  # exact match
            return
        if remaining < 0:
            return                     # overshot — prune

        for i in range(start, len(candidates)):
            current.append(candidates[i])
            # i (not i+1) allows reuse of the same element
            backtrack(i, current, remaining - candidates[i])
            current.pop()

    backtrack(0, [], target)
    return result

Combination Sum II (No Reuse, with Duplicates)

Python
def combinationSum2(candidates, target):  # LC #40
    candidates.sort()                     # sort to handle duplicates
    result = []

    def backtrack(start, current, remaining):
        if remaining == 0:
            result.append(current[:])
            return

            if candidates[i] > remaining:
                break                     # sorted → rest are too large (prune)
            # Skip duplicates at the same recursion level
            if i > start and candidates[i] == candidates[i - 1]:
                continue
            current.append(candidates[i])
            backtrack(i + 1, current, remaining - candidates[i])  # i+1: no reuse
            current.pop()

    backtrack(0, [], target)
    return result

Pattern 4

Grid Backtracking

Backtracking on a 2D grid explores paths cell-by-cell. Instead of a "used" array, you temporarily mark cells as visited in the grid itself (to avoid revisiting), then unmark on backtrack.

Word Search (LC #79)

Find if a word exists in a grid by walking adjacent cells (up/down/left/right), never reusing the same cell in one path.

Python
def exist(board, word):        # LC #79
    rows, cols = len(board), len(board[0])

    def dfs(r, c, k):
        if k == len(word):
            return True                    # found complete word
        if not (0 <= r < rows and 0 <= c < cols):
            return False                   # out of bounds
        if board[r][c] != word[k]:
            return False                   # wrong character

        # CHOOSE: mark cell as visited (in-place)
        temp = board[r][c]
        board[r][c] = '#'                  # sentinel: won't match any letter

        # EXPLORE: try all 4 directions
        found = (dfs(r+1, c, k+1) or dfs(r-1, c, k+1) or
                 dfs(r, c+1, k+1) or dfs(r, c-1, k+1))

        # UNCHOOSE: restore cell
        board[r][c] = temp
        return found

    for r in range(rows):
        for c in range(cols):
            if dfs(r, c, 0):              # try every starting cell
                return True
    return False

Why In-Place Marking Works

Overwriting with a sentinel character during recursion and restoring on backtrack gives us O(1) space for the visited state (no auxiliary set). This works because the grid is modified and restored symmetrically along each DFS path.

Pruning for Word Search: Early termination is key. If the current cell doesn't match word[k], return immediately — don't even explore neighbors. Additionally, if the remaining characters needed exceed the remaining unvisited cells, prune (count remaining cells).
Pattern 5

Constraint Satisfaction

Some problems have global constraints that must be maintained across all positions simultaneously — not just locally within one path. These require checking validity across rows, columns, and regions.

N-Queens (LC #51)

Place N queens on an N×N chessboard so no two queens attack each other (same row, column, or diagonal).

Python
def solveNQueens(n):            # LC #51
    result = []
    # Track which columns and diagonals are occupied
    cols = set()
    diag1 = set()   # r - c = constant for "/" diagonals
    diag2 = set()   # r + c = constant for "\" diagonals
    board = [['.' ] * n for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append(["".join(r) for r in board])
            return

        for col in range(n):
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue               # queen would be attacked → prune

            # CHOOSE: place queen
            board[row][col] = 'Q'
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)

            backtrack(row + 1)         # EXPLORE: next row

            # UNCHOOSE: remove queen
            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    backtrack(0)
    return result

# Key insight: one queen per row → backtrack row by row
# Constraint check in O(1) with sets instead of O(n)

Sudoku Solver (LC #37)

Fill a 9×9 sudoku board so each row, column, and 3×3 box contains digits 1-9 exactly once.

Python
def solveSudoku(board):        # LC #37 (modifies board in-place)
    rows = [set() for _ in range(9)]
    cols = [set() for _ in range(9)]
    boxes = [set() for _ in range(9)]

    # Pre-populate constraint sets from existing clues
    for r in range(9):
        for c in range(9):
            if board[r][c] != '.':
                d = board[r][c]
                rows[r].add(d); cols[c].add(d)
                boxes[(r // 3) * 3 + c // 3].add(d)

    def backtrack():
        # Find the next empty cell
        for r in range(9):
            for c in range(9):
                if board[r][c] == '.':
                    box_id = (r // 3) * 3 + c // 3
                    for d in '123456789':
                        if d in rows[r] or d in cols[c] or d in boxes[box_id]:
                            continue               # digit already used → skip
                        # CHOOSE
                        board[r][c] = d
                        rows[r].add(d); cols[c].add(d); boxes[box_id].add(d)
                        # EXPLORE
                        if backtrack():
                            return True            # propagate success
                        # UNCHOOSE
                        board[r][c] = '.'
                        rows[r].remove(d); cols[c].remove(d); boxes[box_id].remove(d)
                    return False                   # no valid digit → backtrack
        return True                                # no empty cell found → solved

    backtrack()
Constraint propagation (advanced): Instead of trying all 9 digits for each cell, pick the cell with the fewest valid choices (MRV — Minimum Remaining Values heuristic). This can reduce the branching factor dramatically. Python's set-based constraint tracking enables O(1) validity checks vs O(n) scanning.
Optimization

Pruning Strategies

Backtracking without pruning is brute force. Pruning is what makes it efficient. Here are the key strategies, ordered from most to least impactful:

Early Termination

Return immediately when a constraint is violated. Don't wait until the leaf. Example: in combination sum, if remaining < 0, prune immediately.

Sort + Break

Sort candidates first. When the current candidate exceeds the target, all subsequent candidates also will — so break instead of continue. Turns O(n) loop into O(pruned).

Deduplication

Sort + skip duplicate elements at the same recursion level. Prevents generating identical solutions from different orderings of equal elements.

Counting Pruning

Check if enough elements remain to complete the solution. In LC #77, stop when start > n - remaining + 1. Eliminates impossible branches early.

Palindrome Partitioning: Precomputed Palindrome Table

Instead of checking isPalindrome O(n) at each step, precompute a 2D boolean table in O(n²) so each palindrome check is O(1):

Python
def partition(s):              # LC #131: all palindrome partitions
    n = len(s)
    # Precompute: is_pal[i][j] = True if s[i..j] is a palindrome
    is_pal = [[False] * n for _ in range(n)]
    for i in range(n):
        is_pal[i][i] = True                    # single chars
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                is_pal[i][j] = (length == 2) or is_pal[i+1][j-1]

    result = []
    def backtrack(start, current):
        if start == n:
            result.append(current[:])
            return
        for end in range(start, n):
            if is_pal[start][end]:             # O(1) check!
                current.append(s[start:end+1])
                backtrack(end + 1, current)
                current.pop()

    backtrack(0, [])
    return result
Reference

Complexity Reference

Backtracking complexity is always exponential in the worst case — but pruning reduces the constant factor dramatically in practice.

Problem TypeTime (Worst Case)SpaceNotes
Subsets of n elementsO(2ⁿ × n)O(n)2ⁿ subsets, each O(n) to copy
Permutations of n elementsO(n! × n)O(n)n! permutations, each O(n) to copy
Combinations C(n, k)O(C(n,k) × k)O(k)Much better than n! for small k
Combination SumO(N^(T/M) × k)O(T/M)T=target, M=min candidate, N=candidates
N-QueensO(n!)O(n)Constraint sets prune ~85% of branches
Word SearchO(m×n × 4^L)O(L)L=word length, m×n starting cells
Sudoku SolverO(9^M)O(1)*M=empty cells; in practice much faster

* Excluding the board itself, which is O(81) = O(1) for a fixed 9×9 grid.

When NOT to use backtracking: If the problem asks for count of solutions or optimal value (not the actual solutions themselves), dynamic programming is usually O(n²) or O(n·k) while backtracking is exponential. Example: count of subsets summing to k → DP; actual subsets summing to k → backtracking.

#78 Subsets Medium

Given an integer array of unique elements, return all possible subsets (the power set).

Key: Record every node (not just leaves) in the decision tree. Use start to only look forward, preventing duplicates.

Python

#46 Permutations Medium

Given an array of distinct integers, return all possible permutations.

#39 Combination Sum Medium

Find all unique combinations of candidates that sum to the target. The same number may be used unlimited times.

Key: Pass i (not i+1) to the recursive call to allow reuse. Break when candidates[i] > remaining (after sorting) to prune.

Python
def combinationSum(candidates, target):

#131 Palindrome Partitioning Medium

Partition a string so that every substring is a palindrome. Return all possible partitions.

Key: At each position, try all possible first substrings. If the substring is a palindrome, recurse on the rest. Precompute palindrome check table for O(1) checks.

Python
def partition(s):
    n = len(s)
    # O(n²) precompute: is_pal[i][j] = s[i..j] is palindrome
    is_pal = [[False] * n for _ in range(n)]
    for i in range(n): is_pal[i][i] = True
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            is_pal[i][j] = s[i] == s[j] and (length == 2 or is_pal[i+1][j-1])

    result = []
    def backtrack(start, curr):
        if start == n:
            result.append(curr[:])
            return
        for end in range(start, n):
            if is_pal[start][end]:
                curr.append(s[start:end+1])
                backtrack(end + 1, curr)
                curr.pop()
    backtrack(0, [])
    return result
⏱ O(2ⁿ × n)💾 O(n²) for palindrome tablePattern: Partitioning + precomputed DP
#79 Word Search Medium

Given a 2D board and a word, find if the word exists in the grid by moving to adjacent cells. Each cell may only be used once per path.

Key: Mark cells visited by overwriting with a sentinel ('#'). Restore on backtrack. Start DFS from every cell that matches word[0].

Python
def exist(board, word):
    rows, cols = len(board), len(board[0])
    def dfs(r, c, k):
        if k == len(word): return True
        if not (0 <= r < rows and 0 <= c < cols): return False
        if board[r][c] != word[k]: return False
        temp, board[r][c] = board[r][c], '#'   # mark visited
        found = any(dfs(r+dr, c+dc, k+1) for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)])
        board[r][c] = temp                      # restore
        return found
    return any(dfs(r, c, 0) for r in range(rows) for c in range(cols))
⏱ O(m×n × 4^L)💾 O(L) stack depthPattern: Grid DFS with in-place marking
#51 N-Queens Hard

Place N queens on an N×N chessboard so no two queens attack each other. Return all distinct solutions.

Key: Backtrack row-by-row (one queen per row). Track occupied columns and both diagonal types with sets for O(1) conflict checking. Diagonals: r-c (top-right to bottom-left), r+c (top-left to bottom-right).

Python
def solveNQueens(n):
    result = []
    cols, diag1, diag2 = set(), set(), set()
    board = [['.']*n for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2:
                continue                        # attacked → skip
            board[row][col] = 'Q'
            cols.add(col); diag1.add(row-col); diag2.add(row+col)
            backtrack(row + 1)
            board[row][col] = '.'
            cols.remove(col); diag1.remove(row-col); diag2.remove(row+col)

    backtrack(0)
    return result
⏱ O(n!)💾 O(n)Pattern: Constraint Satisfaction (row-by-row)
#40 Combination Sum II Medium

Find all unique combinations where each number may only be used once, and candidates may contain duplicates.

Key: Sort first. Use i+1 (no reuse). At each level, skip candidates[i] == candidates[i-1] when i > start to avoid duplicate combinations.

Python
def combinationSum2(candidates, target):
    candidates.sort()
    result = []
    def backtrack(start, curr, remaining):
        if remaining == 0:
            result.append(curr[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining: break   # prune
            if i > start and candidates[i] == candidates[i-1]: continue  # dedup
            curr.append(candidates[i])
            backtrack(i + 1, curr, remaining - candidates[i])
            curr.pop()
    backtrack(0, [], target)
    return result
⏱ O(2ⁿ)💾 O(n)Pattern: Combination Sum (no reuse + dedup)
#37 Sudoku Solver Hard

Fill a 9×9 Sudoku board so that every row, column, and 3×3 box contains the digits 1-9. The board has a unique solution.

Key: Find the next empty cell, try digits 1-9 (skipping those violating constraints), recurse. If no digit works, return False to trigger backtracking. Use sets for O(1) constraint checks.

Python
def solveSudoku(board):
    rows = [set() for _ in range(9)]
    cols = [set() for _ in range(9)]
    boxes = [set() for _ in range(9)]
    # Pre-populate from existing clues
    for r in range(9):
        for c in range(9):
            if board[r][c] != '.':
                d = board[r][c]
                rows[r].add(d); cols[c].add(d)
                boxes[(r//3)*3 + c//3].add(d)

    def backtrack():
        for r in range(9):
            for c in range(9):
                if board[r][c] != '.': continue
                box_id = (r//3)*3 + c//3
                for d in '123456789':
                    if d in rows[r] or d in cols[c] or d in boxes[box_id]: continue
                    board[r][c] = d
                    rows[r].add(d); cols[c].add(d); boxes[box_id].add(d)
                    if backtrack(): return True
                    board[r][c] = '.'
                    rows[r].remove(d); cols[c].remove(d); boxes[box_id].remove(d)
                return False   # no valid digit for this cell → backtrack
        return True            # all cells filled

    backtrack()
⏱ O(9^M) worst case, M=empty cells💾 O(1) (fixed 9×9 board)Pattern: Constraint Satisfaction (2D)
