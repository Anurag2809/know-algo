Binary Search Tree

A binary tree with one extra rule: left subtree values < root < right subtree values. This single constraint gives O(log n) search, insert, and delete — and makes inorder traversal produce a sorted sequence.

🌲 BST property 🐍 Python implementation 🎯 6 LeetCode problems ⚖️ Balanced BSTs
Core Property

The BST Property

For every node N in a BST:

  • All values in the left subtree are < N.val
  • All values in the right subtree are > N.val
  • Both subtrees are also valid BSTs (recursive definition)
Valid BST: NOT a valid BST: 8 8 / \ / \ 3 10 3 10 / \ \ / \ \ 1 6 14 1 6 14 / \ / / \ / 4 7 13 4 7 3 ← WRONG! 3 < 8 but right of 8's subtree Key insight: BST property applies to the ENTIRE subtree, not just immediate children. Node 3 (right child of 8) must have ALL descendants > 8, not just its direct parent.
The inorder traversal trick Inorder traversal (Left → Root → Right) of a valid BST produces values in sorted ascending order. This is the single most important BST property for interview problems. If you inorder-traverse and it's not sorted, the tree is invalid.
Core Operations

Insert / Search / Delete

Python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# SEARCH: O(h) — binary search on tree
def search(root, val):
    if not root or root.val == val:
        return root
    if val < root.val:
        return search(root.left, val)
    return search(root.right, val)

# Iterative search (O(1) space)
def search_iter(root, val):
    while root:
        if val == root.val: return root
        root = root.left if val < root.val else root.right
    return None

# INSERT: O(h) — find the right spot, create new leaf
def insert(root, val):
    if not root:
        return TreeNode(val)
    if val < root.val:
        root.left = insert(root.left, val)
    elif val > root.val:
        root.right = insert(root.right, val)
    # val == root.val: no duplicates (or handle as needed)
    return root

# DELETE: O(h) — three cases
def delete(root, val):
    if not root:
        return None
    if val < root.val:
        root.left = delete(root.left, val)
    elif val > root.val:
        root.right = delete(root.right, val)
    else:
        # Case 1: leaf → just remove
        if not root.left and not root.right:
            return None
        # Case 2: one child → replace with that child
        if not root.left:
            return root.right
        if not root.right:
            return root.left
        # Case 3: two children → replace with inorder successor
        # (smallest value in right subtree)
        successor = root.right
        while successor.left:
            successor = successor.left
        root.val = successor.val          # copy successor value up
        root.right = delete(root.right, successor.val)  # delete successor
    return root
Unique Properties

BST-Specific Algorithms

Python
# VALIDATE BST: propagate valid range (lo, hi) down the tree
def isValidBST(root, lo=float('-inf'), hi=float('inf')) -> bool:
    if not root:
        return True
    if not (lo < root.val < hi):
        return False
    return (isValidBST(root.left, lo, root.val) and
            isValidBST(root.right, root.val, hi))

# KTH SMALLEST: inorder traversal stops at k-th element
def kthSmallest(root, k: int) -> int:
    # Approach 1: collect inorder, return k-th (O(n) space)
    # Approach 2: iterative inorder with early exit (O(h) space)
    stack = []
    curr = root
    count = 0
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()
        count += 1
        if count == k:
            return curr.val
        curr = curr.right
    return -1

# KTH LARGEST: reverse inorder (Right → Root → Left)
def kthLargest(root, k: int) -> int:
    stack = []
    curr = root
    count = 0
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.right    # go right first
        curr = stack.pop()
        count += 1
        if count == k:
            return curr.val
        curr = curr.left         # then left
    return -1

# RANGE SUM: sum all values in [low, high] using BST property for pruning
def rangeSumBST(root, low: int, high: int) -> int:
    if not root:
        return 0
    total = 0
    if low <= root.val <= high:
        total += root.val
    if root.val > low:    # only explore left if some values there are >= low
        total += rangeSumBST(root.left, low, high)
    if root.val < high:   # only explore right if some values there are <= high
        total += rangeSumBST(root.right, low, high)
    return total

# FLOOR: largest value ≤ target
def floor(root, target):
    result = None
    while root:
        if root.val == target:
            return root.val
        elif root.val < target:
            result = root.val   # potential answer, try to find larger
            root = root.right
        else:
            root = root.left
    return result

# CEILING: smallest value ≥ target
def ceiling(root, target):
    result = None
    while root:
        if root.val == target:
            return root.val
        elif root.val > target:
            result = root.val   # potential answer, try to find smaller
            root = root.left
        else:
            root = root.right
    return result
Navigation

Inorder Successor & Predecessor

Python
# INORDER SUCCESSOR: next larger value
# Case 1: node has right child → leftmost node in right subtree
# Case 2: no right child → nearest ancestor for which node is in left subtree
def inorder_successor(root, p):
    successor = None
    while root:
        if p.val < root.val:
            successor = root   # root could be successor
            root = root.left   # look for smaller valid successor
        else:
            root = root.right  # p is in right subtree, go right
    return successor

# INORDER PREDECESSOR: next smaller value
def inorder_predecessor(root, p):
    predecessor = None
    while root:
        if p.val > root.val:
            predecessor = root # root could be predecessor
            root = root.right  # look for larger valid predecessor
        else:
            root = root.left
    return predecessor

# BST ITERATOR (LC 173): inorder iterator O(h) space
class BSTIterator:
    def __init__(self, root):
        self.stack = []
        self._push_left(root)

    def _push_left(self, node):
        while node:
            self.stack.append(node)
            node = node.left

    def next(self) -> int:
        node = self.stack.pop()
        self._push_left(node.right)
        return node.val

    def hasNext(self) -> bool:
        return bool(self.stack)
Self-Balancing

Balanced BSTs & Python's sortedcontainers

The problem with unbalanced BSTs A sorted input like [1,2,3,4,5] inserts into a BST as a linked list (each node has only a right child). Height = n, so all operations degrade to O(n). This is why self-balancing BSTs exist.
Python
# AVL Tree: rotations after every insert/delete to maintain height balance
# Red-Black Tree: used in C++ std::map, Java TreeMap
# Both guarantee O(log n) for all operations

# Install: pip install sortedcontainers
from sortedcontainers import SortedList, SortedDict, SortedSet

sl = SortedList([3, 1, 4, 1, 5, 9, 2, 6])
print(sl)           # SortedList([1, 1, 2, 3, 4, 5, 6, 9])
sl.add(7)           # O(log n) insert
sl.remove(1)        # O(log n) remove first occurrence
print(sl[0])        # O(log n) min
print(sl[-1])       # O(log n) max
idx = sl.bisect_left(5)  # O(log n) floor index

# SortedList is a skip-list backed sorted array
# Great for: Order Statistics, Range Queries, Sliding Window with order

# Example: find median in a stream using SortedList
class MedianFinder:
    def __init__(self):
        self.data = SortedList()

    def addNum(self, num: int) -> None:
        self.data.add(num)    # O(log n)

    def findMedian(self) -> float:
        n = len(self.data)
        mid = n // 2
        if n % 2 == 1:
            return self.data[mid]
        return (self.data[mid-1] + self.data[mid]) / 2

LC #108Convert Sorted Array to BSTEasy

Build a height-balanced BST from a sorted array. Divide and conquer: always pick the middle element as root.

Python
def sortedArrayToBST(self, nums: list[int]):
    def build(lo, hi):
        if lo > hi:
            return None
        mid = (lo + hi) // 2
        node = TreeNode(nums[mid])
        node.left = build(lo, mid - 1)
        node.right = build(mid + 1, hi)
        return node
    return build(0, len(nums) - 1)
⏱ Time: O(n)💾 Space: O(log n)

LC #653Two Sum IV – Input is a BSTEasy

Find if two nodes sum to target. Use BST iterator from both ends (like two-pointer on sorted array).

Python
def findTarget(self, root, k: int) -> bool:
    # Collect inorder (sorted) and use two pointers
    vals = []
    def inorder(node):
        if node:
            inorder(node.left)
            vals.append(node.val)
            inorder(node.right)
    inorder(root)
    left, right = 0, len(vals) - 1
    while left < right:
        s = vals[left] + vals[right]
        if s == k: return True
        elif s < k: left += 1
        else: right -= 1
    return False
⏱ Time: O(n)💾 Space: O(n)
LC #538Convert BST to Greater TreeMedium

Replace each node's value with the sum of all values greater than it. Reverse inorder (Right → Root → Left) with a running sum.

Python
def convertBST(self, root):
    self.suffix_sum = 0
    def reverse_inorder(node):
        if not node:
            return
        reverse_inorder(node.right)     # visit larger values first
        self.suffix_sum += node.val
        node.val = self.suffix_sum      # replace with cumulative sum
        reverse_inorder(node.left)
    reverse_inorder(root)
    return root
⏱ Time: O(n)💾 Space: O(h)
Real World

Real-World Applications

  • Database indexes: B+ trees (multi-way balanced BSTs) power MySQL/PostgreSQL range queries
  • C++ std::map / std::set: Red-Black trees, guaranteed O(log n)
  • Java TreeMap / TreeSet: Red-Black tree, ordered key-value store
  • Python sortedcontainers: SortedList/SortedDict for competitive programming
  • File systems: directory entries sorted for binary search (some implementations)
  • Network routing: longest-prefix matching via tries and BSTs
  • Interval scheduling: augmented BSTs store intervals + max endpoint for O(log n) overlap queries
