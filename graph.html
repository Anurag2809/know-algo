Graph Traversal

Graphs model anything with pairwise relationships — networks, grids, dependencies. This guide covers the two fundamental traversal strategies (BFS and DFS), grid-as-graph problems, and cycle detection. For weighted shortest paths and advanced algorithms, see Graph Algorithms →

Fundamentals

What is a Graph?

A graph G = (V, E) consists of vertices (nodes) and edges (connections between nodes). Unlike trees, graphs can have cycles, disconnected components, and edges pointing in any direction.

Undirected Graph: Directed Graph (Digraph): 0 ── 1 0 ──→ 1 /| | | | 4 | | ↓ ↓ \| | 2 ──→ 3 2 ── 3 ↑ 4 Weighted (edges have costs): DAG (Directed Acyclic Graph): 0 ──4── 1 Course prereqs, build systems, | | topological ordering 1 2 | | 2 ──5── 3

🌲 Tree vs Graph

  • Tree: connected, acyclic, n-1 edges for n nodes
  • Graph: may be disconnected, cyclic, any number of edges
  • Every tree is a graph; not every graph is a tree

🔑 Key Properties

  • Directed: edges have direction (one-way roads)
  • Undirected: edges are bidirectional (friendships)
  • Weighted: edges have costs (distances, times)
  • DAG: directed + no cycles (dependency graphs)

Graph Representations

Python
from collections import defaultdict

# ── ADJACENCY LIST (preferred for most problems) ──────────────────
# Space: O(V + E)  |  Edge check: O(degree)  |  Iterate neighbors: O(degree)

# Undirected graph: 0-1, 0-2, 0-4, 1-3, 2-3
graph = defaultdict(list)
edges = [(0,1),(0,2),(0,4),(1,3),(2,3)]
for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)   # both directions for undirected

# Directed graph: only add one direction
directed = defaultdict(list)
for u, v in edges:
    directed[u].append(v)

# Weighted adjacency list: {node: [(neighbor, weight), ...]}
weighted = {
    0: [(1, 4), (2, 1)],
    1: [(3, 1)],
    2: [(1, 2), (3, 5)],
    3: []
}

# Build from LeetCode-style input
def build(n, edges, directed=False):
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        if not directed:
            graph[v].append(u)
    return graph

# ── ADJACENCY MATRIX ──────────────────────────────────────────────
# Space: O(V²)  |  Edge check: O(1)  |  Use for dense graphs

n = 4
matrix = [[0] * n for _ in range(n)]
matrix[0][1] = 1   # edge 0→1
matrix[1][0] = 1   # edge 1→0 (undirected)
Which representation to use?
  • Adjacency list: almost always correct for interview problems (sparse graphs, V+E space)
  • Adjacency matrix: when you need O(1) edge existence check, or graph is dense (lots of edges)
  • Grid (2D array): implicit graph — cells are nodes, 4-directional neighbors are edges. No need to build adjacency list explicitly
Traversal

BFS — Breadth-First Search

BFS explores level by level using a queue. It finds the shortest path (in hops) in unweighted graphs. Every node at distance d is visited before any node at distance d+1.

Graph: 0-1, 0-2, 0-4, 1-3, 2-3 BFS from node 0: Queue: [0] visited: {0} Process 0: add neighbors 1,2,4 Queue: [1,2,4] visited: {0,1,2,4} Process 1: add neighbor 3 Queue: [2,4,3] visited: {0,1,2,4,3} Process 2: neighbors 0,3 already visited Queue: [4,3] Process 4: neighbor 0 already visited Queue: [3] Process 3: all neighbors visited Order: 0, 1, 2, 4, 3 (by distance from 0: 0=dist0, 1/2/4=dist1, 3=dist2)
Python
from collections import deque

# BFS — template
def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    result = []
    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return result

# BFS with SHORTEST PATH distances
def bfs_distances(graph, start):
    dist = {start: 0}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in dist:
                dist[neighbor] = dist[node] + 1
                queue.append(neighbor)
    return dist

# BFS with PATH RECONSTRUCTION
def bfs_path(graph, start, end):
    if start == end:
        return [start]
    parent = {start: None}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in parent:
                parent[neighbor] = node
                if neighbor == end:
                    # Reconstruct path
                    path = []
                    curr = end
                    while curr is not None:
                        path.append(curr)
                        curr = parent[curr]
                    return path[::-1]
                queue.append(neighbor)
    return []  # no path

# MULTI-SOURCE BFS (start from multiple sources simultaneously)
def multi_source_bfs(grid, sources):
    """BFS from all sources at once — finds min distance to nearest source."""
    rows, cols = len(grid), len(grid[0])
    dist = {}
    queue = deque()
    for r, c in sources:
        dist[(r, c)] = 0
        queue.append((r, c, 0))
    while queue:
        r, c, d = queue.popleft()
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r+dr, c+dc
            if (0 <= nr < rows and 0 <= nc < cols and
                    grid[nr][nc] != '#' and (nr,nc) not in dist):
                dist[(nr, nc)] = d + 1
                queue.append((nr, nc, d + 1))
    return dist
Traversal

DFS — Depth-First Search

DFS explores as deep as possible along each branch before backtracking. Uses a stack (either the call stack via recursion, or an explicit stack). Key for: connected components, cycle detection, topological sort, and "mark all reachable nodes" problems.

Python
# DFS — Recursive (cleaner, uses call stack)
def dfs_recursive(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)
    return visited

# DFS — Iterative (explicit stack; avoids Python recursion limit)
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]
    order = []
    while stack:
        node = stack.pop()
        if node not in visited:
            visited.add(node)
            order.append(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    stack.append(neighbor)
    return order

# COUNT CONNECTED COMPONENTS
def count_components(n, edges):
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    visited = set()
    count = 0
    def dfs(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)
    for node in range(n):
        if node not in visited:
            dfs(node)
            count += 1
    return count

# CHECK IF PATH EXISTS between source and destination
def has_path(graph, src, dst):
    if src == dst:
        return True
    visited = set()
    def dfs(node):
        if node == dst:
            return True
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited and dfs(neighbor):
                return True
        return False
    return dfs(src)

🌊 BFS — Use when:

  • You need the shortest path (fewest edges)
  • You need to explore level by level
  • You want the minimum number of steps
  • Multi-source problems (multiple starting points)

🌿 DFS — Use when:

  • You need to explore all paths (backtracking)
  • You need to detect cycles
  • You're doing topological sort
  • You need to mark entire connected regions
Key Pattern

Grid / Matrix as Graph

2D grids are one of the most common graph problem types. Treat each cell as a node and its 4 (or 8) neighbors as edges. No need to build an explicit adjacency list — just check bounds and conditions inline.

Grid graph: [1, 1, 0, 0] 1 = land (node), 0 = water (no edge) [1, 1, 0, 0] [0, 0, 1, 0] Cell (r,c) connects to: [0, 0, 0, 1] (r-1,c), (r+1,c), (r,c-1), (r,c+1) 4 connected components of 1s = 3 islands
Python
DIRS = [(0,1),(0,-1),(1,0),(-1,0)]   # 4-directional (up/down/left/right)
DIRS8 = [(0,1),(0,-1),(1,0),(-1,0),  # 8-directional (include diagonals)
         (1,1),(1,-1),(-1,1),(-1,-1)]

def in_bounds(r, c, rows, cols):
    return 0 <= r < rows and 0 <= c < cols

# DFS on grid — "flood fill" a connected region
def dfs_grid(grid, r, c, visited):
    rows, cols = len(grid), len(grid[0])
    if not in_bounds(r, c, rows, cols):
        return
    if (r, c) in visited or grid[r][c] == '0':
        return
    visited.add((r, c))
    for dr, dc in DIRS:
        dfs_grid(grid, r+dr, c+dc, visited)

# BFS on grid — shortest path with obstacles
def bfs_shortest_path_grid(grid, start, end):
    rows, cols = len(grid), len(grid[0])
    if grid[start[0]][start[1]] == 1 or grid[end[0]][end[1]] == 1:
        return -1  # start or end is blocked
    visited = {start}
    queue = deque([(start, 0)])
    while queue:
        (r, c), dist = queue.popleft()
        if (r, c) == end:
            return dist
        for dr, dc in DIRS:
            nr, nc = r+dr, c+dc
            if (in_bounds(nr, nc, rows, cols) and
                    grid[nr][nc] == 0 and
                    (nr, nc) not in visited):
                visited.add((nr, nc))
                queue.append(((nr, nc), dist + 1))
    return -1  # no path
Key Technique

Cycle Detection

Python
# UNDIRECTED graph: DFS with parent tracking
# A cycle exists if we visit a node already visited that is NOT our parent
def has_cycle_undirected(graph, n):
    visited = set()
    def dfs(node, parent):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                if dfs(neighbor, node):
                    return True
            elif neighbor != parent:  # visited and not our parent → cycle!
                return True
        return False
    for node in range(n):
        if node not in visited:
            if dfs(node, -1):
                return True
    return False

# DIRECTED graph: DFS with 3-color marking
# WHITE=0 (unvisited), GRAY=1 (in current path), BLACK=2 (done)
# Back edge (to GRAY node) = cycle in directed graph
def has_cycle_directed(graph, n):
    color = [0] * n  # 0=white, 1=gray, 2=black
    def dfs(node):
        color[node] = 1  # mark as in-progress
        for neighbor in graph[node]:
            if color[neighbor] == 1:   # back edge → cycle!
                return True
            if color[neighbor] == 0:   # unvisited → recurse
                if dfs(neighbor):
                    return True
        color[node] = 2  # mark as finished
        return False
    for node in range(n):
        if color[node] == 0:
            if dfs(node):
                return True
    return False

Practice

LeetCode Problems

LC #200Number of IslandsMedium

Count connected components of '1's in a 2D grid. DFS flood-fill: when we find a '1', DFS to mark its entire island as visited.

Python
def numIslands(self, grid: list[list[str]]) -> int:
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] == '0':
            return
        grid[r][c] = '0'  # mark visited by sinking the land
        dfs(r+1,c); dfs(r-1,c); dfs(r,c+1); dfs(r,c-1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1
    return count
⏱ Time: O(m·n)💾 Space: O(m·n) recursion
LC #994Rotting OrangesMedium

Find minimum minutes until all fresh oranges rot. Rotten oranges spread to neighbors each minute. Multi-source BFS from all initially rotten oranges simultaneously.

Python
def orangesRotting(self, grid: list[list[int]]) -> int:
    rows, cols = len(grid), len(grid[0])
    fresh = 0
    queue = deque()

    # Initialize: collect all rotten oranges, count fresh
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2:
                queue.append((r, c, 0))  # (row, col, minutes)
            elif grid[r][c] == 1:
                fresh += 1

    minutes = 0
    while queue:
        r, c, t = queue.popleft()
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                grid[nr][nc] = 2  # rot it
                fresh -= 1
                minutes = t + 1
                queue.append((nr, nc, t+1))

    return minutes if fresh == 0 else -1
⏱ Time: O(m·n)💾 Space: O(m·n)
LC #130Surrounded RegionsMedium

Capture all 'O' regions not connected to the border. Key insight: instead of finding surrounded regions directly, find all O's connected to the border (safe) and flip everything else.

Python
def solve(self, board: list[list[str]]) -> None:
    rows, cols = len(board), len(board[0])

    def dfs(r, c):
        if not (0 <= r < rows and 0 <= c < cols) or board[r][c] != 'O':
            return
        board[r][c] = 'S'  # mark as "safe" (connected to border)
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            dfs(r+dr, c+dc)

    # Step 1: mark all border-connected O's as safe
    for r in range(rows):
        for c in [0, cols-1]:
            if board[r][c] == 'O': dfs(r, c)
    for c in range(cols):
        for r in [0, rows-1]:
            if board[r][c] == 'O': dfs(r, c)

    # Step 2: flip remaining O's to X; restore S to O
    for r in range(rows):
        for c in range(cols):
            if board[r][c] == 'O': board[r][c] = 'X'
            elif board[r][c] == 'S': board[r][c] = 'O'
⏱ Time: O(m·n)💾 Space: O(m·n)
LC #207Course ScheduleMedium

Can all courses be finished given prerequisites? Equivalent to: does the directed graph have a cycle?

Python
def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool:
    graph = defaultdict(list)
    for a, b in prerequisites:
        graph[b].append(a)  # b must be taken before a

    # 0=unvisited, 1=in-progress, 2=done
    state = [0] * numCourses

    def dfs(node):
        if state[node] == 1: return False  # back edge → cycle
        if state[node] == 2: return True   # already cleared
        state[node] = 1
        for neighbor in graph[node]:
            if not dfs(neighbor): return False
        state[node] = 2
        return True

    return all(dfs(i) for i in range(numCourses) if state[i] == 0)
⏱ Time: O(V+E)💾 Space: O(V+E)
LC #695Max Area of IslandMedium

Find the maximum area of an island. DFS flood-fill returning the count of cells visited.

Python
def maxAreaOfIsland(self, grid: list[list[int]]) -> int:
    rows, cols = len(grid), len(grid[0])

    def dfs(r, c) -> int:
        if not (0 <= r < rows and 0 <= c < cols) or grid[r][c] == 0:
            return 0
        grid[r][c] = 0  # mark visited
        return 1 + dfs(r+1,c) + dfs(r-1,c) + dfs(r,c+1) + dfs(r,c-1)

    return max(dfs(r, c) for r in range(rows) for c in range(cols))
⏱ Time: O(m·n)💾 Space: O(m·n)
LC #127Word LadderHard

Find the shortest transformation sequence from beginWord to endWord, changing one letter at a time. This is BFS on an implicit graph where edges connect words differing by one character.

Python
def ladderLength(self, beginWord: str, endWord: str, wordList: list[str]) -> int:
    wordSet = set(wordList)
    if endWord not in wordSet:
        return 0
    queue = deque([(beginWord, 1)])
    visited = {beginWord}
    while queue:
        word, length = queue.popleft()
        for i in range(len(word)):
            for c in 'abcdefghijklmnopqrstuvwxyz':
                new_word = word[:i] + c + word[i+1:]
                if new_word == endWord:
                    return length + 1
                if new_word in wordSet and new_word not in visited:
                    visited.add(new_word)
                    queue.append((new_word, length + 1))
    return 0
⏱ Time: O(M²·N) M=word length, N=dict size💾 Space: O(M²·N)
Real World

Real-World Applications

  • BFS — social networks: friend-of-friend discovery (LinkedIn "2nd connections"), degrees of separation
  • BFS — GPS: shortest route in unweighted road networks (number of turns, not distance)
  • BFS — web crawlers: crawl URLs level by level from a seed URL
  • DFS — maze solving: find any path through a maze (not necessarily shortest)
  • DFS — garbage collection: mark all reachable objects from roots
  • DFS — cycle detection: deadlock detection in OS resource allocation graphs
  • Grid problems: image segmentation (flood fill in Photoshop), fire simulation, game pathfinding (A*)
