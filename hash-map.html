Hash Map / Hash Table

The workhorse of coding interviews. O(1) average-case insert, lookup, and delete. Understanding hash maps unlocks solutions to a huge class of problems that would otherwise be O(n²).

⚡ O(1) average operations 🐍 Python dict internals 🎯 6 LeetCode problems 📊 Collision resolution
Introduction

What is a Hash Map?

A hash map (also called hash table, dictionary, or associative array) is a data structure that stores key-value pairs and provides O(1) average-case access. It's the backbone of Python's dict and set, Java's HashMap, and JavaScript's Map.

KEY HASH FUNCTION ARRAY (buckets) ───────── ───────────────── ────────────────── "apple" ──→ hash("apple") % 8 = 3 ──→ arr[3] = 5 "banana" ──→ hash("banana") % 8 = 1 ──→ arr[1] = 2 "cherry" ──→ hash("cherry") % 8 = 6 ──→ arr[6] = 8 Index: [0] [1] [2] [3] [4] [5] [6] [7] Value: - 2 - 5 - - 8 -

The core idea: transform a key into an array index using a hash function, then store the value at that index. This gives O(1) average access because array indexing is O(1).

Why O(1) vs alternatives
  • Unsorted array: O(n) search — must scan every element
  • Sorted array: O(log n) binary search, but O(n) insert/delete
  • BST: O(log n) for all operations (O(n) worst case without balancing)
  • Hash Map: O(1) average for all operations — best of all worlds
Core Concept

Hash Functions & Collisions

What Makes a Good Hash Function?

A good hash function must be:

  • Deterministic: same key always produces same hash
  • Uniform distribution: spreads keys evenly across buckets (minimize collisions)
  • Fast to compute: O(1) or O(key length)
  • Avalanche effect: small change in key → big change in hash

Collisions: When Two Keys Map to the Same Index

"listen" → hash % 8 = 3 ──→ arr[3] ??? ←── "silent" → hash % 8 = 3 COLLISION: Both keys want arr[3]! Solution: Chaining (linked list at each bucket) or Open Addressing (find next empty slot)

Load Factor & Resizing

The load factor α = n/m (items stored / total buckets). When α gets too high, collisions increase dramatically.

  • Python's dict resizes when α > 2/3 (~66%)
  • Java's HashMap resizes when α > 0.75
  • Resize doubles capacity and rehashes all keys: O(n) operation
  • Amortized over all insertions: still O(1) per insert
The Birthday Paradox & Why Collisions Are Inevitable With only 23 people in a room, there's a 50% chance two share a birthday (365 days). Similarly, with 23 items in a 365-bucket table (α ≈ 0.063), collisions are already 50% likely. This is why we need collision resolution strategies even at low load factors.

From Scratch

Python Implementation

Building a hash map from scratch using separate chaining (each bucket holds a linked list of key-value pairs to handle collisions).

Python
class HashMap:
    def __init__(self, capacity=16):
        self.capacity = capacity
        self.size = 0
        self.load_factor = 0.75
        # Each bucket is a list of (key, value) pairs (chaining)
        self.buckets = [[] for _ in range(self.capacity)]

    def _hash(self, key):
        """Map key to bucket index."""
        return hash(key) % self.capacity

    def put(self, key, value):
        """Insert or update key-value pair. O(1) average."""
        idx = self._hash(key)
        bucket = self.buckets[idx]
        # Check if key already exists → update
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        # New key → append
        bucket.append((key, value))
        self.size += 1
        # Resize if load factor exceeded
        if self.size / self.capacity > self.load_factor:
            self._resize()

    def get(self, key, default=None):
        """Retrieve value by key. O(1) average."""
        idx = self._hash(key)
        for k, v in self.buckets[idx]:
            if k == key:
                return v
        return default

    def remove(self, key):
        """Delete key-value pair. O(1) average."""
        idx = self._hash(key)
        bucket = self.buckets[idx]
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket.pop(i)
                self.size -= 1
                return True
        return False  # key not found

    def _resize(self):
        """Double capacity and rehash all keys. O(n)."""
        old_buckets = self.buckets
        self.capacity *= 2
        self.buckets = [[] for _ in range(self.capacity)]
        self.size = 0
        for bucket in old_buckets:
            for k, v in bucket:
                self.put(k, v)  # rehash all existing pairs

    def __contains__(self, key):
        return self.get(key) is not None

    def __len__(self):
        return self.size

    def keys(self):
        for bucket in self.buckets:
            for k, v in bucket:
                yield k

    def items(self):
        for bucket in self.buckets:
            for k, v in bucket:
                yield k, v


# Usage
hm = HashMap()
hm.put("name", "Alice")
hm.put("age", 30)
print(hm.get("name"))    # "Alice"
print(hm.get("city", "Unknown"))  # "Unknown"
hm.remove("age")
print(len(hm))           # 1
Deep Dive

Collision Resolution Strategies

1. Separate Chaining (used above)

Each bucket contains a list (or linked list) of all key-value pairs that hash to that index.

Bucket 3: [("apple", 5)] → [("silent", 9)] → [("listen", 7)] → None ↑ all three hash to index 3 Worst case: all n keys in one bucket → O(n) lookup Average case (good hash function): O(1 + α) ≈ O(1)

2. Open Addressing (Linear Probing)

If a bucket is occupied, probe the next bucket (linear, quadratic, or double hashing).

Insert "silent" (wants bucket 3, occupied by "apple"): Probe 3 → occupied Probe 4 → empty! → store here Index: [0] [1] [2] [apple,5] [silent,9] [5] [6] [7] ↑ wanted 3 ↑ stored at 4
Python
class HashMapLinearProbing:
    """Open addressing with linear probing."""
    DELETED = object()  # sentinel for deleted slots

    def __init__(self, capacity=16):
        self.capacity = capacity
        self.size = 0
        self.keys = [None] * capacity
        self.vals = [None] * capacity

    def _probe(self, key):
        """Find slot for key using linear probing."""
        idx = hash(key) % self.capacity
        while (self.keys[idx] is not None and
               self.keys[idx] is not self.DELETED and
               self.keys[idx] != key):
            idx = (idx + 1) % self.capacity  # wrap around
        return idx

    def put(self, key, value):
        idx = self._probe(key)
        if self.keys[idx] is None or self.keys[idx] is self.DELETED:
            self.size += 1
        self.keys[idx] = key
        self.vals[idx] = value

    def get(self, key, default=None):
        idx = self._probe(key)
        if self.keys[idx] == key:
            return self.vals[idx]
        return default

    def remove(self, key):
        idx = self._probe(key)
        if self.keys[idx] == key:
            self.keys[idx] = self.DELETED  # mark as deleted
            self.vals[idx] = None
            self.size -= 1
            return True
        return False
Chaining vs Open Addressing
  • Chaining: simpler, handles high load factors well, but pointer overhead (cache unfriendly)
  • Linear probing: cache-friendly (data is contiguous), but clustering degrades performance at high load
  • Python uses: open addressing with a custom probing sequence (pseudo-random)
Analysis

Time & Space Complexity

OperationAverage CaseWorst CaseNotes
InsertO(1)O(n)Worst = all keys collide in one bucket
LookupO(1)O(n)Rare with a good hash function
DeleteO(1)O(n)Same reasoning as lookup
ResizeO(n)O(n)Amortized O(1) per insert

Why "average case" matters The worst case requires an adversary who knows your hash function and deliberately creates collisions. In practice (random keys, cryptographic hash), worst case is astronomically unlikely. That's why hash maps are used everywhere despite the theoretical O(n) worst case.
Interview Prep

Interview Patterns

Frequency Counting

Count occurrences of elements. Use Counter or defaultdict(int). Anagram detection, character frequency, most common element.

Two-Sum Pattern

Store seen values in a map. For each element, check if its complement exists. Transforms O(n²) nested loops into O(n).

Grouping / Bucketing

Group items sharing a property. Use defaultdict(list). Key = canonical form (sorted string, tuple). Value = list of originals.

Prefix Sum + Map

Track running sums. Check if prefix - k seen before. Subarray sum, number of ways to partition array.

Sliding Window

Maintain character/element frequency in a window. Expand right, shrink left when constraint violated. Substring problems.

Memoization

Cache recursive results. @lru_cache or manual dict. Transforms exponential DP into polynomial time.

Practice

LeetCode Problems

LC #1 Two Sum Easy

Given an array of integers and a target, return indices of two numbers that add to target. Each input has exactly one solution.

Key insight: For each number, check if its complement (target - num) has been seen. Store {value: index} in a map.

Python
def twoSum(self, nums: list[int], target: int) -> list[int]:
    seen = {}  # {value: index}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# Example: nums = [2, 7, 11, 15], target = 9
# i=0: num=2, complement=7, not in seen → seen={2:0}
# i=1: num=7, complement=2, IN seen → return [0, 1]
⏱ Time: O(n)💾 Space: O(n)
LC #49 Group Anagrams Medium

Group an array of strings so that anagrams are in the same group.

Key insight: Anagrams have the same characters — use sorted string (or character frequency tuple) as the canonical key.

Python
from collections import defaultdict

def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))   # "eat","tea","ate" → ('a','e','t')
        groups[key].append(s)
    return list(groups.values())

# O(k) key with character count (avoids sort):
def groupAnagrams_v2(self, strs):
    groups = defaultdict(list)
    for s in strs:
        count = [0] * 26
        for c in s:
            count[ord(c) - ord('a')] += 1
        groups[tuple(count)].append(s)
    return list(groups.values())
⏱ Time: O(n·k log k) where k = max string length💾 Space: O(n·k)
LC #146 LRU Cache Medium

Design a data structure that follows the Least Recently Used cache policy with O(1) get and put.

Key insight: Combine a hash map (O(1) lookup) with an OrderedDict (tracks recency). The Python OrderedDict supports move_to_end in O(1).

Python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = OrderedDict()  # maintains insertion order

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)  # mark as recently used
        return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)  # evict LRU (first item)

# Follow-up: implement without OrderedDict using doubly linked list + dict
# for true O(1) on all operations (same idea, more code)
⏱ Time: O(1) for get and put💾 Space: O(capacity)
LC #560 Subarray Sum Equals K Medium

Find the number of subarrays that sum to exactly k.

Key insight: prefix[j] - prefix[i] = k means subarray[i+1..j] = k. Track prefix sum frequencies in a map.

Python
def subarraySum(self, nums: list[int], k: int) -> int:
    count = 0
    prefix = 0
    freq = {0: 1}  # prefix sum 0 seen once (empty subarray)

    for num in nums:
        prefix += num
        # If (prefix - k) was seen before, those subarrays sum to k
        count += freq.get(prefix - k, 0)
        freq[prefix] = freq.get(prefix, 0) + 1

    return count

# Example: nums=[1,1,1], k=2
# prefix=1: freq={0:1,1:1}, count += freq[1-2]=-1? No, freq[-1]=0
# Wait: prefix=1, prefix-k=1-2=-1, freq.get(-1,0)=0 → count=0
# prefix=2: freq={0:1,1:1,2:1}, prefix-k=0, freq[0]=1 → count=1
# prefix=3: freq={0:1,1:1,2:1,3:1}, prefix-k=1, freq[1]=1 → count=2
# Result: 2  ✓
⏱ Time: O(n)💾 Space: O(n)
LC #3 Longest Substring Without Repeating Characters Medium

Find the length of the longest substring without repeating characters.

Key insight: Sliding window. Map each character to its last seen index. When a repeat is found, jump the left pointer past the previous occurrence.

Python
def lengthOfLongestSubstring(self, s: str) -> int:
    last_seen = {}  # {char: last_index}
    left = 0
    max_len = 0

    for right, char in enumerate(s):
        # If char seen and within current window → shrink window
        if char in last_seen and last_seen[char] >= left:
            left = last_seen[char] + 1
        last_seen[char] = right
        max_len = max(max_len, right - left + 1)

    return max_len

# Example: s = "abcabcbb"
# right=0,a: last_seen={a:0}, left=0, max=1
# right=1,b: last_seen={a:0,b:1}, left=0, max=2
# right=2,c: last_seen={a:0,b:1,c:2}, left=0, max=3
# right=3,a: a seen at 0 >= left=0 → left=1, last_seen={a:3}, max=3
# right=4,b: b seen at 1 >= left=1 → left=2, last_seen={b:4}, max=3
# ...
# Result: 3 ("abc")
⏱ Time: O(n)💾 Space: O(min(n, charset))
LC #76 Minimum Window Substring Hard

Find the minimum window in string s that contains all characters of string t.

Key insight: Sliding window with two frequency maps. Track how many characters are "satisfied" (have required count). Shrink window when all characters are covered.

Python
from collections import Counter

def minWindow(self, s: str, t: str) -> str:
    if not t or not s:
        return ""

    need = Counter(t)   # required character counts
    missing = len(t)    # total characters still needed
    best_start, best_end = 0, float('inf')
    left = 0

    for right, char in enumerate(s, 1):
        # Expand right: add char to window
        if need[char] > 0:
            missing -= 1    # satisfied one required character
        need[char] -= 1

        # When all characters satisfied, try to shrink from left
        if missing == 0:
            # Move left pointer past any excess characters
            while need[s[left]] < 0:
                need[s[left]] += 1
                left += 1
            # Update best window
            if right - left < best_end - best_start:
                best_start, best_end = left, right
            # Shrink: remove left char and look for next valid window
            need[s[left]] += 1
            missing += 1
            left += 1

    return s[best_start:best_end] if best_end != float('inf') else ""
⏱ Time: O(|s| + |t|)💾 Space: O(|t|)
Real World

Real-World Applications

  • Database indexing: Hash indexes for O(1) equality lookups (vs B-tree for range queries)
  • Caching: Redis, Memcached — distributed hash maps for sub-millisecond retrieval
  • Symbol tables: Compilers map variable names → memory addresses
  • DNS resolution: domain name → IP address lookup tables
  • Python internals: Every Python object's attributes stored in a __dict__ hash map
  • Blockchain: Hash maps for UTXO (Unspent Transaction Output) sets
  • Deduplication: Track seen URLs (web crawlers), seen events (stream processing)
  • Routing tables: Network switches map MAC addresses → ports
