Binary Tree & BST

The hierarchical data structure that appears in nearly every hard interview problem. Master the 4 traversals, BST operations, and the DFS recursive patterns that unlock problems from easy to hard.

Introduction

What is a Binary Tree?

A binary tree is a hierarchical data structure where each node has at most two children, called left and right. It's one of the most fundamental structures in computer science — file systems, expression parsers, and databases all use trees internally.

1 ← root (depth 0) / \ 2 3 ← depth 1 / \ / \ 4 5 6 7 ← depth 2 (leaves) Terminology: - Root: topmost node (no parent) → 1 - Leaf: node with no children → 4, 5, 6, 7 - Height: longest path from root to leaf → 2 - Depth of node: distance from root → depth(6) = 2 - Subtree: any node + its descendants
Python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

Variants

Tree Types

Key tree types you'll encounter in interviews
  • Full Binary Tree: every node has exactly 0 or 2 children
  • Complete Binary Tree: all levels filled except possibly the last (filled left to right) — this is the property heaps use
  • Perfect Binary Tree: all internal nodes have 2 children, all leaves at same level; n = 2^(h+1) - 1 nodes
  • BST (Binary Search Tree): left subtree values < root < right subtree values — enables O(log n) search
  • Balanced BST: height O(log n), e.g. AVL trees, Red-Black trees — guarantees O(log n) worst case

Degenerate (Skewed) BST — The Worst Case

Inserting sorted values 1,2,3,4,5 into a BST: 1 \ 2 ← becomes a linked list! \ height = n-1 = O(n) 3 all BST operations degrade to O(n) \ 4 \ 5

This is why self-balancing trees (AVL, Red-Black) exist — they ensure height stays O(log n).

Python
from collections import deque

# RECURSIVE traversals (most concise)
def preorder(root):     # Root → Left → Right
    if not root: return []
    return [root.val] + preorder(root.left) + preorder(root.right)

def inorder(root):      # Left → Root → Right (sorted for BST)
    if not root: return []
    return inorder(root.left) + [root.val] + inorder(root.right)

def postorder(root):    # Left → Right → Root
    if not root: return []
    return postorder(root.left) + postorder(root.right) + [root.val]

# ITERATIVE Inorder (most common interview follow-up)
def inorder_iterative(root):
    result, stack, curr = [], [], root
    while curr or stack:
        while curr:           # go as left as possible
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()    # backtrack
        result.append(curr.val)
        curr = curr.right     # try right subtree
    return result

# ITERATIVE Preorder
def preorder_iterative(root):
    if not root: return []
    result, stack = [], [root]
    while stack:
        node = stack.pop()
        result.append(node.val)
        # Push right first (so left is processed first)
        if node.right: stack.append(node.right)
        if node.left: stack.append(node.left)
    return result

# LEVEL-ORDER (BFS) — returns list of lists, one per level
def level_order(root):
    if not root: return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):  # process exactly one level
            node = queue.popleft()
            level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result
BST Deep Dive

BST Operations

Python
class BST:
    def __init__(self):
        self.root = None

    def insert(self, val):
        """Insert val maintaining BST property. O(h) time."""
        def _insert(node, val):
            if not node:
                return TreeNode(val)
            if val < node.val:
                node.left = _insert(node.left, val)
            elif val > node.val:
                node.right = _insert(node.right, val)
            # val == node.val: no duplicates
            return node
        self.root = _insert(self.root, val)

    def search(self, val) -> bool:
        """Search for val. O(h) time."""
        node = self.root
        while node:
            if val == node.val: return True
            elif val < node.val: node = node.left
            else: node = node.right
        return False

    def delete(self, val):
        """Delete val. O(h) time. Three cases:
        1. Leaf → just remove
        2. One child → replace with child
        3. Two children → replace with inorder successor (leftmost in right subtree)
        """
        def _delete(node, val):
            if not node:
                return None
            if val < node.val:
                node.left = _delete(node.left, val)
            elif val > node.val:
                node.right = _delete(node.right, val)
            else:
                # Found the node to delete
                if not node.left:       # case 1 or 2 (no left child)
                    return node.right
                if not node.right:      # case 2 (no right child)
                    return node.left
                # Case 3: two children — find inorder successor
                successor = node.right
                while successor.left:
                    successor = successor.left
                node.val = successor.val  # copy successor value
                node.right = _delete(node.right, successor.val)
            return node
        self.root = _delete(self.root, val)

    def validate_bst(self, root, lo=float('-inf'), hi=float('inf')) -> bool:
        """Verify BST property: every node's value is in range (lo, hi)."""
        if not root:
            return True
        if not (lo < root.val < hi):
            return False
        return (self.validate_bst(root.left, lo, root.val) and
                self.validate_bst(root.right, root.val, hi))

Common Tree Utilities

Python
# Build tree from level-order array (LeetCode format)
# [1,2,3,None,5,None,4] → standard LeetCode tree input
def build_tree(vals):
    if not vals:
        return None
    root = TreeNode(vals[0])
    queue = deque([root])
    i = 1
    while queue and i < len(vals):
        node = queue.popleft()
        if i < len(vals) and vals[i] is not None:
            node.left = TreeNode(vals[i])
            queue.append(node.left)
        i += 1
        if i < len(vals) and vals[i] is not None:
            node.right = TreeNode(vals[i])
            queue.append(node.right)
        i += 1
    return root

# Height of tree
def height(root):
    if not root: return 0
    return 1 + max(height(root.left), height(root.right))

# Count nodes
def count_nodes(root):
    if not root: return 0
    return 1 + count_nodes(root.left) + count_nodes(root.right)

# Check if balanced (every node's left/right subtree heights differ by ≤ 1)
def is_balanced(root) -> bool:
    def check_height(node):
        if not node: return 0
        lh = check_height(node.left)
        rh = check_height(node.right)
        if lh == -1 or rh == -1: return -1   # propagate failure
        if abs(lh - rh) > 1: return -1
        return 1 + max(lh, rh)
    return check_height(root) != -1
Interview Patterns

DFS Patterns for Interviews

Most tree problems follow one of these recursive DFS patterns. Recognizing the pattern is half the battle.

Pattern 1: Return value from subtrees

Python
# Pattern: solve subproblem at each node, use children's results
def maxDepth(root) -> int:
    if not root: return 0
    return 1 + max(maxDepth(root.left), maxDepth(root.right))

def hasPathSum(root, target) -> bool:
    if not root: return False
    if not root.left and not root.right:  # leaf
        return root.val == target
    return (hasPathSum(root.left, target - root.val) or
            hasPathSum(root.right, target - root.val))

Pattern 2: Pass state down

Python
# Pattern: pass information from parent to children
def pathSumAllPaths(root, target):
    """Find all root-to-leaf paths that sum to target."""
    result = []
    def dfs(node, path, remaining):
        if not node: return
        path.append(node.val)
        if not node.left and not node.right and remaining == node.val:
            result.append(list(path))
        dfs(node.left, path, remaining - node.val)
        dfs(node.right, path, remaining - node.val)
        path.pop()  # backtrack
    dfs(root, [], target)
    return result

Pattern 3: LCA (Lowest Common Ancestor)

Python
# Classic LCA for any binary tree (not just BST)
def lowestCommonAncestor(root, p, q):
    if not root or root is p or root is q:
        return root  # base case: found one of the targets
    left = lowestCommonAncestor(root.left, p, q)
    right = lowestCommonAncestor(root.right, p, q)
    if left and right:
        return root   # p in left subtree, q in right → root is LCA
    return left or right  # both in same subtree

# LCA for BST — faster using BST property
def lca_bst(root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left   # both smaller → go left
        elif p.val > root.val and q.val > root.val:
            root = root.right  # both larger → go right
        else:
            return root        # split point → LCA found
    return None

Pattern 4: Track global maximum (nonlocal)

Python
# Pattern: DFS returns local result, updates global max as side effect
def diameterOfBinaryTree(root) -> int:
    max_diameter = 0
    def depth(node):
        nonlocal max_diameter
        if not node: return 0
        left = depth(node.left)
        right = depth(node.right)
        max_diameter = max(max_diameter, left + right)  # update global
        return 1 + max(left, right)
    depth(root)
    return max_diameter

LeetCode Problems

LC #104 Maximum Depth of Binary Tree Easy

Find the maximum depth (number of nodes along the longest path from root to leaf).

Python
def maxDepth(self, root) -> int:
    if not root:
        return 0
    return 1 + max(self.maxDepth(root.left), self.maxDepth(root.right))
⏱ Time: O(n)💾 Space: O(h) recursion stack
LC #226 Invert Binary Tree Easy

Invert a binary tree (mirror it). Classic "Homebrew" interview problem.

Python
def invertTree(self, root):
    if not root:
        return None
    # Swap children, then recurse
    root.left, root.right = self.invertTree(root.right), self.invertTree(root.left)
    return root
⏱ Time: O(n)💾 Space: O(h)
LC #235 Lowest Common Ancestor of BST Medium

Find the LCA of two nodes in a BST. Use the BST property: if both values are smaller, go left; if both larger, go right; otherwise current node is the LCA.

Python
def lowestCommonAncestor(self, root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left
        elif p.val > root.val and q.val > root.val:
            root = root.right
        else:
            return root  # split point = LCA
    return None
⏱ Time: O(h)💾 Space: O(1)
LC #102 Binary Tree Level Order Traversal Medium

Return level-order traversal as a list of lists (one list per level). BFS with level-size tracking.

Python
from collections import deque

def levelOrder(self, root):
    if not root:
        return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):  # snapshot current level size
            node = queue.popleft()
            level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result
⏱ Time: O(n)💾 Space: O(w) where w = max width
LC #98 Validate Binary Search Tree Medium

Verify that a tree is a valid BST. Common mistake: only checking immediate parent. Correct approach: propagate valid range down the tree.

Python
def isValidBST(self, root, lo=float('-inf'), hi=float('inf')) -> bool:
    if not root:
        return True
    if not (lo < root.val < hi):
        return False
    return (self.isValidBST(root.left, lo, root.val) and
            self.isValidBST(root.right, root.val, hi))
⏱ Time: O(n)💾 Space: O(h)
LC #124 Binary Tree Maximum Path Sum Hard

Find the maximum sum path through any nodes (not necessarily root-to-leaf).

Key insight: At each node, the best path can either pass through it (left + node + right) or just extend one branch upward. Track the global max separately.

Python
def maxPathSum(self, root) -> int:
    self.max_sum = float('-inf')

    def max_gain(node):
        if not node:
            return 0
        # Only include subtree if it contributes positively
        left_gain = max(max_gain(node.left), 0)
        right_gain = max(max_gain(node.right), 0)
        # Price of path through this node (cannot go up both sides at once)
        path_through = node.val + left_gain + right_gain
        self.max_sum = max(self.max_sum, path_through)
        # Return max gain for parent (can only extend one side)
        return node.val + max(left_gain, right_gain)

    max_gain(root)
    return self.max_sum
⏱ Time: O(n)💾 Space: O(h)
Real World

Real-World Applications

  • File systems: directory trees (folders contain files and sub-folders)
  • DOM tree: browser HTML is a tree; React's virtual DOM is a tree
  • Expression trees: compilers parse mathematical expressions as trees
  • Database B-trees: MySQL/PostgreSQL indexes use B+ trees (balanced, multi-way)
  • Decision trees: machine learning models for classification
  • Huffman coding: frequency-based compression uses a binary tree
  • XML/JSON parsing: document object models are hierarchical trees
  • Game AI: minimax trees for chess, tic-tac-toe decision making
