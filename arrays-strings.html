Arrays & Strings

The foundation of every coding interview. Arrays and strings don't need fancy data structures — they need patterns. Two pointers, sliding window, prefix sums, and binary search together solve the majority of array problems you'll encounter.

Pattern 1

Two Pointers

Two pointers is a class of techniques where you maintain two indices into an array and move them based on some condition. The most common variant is opposite ends: start lo at index 0 and hi at index n-1, then converge toward the center.

When to use Two Pointers (opposite ends): The array is sorted (or you need a pair/triplet that meets a condition). You want O(n) instead of O(n²) brute force. Classic signals: "find two numbers that sum to X", "maximize area", "move all zeros to end".

Opposite Ends: Template

Python
def two_pointer_opposite(arr):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        current = compute(arr[lo], arr[hi])
        if current == target:
            return (lo, hi)           # found answer
        elif current < target:
            lo += 1                   # need larger value → move lo right
        else:
            hi -= 1                   # need smaller value → move hi left
    return None                       # not found

Why This Works

At each step, you eliminate at least one element from consideration. If the current sum is too small, moving lo right can only increase it (array is sorted). If too large, moving hi left can only decrease it. This gives O(n) vs O(n²) for the brute-force nested loop.

Example: Two Sum II (sorted array) [2, 7, 11, 15], target = 9 lo=0 hi=3 sum = 2+15 = 17 > 9 → hi-- lo=0 hi=2 sum = 2+11 = 13 > 9 → hi-- lo=0 hi=1 sum = 2+7 = 9 == 9 ✓ return [1, 2] Key insight: moving lo right can only increase the sum moving hi left can only decrease the sum → we never miss the solution

Trapping Rain Water Preview

Two pointers also solves "two-sided" accumulation problems. Water trapped at position i = min(left_max, right_max) - height[i]. Instead of precomputing both arrays, two pointers maintain running maxima from each side in O(1) space.

Python
def trap(height):
    lo, hi = 0, len(height) - 1
    left_max = right_max = 0
    water = 0
    while lo <= hi:
        if height[lo] <= height[hi]:
            # water at lo is determined by left_max (right side is >= left)
            if height[lo] >= left_max:
                left_max = height[lo]   # new max, no water trapped here
            else:
                water += left_max - height[lo]
            lo += 1
        else:
            if height[hi] >= right_max:
                right_max = height[hi]
            else:
                water += right_max - height[hi]
            hi -= 1
    return water
Why process the side with smaller height? Water at lo is bounded by min(left_max, right_max). If height[lo] <= height[hi], we know right_max >= height[hi] >= height[lo], so the bottleneck is definitely left_max. We can safely compute water without knowing the full right max.

Same-Direction: Fast & Slow

When both pointers move in the same direction, the technique is often called "fast/slow" or "read/write". It's used for in-place operations:

Python
# Remove duplicates from sorted array in-place (LC #26)
def removeDuplicates(nums):
    if not nums:
        return 0
    write = 1                         # next write position
    for read in range(1, len(nums)):
        if nums[read] != nums[read - 1]:
            nums[write] = nums[read]  # write only unique elements
            write += 1
    return write                      # new logical length

# Move all zeros to end (LC #283) — same pattern
def moveZeroes(nums):
    write = 0
    for read in range(len(nums)):
        if nums[read] != 0:
            nums[write] = nums[read]
            write += 1
    while write < len(nums):
        nums[write] = 0
        write += 1
Pattern 2

Sliding Window

The sliding window is a two-pointer technique where the two pointers define a contiguous subarray. It transforms O(n²) brute force (try all subarrays) into O(n) by maintaining a running aggregate and only updating on changes at the edges.

Fixed-Size Window

When the problem asks for the best subarray of exactly size k, maintain a window of exactly that size. Add the right element, remove the left element.

Python
# Maximum sum subarray of size k
def maxSumFixed(nums, k):
    # Build first window
    window_sum = sum(nums[:k])
    best = window_sum

    # Slide: add right element, remove left element
    for i in range(k, len(nums)):
        window_sum += nums[i]         # right element enters
        window_sum -= nums[i - k]     # left element leaves
        best = max(best, window_sum)
    return best

# Fixed window with a frequency map (e.g., anagram detection)
def findAnagrams(s, p):           # LC #438
    from collections import Counter
    need = Counter(p)
    have = Counter(s[:len(p)])
    result = []
    if have == need:
        result.append(0)
    for i in range(len(p), len(s)):
        # add right
        have[s[i]] += 1
        # remove left
        left = s[i - len(p)]
        have[left] -= 1
        if have[left] == 0:
            del have[left]
        if have == need:
            result.append(i - len(p) + 1)
    return result

Variable-Size Window

The more powerful variant: expand the right pointer unconditionally, shrink from the left when the window violates a condition. This finds the smallest/largest valid window.

Python
# Generic variable sliding window template
def variableWindow(arr, condition):
    lo = 0
    window_state = initialize()       # e.g., sum=0, counter={}, etc.
    ans = 0

    for hi in range(len(arr)):
        window_state = add(arr[hi])   # expand window: include arr[hi]

        while not valid(window_state):
            window_state = remove(arr[lo])
            lo += 1                   # shrink from left until valid

        ans = max(ans, hi - lo + 1)   # update answer (longest valid window)
    return ans

# ─────────────────────────────────────────────────────────
# Longest Repeating Character Replacement (LC #424)
# Find the longest subarray where we can replace ≤ k chars
# so all characters are the same.
# ─────────────────────────────────────────────────────────
def characterReplacement(s, k):
    count = [0] * 26
    max_freq = 0   # max frequency of any single character in current window
    lo = 0
    ans = 0

    for hi in range(len(s)):
        count[ord(s[hi]) - ord('A')] += 1
        max_freq = max(max_freq, count[ord(s[hi]) - ord('A')])

        # window_size - max_freq = number of chars we need to replace
        # if this exceeds k, shrink from left
        if (hi - lo + 1) - max_freq > k:
            count[ord(s[lo]) - ord('A')] -= 1
            lo += 1
        # Note: we don't update max_freq when shrinking — this is intentional!
        # max_freq only grows, ensuring our answer can only stay or grow.
        ans = max(ans, hi - lo + 1)
    return ans
The "shrink once" optimization: In LC #424, we don't recalculate max_freq when shrinking. This is fine because we're tracking the best possible window size seen so far. max_freq never decreases, so the window size never decreases — we're essentially tracking the maximum valid window ever seen. If the window is invalid, we slide (both lo and hi move), which is O(n) total.

The "At Most K" Trick

Many "exactly K" problems can be solved via f(atMost K) - f(atMost K-1) where f is easier to compute with the variable window template:

Python
# Subarrays with exactly K distinct integers (LC #992)
# = atMost(k) - atMost(k-1)
def subarraysWithKDistinct(nums, k):
    def atMost(k):
        count = {}
        lo = 0
        ans = 0
        for hi in range(len(nums)):
            count[nums[hi]] = count.get(nums[hi], 0) + 1
            while len(count) > k:
                count[nums[lo]] -= 1
                if count[nums[lo]] == 0:
                    del count[nums[lo]]
                lo += 1
            ans += hi - lo + 1     # all subarrays ending at hi, starting [lo..hi]
        return ans
    return atMost(k) - atMost(k - 1)
Pattern 3

Prefix Sum

A prefix sum array lets you answer "what is the sum of elements from index l to r?" in O(1) after O(n) preprocessing. It's especially powerful when combined with a hash map to find subarrays with a target sum or property.

nums = [1, 2, -3, 4, 2] index 0 1 2 3 4 pre[0] = 0 ← sentinel: sum of 0 elements pre[1] = 0 + 1 = 1 pre[2] = 1 + 2 = 3 pre[3] = 3 + (-3) = 0 pre[4] = 0 + 4 = 4 pre[5] = 4 + 2 = 6 sum(l, r) = pre[r+1] - pre[l] (0-indexed, inclusive) sum(1, 3) = pre[4] - pre[1] = 4 - 1 = 3 ✓ (2 + -3 + 4 = 3)
Python
# 1-D Prefix Sum
def buildPrefix(nums):
    n = len(nums)
    pre = [0] * (n + 1)          # pre[0] = 0 sentinel
    for i in range(n):
        pre[i + 1] = pre[i] + nums[i]
    return pre

def rangeSum(pre, l, r):         # inclusive, 0-indexed
    return pre[r + 1] - pre[l]

# ─────────────────────────────────────────────────────────
# Prefix Sum + Hash Map: subarray sum equals k (LC #560)
# Key insight: sum(l, r) == k  ←→  pre[r+1] - pre[l] == k
#                               ←→  pre[l] == pre[r+1] - k
# So: for each prefix sum p, count how many earlier prefix
# sums equal (p - k).  That's exactly the number of valid
# subarrays ending at the current position.
# ─────────────────────────────────────────────────────────
def subarraySum(nums, k):
    seen = {0: 1}                # prefix_sum → count; seed with 0
    curr = 0
    ans = 0
    for x in nums:
        curr += x                # current prefix sum
        ans += seen.get(curr - k, 0)   # any prefix that creates a gap of k
        seen[curr] = seen.get(curr, 0) + 1
    return ans

Product Array (No Division)

Product of Array Except Self (LC #238) uses the same prefix/suffix idea but with multiplication. The trick: left pass gives prefix products, right pass gives suffix products, combining them gives the answer without ever dividing.

Python
def productExceptSelf(nums):
    n = len(nums)
    ans = [1] * n

    # Left pass: ans[i] = product of nums[0..i-1]
    prefix = 1
    for i in range(n):
        ans[i] = prefix
        prefix *= nums[i]

    # Right pass: multiply ans[i] by product of nums[i+1..n-1]
    suffix = 1
    for i in range(n - 1, -1, -1):
        ans[i] *= suffix
        suffix *= nums[i]

    return ans

# Example: nums = [1, 2, 3, 4]
# After left pass:  ans = [1, 1, 2, 6]
# Suffix products:       × [24, 12, 4, 1]
# Final:                   [24, 12, 8, 6]  ✓

2-D Prefix Sum

For grid/matrix problems, 2D prefix sums enable O(1) rectangle sum queries after O(m×n) preprocessing:

Python
class NumMatrix:  # LC #304
    def __init__(self, matrix):
        m, n = len(matrix), len(matrix[0])
        # pre[i][j] = sum of rectangle (0,0) to (i-1, j-1)
        self.pre = [[0] * (n + 1) for _ in range(m + 1)]
        for r in range(m):
            for c in range(n):
                self.pre[r+1][c+1] = (
                    matrix[r][c]
                    + self.pre[r][c+1]    # top
                    + self.pre[r+1][c]    # left
                    - self.pre[r][c]      # subtract double-counted top-left
                )

    def sumRegion(self, r1, c1, r2, c2):
        p = self.pre
        return (p[r2+1][c2+1]
                - p[r1][c2+1]             # remove top strip
                - p[r2+1][c1]             # remove left strip
                + p[r1][c1])              # add back double-subtracted corner
Pattern 4

Binary Search

Binary search works on any monotone predicate — not just sorted arrays. The key insight: if the array has a property that splits it cleanly into "No, No, No, Yes, Yes, Yes", binary search can find the first "Yes" in O(log n).

The Binary Search Contract: Binary search requires that the search space is monotone — once the condition becomes true, it stays true (or vice versa). Think of it as the predicate array [F, F, F, T, T, T]. You're finding the boundary between F and T.

Template 1: Find Exact Target

Python
def binarySearch(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:                       # ← inclusive both ends
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1                             # not found

Template 2: Find Left Boundary (first True)

Use when you want the leftmost position satisfying a condition: [F, F, F, T, T, T] → find first T.

Python
def findLeftBoundary(nums, target):
    lo, hi = 0, len(nums)         # hi = len, not len-1 (handles "insert after all")
    while lo < hi:                # ← strict: loop ends when lo == hi
        mid = (lo + hi) // 2
        if nums[mid] < target:    # mid is in the "False" zone → move lo up
            lo = mid + 1
        else:                     # mid might be the answer → preserve it
            hi = mid
    return lo                     # lo == hi == first position where nums[lo] >= target

# Example: bisect_left equivalent
# nums = [1, 2, 4, 4, 5], target = 4
# lo=0  hi=5  mid=2  nums[2]=4 >= 4  →  hi=2
# lo=0  hi=2  mid=1  nums[1]=2  < 4  →  lo=2
# lo=2  hi=2  → return 2  ✓ (first 4 is at index 2)

Template 3: Find Right Boundary (last True)

Use when you want the rightmost position satisfying a condition: [T, T, T, F, F, F] → find last T.

Python
def findRightBoundary(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        mid = (lo + hi + 1) // 2  # ← bias mid toward right to avoid infinite loop!
        if nums[mid] <= target:   # mid might be the answer → preserve it
            lo = mid
        else:                     # mid is in the "False" zone → move hi down
            hi = mid - 1
    return lo                     # lo == hi == last position where nums[lo] <= target

# WHY bias right? If lo=0, hi=1, mid=(0+1)//2=0, if we set lo=mid=0 → infinite loop!
# With +1: mid=(0+1+1)//2=1, so lo moves forward.

Binary Search on a Rotated Array

Even when an array isn't fully sorted, binary search works if at least one half is always fully sorted. At each step, determine which half is sorted and check if the target lies in it.

Python
# LC #153: Find Minimum in Rotated Sorted Array
# The minimum is where the array "drops". Binary search finds the drop.
def findMin(nums):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        mid = (lo + hi) // 2
        if nums[mid] > nums[hi]:
            # mid is in the larger left portion → min is in right half
            lo = mid + 1
        else:
            # mid is in the smaller right portion (or mid IS the min)
            hi = mid
    return nums[lo]

# Example: [4, 5, 6, 7, 0, 1, 2]
# lo=0  hi=6  mid=3  nums[3]=7 > nums[6]=2  →  lo=4
# lo=4  hi=6  mid=5  nums[5]=1 ≤ nums[6]=2  →  hi=5
# lo=4  hi=5  mid=4  nums[4]=0 ≤ nums[5]=1  →  hi=4
# lo=4  hi=4  → return nums[4] = 0  ✓


# LC #33: Search in Rotated Sorted Array
# At every mid, one of the two halves is always fully sorted.
def search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        # Left half is sorted
        if nums[lo] <= nums[mid]:
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1           # target in sorted left half
            else:
                lo = mid + 1           # target in right half
        # Right half is sorted
        else:
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1           # target in sorted right half
            else:
                hi = mid - 1           # target in left half
    return -1

Binary Search on the Answer

A powerful application: instead of searching in an array, binary search on the answer space. Works when the problem has monotone feasibility — "is X possible?" is easier to check than finding the optimal X directly.

Python
# Koko Eating Bananas (LC #875): minimum eating speed to finish in h hours
# "Can Koko eat all bananas at speed k?" is O(n) to check.
# Binary search on k from 1 to max(piles).
def minEatingSpeed(piles, h):
    import math
    lo, hi = 1, max(piles)
    while lo < hi:
        mid = (lo + hi) // 2
        hours = sum(math.ceil(p / mid) for p in piles)
        if hours <= h:        # mid is fast enough → maybe go slower
            hi = mid
        else:                 # mid is too slow → need to go faster
            lo = mid + 1
    return lo
Pattern 5

String Tricks

Strings in Python are immutable sequences. Most operations are O(n). The key patterns for interview string problems:

Anagram & Frequency Problems

Python
from collections import Counter

# Are two strings anagrams?
def isAnagram(s, t):
    return Counter(s) == Counter(t)    # O(n)
    # Alternative: return sorted(s) == sorted(t)   O(n log n) — simpler but slower

# Character frequency array (faster for lowercase ASCII)
def freqArray(s):
    freq = [0] * 26
    for c in s:
        freq[ord(c) - ord('a')] += 1
    return freq

# Grouping anagrams: sorted string as canonical key (LC #49)
def groupAnagrams(strs):
    groups = {}
    for s in strs:
        key = tuple(sorted(s))         # canonical form
        groups.setdefault(key, []).append(s)
    return list(groups.values())

Palindrome Checking

Python
# Simple check
def isPalindrome(s):
    return s == s[::-1]               # O(n) time and space

# In-place two-pointer (no extra memory)
def isPalindromeInPlace(s):
    lo, hi = 0, len(s) - 1
    while lo < hi:
        if s[lo] != s[hi]:
            return False
        lo += 1; hi -= 1
    return True

# Longest palindromic substring: expand around center (LC #5)
def longestPalindrome(s):
    def expand(lo, hi):
        while lo >= 0 and hi < len(s) and s[lo] == s[hi]:
            lo -= 1; hi += 1
        return s[lo+1:hi]             # slice after overshooting

    best = ""
    for i in range(len(s)):
        odd  = expand(i, i)           # odd-length palindrome
        even = expand(i, i + 1)       # even-length palindrome
        best = max(best, odd, even, key=len)
    return best

LeetCode Problems

8 problems covering every pattern — from straightforward one-pass scans to two-hard classics.

#121 Best Time to Buy and Sell Stock Easy

Given an array where prices[i] is the price on day i, find the maximum profit from one buy and one sell (buy before sell).

Insight: Track the minimum price seen so far. At each day, the best profit = price - min_so_far. One pass suffices — this is essentially a sliding window where the left end is the best buy point.

Python
def maxProfit(prices):
    min_price = float('inf')
    max_profit = 0
    for price in prices:
        min_price = min(min_price, price)
        max_profit = max(max_profit, price - min_price)
    return max_profit

# prices = [7, 1, 5, 3, 6, 4]
# min:     [7, 1, 1, 1, 1, 1]
# profit:  [0, 0, 4, 2, 5, 3]  → max = 5 ✓
⏱ O(n) time💾 O(1) spacePattern: Greedy / Sliding Min
#11 Container With Most Water Medium

Given n vertical lines at positions 0..n-1 with heights in array height, find two lines that together with the x-axis form a container with the most water.

Insight: Area = (hi - lo) × min(height[lo], height[hi]). Start with the widest container (lo=0, hi=n-1). The only way to potentially improve is to increase the minimum height — so move the shorter wall inward.

Python
def maxArea(height):
    lo, hi = 0, len(height) - 1
    best = 0
    while lo < hi:
        area = (hi - lo) * min(height[lo], height[hi])
        best = max(best, area)
        # Move the shorter wall — moving the taller wall can only
        # decrease or maintain the limiting factor (min height)
        # while also decreasing width → guaranteed worse or equal
        if height[lo] < height[hi]:
            lo += 1
        else:
            hi -= 1
    return best
⏱ O(n) time💾 O(1) spacePattern: Two Pointers (opposite ends)
#238 Product of Array Except Self Medium

Return an array where ans[i] equals the product of all elements in nums except nums[i]. Must run in O(n) without division.

Insight: ans[i] = (product of all elements left of i) × (product of all elements right of i). Two passes: left pass builds prefix products, right pass multiplies in suffix products.

Python
def productExceptSelf(nums):
    n = len(nums)
    ans = [1] * n

    # Left pass: ans[i] = product of nums[0..i-1]
    prefix = 1
    for i in range(n):
        ans[i] = prefix          # store before multiplying
        prefix *= nums[i]

    # Right pass: multiply by product of nums[i+1..n-1]
    suffix = 1
    for i in range(n - 1, -1, -1):
        ans[i] *= suffix         # combine prefix and suffix
        suffix *= nums[i]

    return ans

# nums = [1,  2,  3,  4]
# left  = [1,  1,  2,  6]   (prefix products)
# right = [24, 12, 4,  1]   (suffix products)
# ans   = [24, 12, 8,  6]   ✓
⏱ O(n) time💾 O(1) extra (output array doesn't count)Pattern: Prefix/Suffix product
#209 Minimum Size Subarray Sum Medium

Given an array of positive integers and target, find the minimum length subarray with sum ≥ target. Return 0 if impossible.

Insight: Variable sliding window. Expand right; whenever sum ≥ target, record the length and shrink from left to find the minimum window satisfying the condition.

Python
def minSubArrayLen(target, nums):
    lo = 0
    curr_sum = 0
    ans = float('inf')

    for hi in range(len(nums)):
        curr_sum += nums[hi]           # expand right

        # as long as sum >= target, try to shrink from left
        while curr_sum >= target:
            ans = min(ans, hi - lo + 1)
            curr_sum -= nums[lo]
            lo += 1

    return ans if ans != float('inf') else 0

# Works because all elements are positive:
# removing from left decreases sum, ensuring termination.
⏱ O(n) time💾 O(1) spacePattern: Variable Sliding Window
#424 Longest Repeating Character Replacement Medium

Given a string s and integer k, you can replace any character in s with any letter at most k times. Return the length of the longest substring containing only one distinct character after replacements.

Insight: In a valid window, (window_size - max_freq) ≤ k. Maintain the max frequency of any single character. If the window becomes invalid, slide it (increment both lo and hi) — we never shrink below the largest valid window seen so far.

Python
def characterReplacement(s, k):
    count = [0] * 26
    max_freq = 0
    lo = 0
    ans = 0

    for hi in range(len(s)):
        idx = ord(s[hi]) - ord('A')
        count[idx] += 1
        max_freq = max(max_freq, count[idx])

        # chars to replace = window_size - max_freq
        if (hi - lo + 1) - max_freq > k:
            count[ord(s[lo]) - ord('A')] -= 1
            lo += 1                    # slide (don't shrink below best)

        ans = max(ans, hi - lo + 1)
    return ans

# s="AABABBA", k=1
# When hi=6: window "ABABBA" (size 6), max_freq=3 (A appears 3x)
# 6 - 3 = 3 > 1, invalid → slide.
# But window size stays at 5, which was valid earlier.
⏱ O(n) time💾 O(1) spacePattern: Sliding Window + Freq Array
#153 Find Minimum in Rotated Sorted Array Medium

A sorted array was rotated at some pivot. Find the minimum element. Assume no duplicates.

Insight: Compare nums[mid] with nums[hi]. If nums[mid] > nums[hi], the minimum is in the right half (the rotation point is there). Otherwise, it's in the left half including mid.

Python
def findMin(nums):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        mid = (lo + hi) // 2
        if nums[mid] > nums[hi]:
            # mid is in the larger left part → min is to the right
            lo = mid + 1
        else:
            # mid is in the smaller right part (or mid is the min)
            hi = mid
    return nums[lo]

# [4, 5, 6, 7, 0, 1, 2]:
# lo=0 hi=6 mid=3 → 7 > 2 → lo=4
# lo=4 hi=6 mid=5 → 1 ≤ 2 → hi=5
# lo=4 hi=5 mid=4 → 0 ≤ 1 → hi=4
# lo=4 hi=4 → return nums[4]=0 ✓
⏱ O(log n) time💾 O(1) spacePattern: Binary Search (left boundary)
#33 Search in Rotated Sorted Array Medium

Search for a target in a rotated sorted array. Return its index or -1.

Insight: At every mid, one of the two halves is always fully sorted. Determine which half is sorted, then check if the target lies within that range. If yes, search there; otherwise search the other side.

Python
def search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        # Determine which half is sorted
        if nums[lo] <= nums[mid]:          # left half is sorted
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1              # target in sorted left
            else:
                lo = mid + 1              # target in right
        else:                             # right half is sorted
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1              # target in sorted right
            else:
                hi = mid - 1              # target in left
    return -1
⏱ O(log n) time💾 O(1) spacePattern: Binary Search (half-sorted)
#42 Trapping Rain Water Hard

Given an elevation map, compute how much water it can trap after rain. The water above position i = min(max_left, max_right) - height[i].

Approach 1 (prefix/suffix arrays): Precompute left_max[i] and right_max[i], then answer in one pass. O(n) time, O(n) space.

Approach 2 (two pointers): Maintain running max from each side. Process whichever side has the smaller current height — that side's water is fully determined by its own running max.

Python
def trap(height):
    lo, hi = 0, len(height) - 1
    left_max = right_max = 0
    water = 0

    while lo <= hi:
        if height[lo] <= height[hi]:
            # water at lo = left_max - height[lo]
            # (right side is ≥ height[hi] ≥ height[lo], so left_max is bottleneck)
            if height[lo] >= left_max:
                left_max = height[lo]     # new max, no water here
            else:
                water += left_max - height[lo]
            lo += 1
        else:
            if height[hi] >= right_max:
                right_max = height[hi]
            else:
                water += right_max - height[hi]
            hi -= 1

    return water

# height = [0,1,0,2,1,0,1,3,2,1,2,1]
# Expected output: 6
⏱ O(n) time💾 O(1) spacePattern: Two Pointers (running max)
