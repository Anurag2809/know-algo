Data Structure Deep Dive

Heap / Priority Queue

A complete binary tree that always keeps the minimum (or maximum) element at the top — enabling O(log n) insert and extract, and O(1) peek. The engine behind Dijkstra's, A*, and median-finding algorithms.

⏱ 25 min read 🎯 5 LeetCode solutions 🐍 Python code 📊 Complexity analysis

▲What is a Heap?

A heap is a specialized complete binary tree that satisfies the heap property: in a min-heap, every parent node is less than or equal to its children. In a max-heap, every parent is greater than or equal to its children. This single constraint guarantees that the root always holds the extreme value — minimum or maximum.

Because a heap is a complete binary tree (all levels are fully filled except possibly the last, which fills left-to-right), it can be stored compactly as a plain array without any pointers. This makes heaps extremely cache-friendly and memory-efficient.

Heap vs. Priority Queue A Priority Queue is an abstract data type: a queue where each element has a priority and the highest-priority element is always dequeued first. A heap is the most common concrete implementation of a priority queue. Python's heapq module implements a min-heap directly on a list.

Key Properties at a Glance

🔍

Peek O(1)

The root is always the min (or max). Access it directly at index 0 in the array — no traversal needed.

📈

Insert O(log n)

Append to end of array, then bubble up (heapify-up) until the heap property is restored. At most log n swaps.

📁

Extract O(log n)

Remove root, move last element to root, then sink down (heapify-down). At most log n swaps.

⚡

Build Heap O(n)

Surprisingly, you can build a heap from an unsorted array in O(n) — faster than sorting. Key insight explained in the Internals section.

Min-Heap vs. Max-Heap

Min-Heap
  • Root holds the minimum element
  • Every parent ≤ its children
  • Python's heapq is a min-heap by default
  • Use for: Dijkstra's, Kth largest (invert), merge sorted lists
Max-Heap
  • Root holds the maximum element
  • Every parent ≥ its children
  • Python: negate values to simulate with heapq
  • Use for: Kth smallest, top-K problems, median finding

⚙How It Works

Array Representation

The beauty of a heap is that the tree structure is implicit in the array. For any node at index i:

  • Parent: (i - 1) // 2
  • Left child: 2 * i + 1
  • Right child: 2 * i + 2

No pointers, no extra memory. The tree is encoded entirely by position. For example, the root is at index 0. Its left child is at index 1, right child at index 2. The left child of node 1 is at index 3, and so on.

Why 0-indexed? Using 0-based indexing, parent(i) = (i-1)//2. If you use 1-based indexing, the formula simplifies to parent(i) = i//2, children at 2i and 2i+1. Both work — 0-based is more natural in Python since lists start at 0.

Heapify-Up (Bubble Up) — After Insert

After inserting a new element at the end of the array, it may violate the heap property with its parent. The fix: repeatedly swap with the parent while the element is smaller than its parent (for min-heap). This bubbles the element up to its correct position.

1

Append to end

Place the new element at index len(heap). The tree structure is still complete.

2

Compare with parent

If heap[i] < heap[parent(i)], swap them. Update i = parent(i).

3

Repeat until done

Stop when either i == 0 (reached root) or heap[i] ≥ heap[parent(i)] (heap property satisfied).

Heapify-Down (Sink Down) — After Extract

After removing the root, move the last element to the root position (to maintain completeness). Now the root may be larger than its children. The fix: repeatedly swap with the smaller child until the heap property is restored.

1

Move last to root

Swap heap[0] with heap[-1], then pop the last element. The tree is still complete.

2

Find smaller child

Compare left child (2i+1) and right child (2i+2). The smaller child is the swap candidate.

3

Swap if needed

If the current element is larger than the smaller child, swap. Continue sinking down.

4

Stop condition

Stop when both children are larger (or no children exist). Heap property is restored.

Build Heap from Array — O(n) Not O(n log n)

Naively inserting n elements one-by-one costs O(n log n). But there is a smarter approach: start from the last non-leaf node (index (n//2) - 1) and heapify-down every node from that index down to 0.

Why O(n) and not O(n log n)? Most nodes are leaves — they don't need any sinking. In a tree with n nodes, there are roughly n/2 leaves (no work), n/4 nodes at height 1 (at most 1 swap each), n/8 at height 2 (at most 2 swaps), and so on. Summing this series: n/2 * 0 + n/4 * 1 + n/8 * 2 + ... converges to O(n). The key insight is that the work per node is proportional to its height, and most nodes are short.

🛠Core Operations

Insert + Heapify-Up

python 
def insert(self, val: int) -> None:
    """Insert val and restore heap property via heapify-up."""
    self.heap.append(val)          # Step 1: append to end
    self._heapify_up(len(self.heap) - 1)  # Step 2: bubble up

def _heapify_up(self, i: int) -> None:
    """Bubble element at index i up until heap property holds."""
    while i > 0:
        parent = (i - 1) // 2
        if self.heap[i] < self.heap[parent]:  # min-heap: child < parent → swap
            self.heap[i], self.heap[parent] = self.heap[parent], self.heap[i]
            i = parent
        else:
            break  # heap property satisfied

# Example:
# heap = [1, 3, 6, 5, 9, 8, 7]
# insert(2):
#   append → [1, 3, 6, 5, 9, 8, 7, 2]  (index 7)
#   parent(7) = 3, heap[3] = 5 > 2 → swap
#   [1, 3, 6, 2, 9, 8, 7, 5]            (index 3)
#   parent(3) = 1, heap[1] = 3 > 2 → swap
#   [1, 2, 6, 3, 9, 8, 7, 5]            (index 1)
#   parent(1) = 0, heap[0] = 1 <= 2 → stop

Extract Min + Heapify-Down

python 
def extract_min(self) -> int:
    """Remove and return the minimum element."""
    if not self.heap:
        raise IndexError("extract from empty heap")
    if len(self.heap) == 1:
        return self.heap.pop()

    min_val = self.heap[0]           # Save root (the minimum)
    self.heap[0] = self.heap.pop()   # Move last element to root
    self._heapify_down(0)            # Sink the new root down
    return min_val

def _heapify_down(self, i: int) -> None:
    """Sink element at index i down until heap property holds."""
    n = len(self.heap)
    while True:
        smallest = i
        left  = 2 * i + 1
        right = 2 * i + 2

        if left < n and self.heap[left] < self.heap[smallest]:
            smallest = left
        if right < n and self.heap[right] < self.heap[smallest]:
            smallest = right

        if smallest == i:
            break  # no swap needed — heap property holds

        self.heap[i], self.heap[smallest] = self.heap[smallest], self.heap[i]
        i = smallest

# Example:
# heap = [1, 3, 6, 5, 9, 8, 7]
# extract_min():
#   save 1; move 7 to root → [7, 3, 6, 5, 9, 8]
#   heapify_down(0): smallest child is 3 at index 1 → swap
#   [3, 7, 6, 5, 9, 8]
#   heapify_down(1): smallest child is 5 at index 3 → swap
#   [3, 5, 6, 7, 9, 8]  ← heap restored, returns 1

Build Heap — O(n)

python 
def heapify(self, arr: list) -> None:
    """Build a heap from an arbitrary array in O(n)."""
    self.heap = arr[:]  # copy the array
    n = len(self.heap)

    # Start from the last non-leaf node and heapify-down each
    # Last non-leaf is at index (n // 2) - 1
    for i in range(n // 2 - 1, -1, -1):
        self._heapify_down(i)

# Example:
# arr = [9, 4, 7, 1, 8, 3, 6]
# heapify(arr):
#   n=7, start at i=2 (last non-leaf)
#   i=2: node=7, children=3,6. min child=3 → swap → 7 goes to 5
#   i=1: node=4, children=1,8. min child=1 → swap → 4 goes to 3
#   i=0: node=9, children=1,3. min child=1 → swap → 9 goes to 1
#        at index 1: node=9, children=4,8. min child=4 → swap
#   Result: [1, 4, 3, 9, 8, 7, 6] ← valid min-heap

Python's heapq Module

Python's standard library provides a min-heap via heapq. It operates directly on a Python list, so no class instantiation is needed.

python 
import heapq

# ── Build / Convert ──────────────────────────────────────────────
h = [9, 4, 7, 1, 8, 3, 6]
heapq.heapify(h)          # converts list to min-heap IN PLACE — O(n)
print(h[0])               # 1  (min element always at index 0)

# ── Insert ───────────────────────────────────────────────────────
heapq.heappush(h, 2)      # insert 2 and maintain heap property — O(log n)

# ── Extract Min ──────────────────────────────────────────────────
min_val = heapq.heappop(h)  # remove and return smallest — O(log n)

# ── Peek ─────────────────────────────────────────────────────────
min_val = h[0]            # O(1) — no pop needed

# ── Push then Pop (more efficient than separate calls) ───────────
heapq.heappushpop(h, 5)   # push 5, then pop min (faster than push+pop)
heapq.heapreplace(h, 5)   # pop min, then push 5 (raises error if empty)

# ── n Largest / Smallest ─────────────────────────────────────────
largest  = heapq.nlargest(3, h)   # top-3 largest values
smallest = heapq.nsmallest(3, h)  # top-3 smallest values

# ── Max-Heap Trick (negate values) ───────────────────────────────
nums = [3, 1, 4, 1, 5, 9, 2, 6]
max_heap = [-x for x in nums]
heapq.heapify(max_heap)
max_val = -heapq.heappop(max_heap)   # returns 9 (the original max)
heapq.heappush(max_heap, -7)         # insert 7 into max-heap

# ── Heap of Tuples (for priority + data) ────────────────────────
events = []
heapq.heappush(events, (priority, data))  # sorted by priority (first element)
p, d = heapq.heappop(events)             # returns smallest priority first

📄Full Python Implementation

A complete MinHeap class built from scratch — no heapq dependency. This is exactly what you might be asked to implement in a system design or coding interview.

python 
class MinHeap:
    """
    Min-Heap implementation backed by a Python list.

    All index arithmetic uses 0-based indexing:
      parent(i)     = (i - 1) // 2
      left_child(i) = 2 * i + 1
      right_child(i)= 2 * i + 2
    """

    def __init__(self, data: list = None):
        if data:
            self.heap = data[:]
            self._build_heap()
        else:
            self.heap: list = []

    # ── Core Index Helpers ────────────────────────────────────────
    def _parent(self, i: int) -> int:
        return (i - 1) // 2

    def _left(self, i: int) -> int:
        return 2 * i + 1

    def _right(self, i: int) -> int:
        return 2 * i + 2

    def _swap(self, i: int, j: int) -> None:
        self.heap[i], self.heap[j] = self.heap[j], self.heap[i]

    # ── Heapify Operations ────────────────────────────────────────
    def _heapify_up(self, i: int) -> None:
        """Bubble element at i up to its correct position."""
        while i > 0:
            p = self._parent(i)
            if self.heap[i] < self.heap[p]:
                self._swap(i, p)
                i = p
            else:
                break

    def _heapify_down(self, i: int) -> None:
        """Sink element at i down to its correct position."""
        n = len(self.heap)
        while True:
            smallest = i
            l, r = self._left(i), self._right(i)
            if l < n and self.heap[l] < self.heap[smallest]:
                smallest = l
            if r < n and self.heap[r] < self.heap[smallest]:
                smallest = r
            if smallest == i:
                break
            self._swap(i, smallest)
            i = smallest

    def _build_heap(self) -> None:
        """Convert self.heap list into a valid heap in O(n)."""
        for i in range(len(self.heap) // 2 - 1, -1, -1):
            self._heapify_down(i)

    # ── Public API ────────────────────────────────────────────────
    def push(self, val) -> None:
        """Insert val — O(log n)."""
        self.heap.append(val)
        self._heapify_up(len(self.heap) - 1)

    def pop(self):
        """Remove and return min — O(log n)."""
        if self.is_empty():
            raise IndexError("pop from empty heap")
        if len(self.heap) == 1:
            return self.heap.pop()
        root = self.heap[0]
        self.heap[0] = self.heap.pop()  # move last to root
        self._heapify_down(0)
        return root

    def peek(self):
        """Return min without removing — O(1)."""
        if self.is_empty():
            raise IndexError("peek from empty heap")
        return self.heap[0]

    def pushpop(self, val):
        """Push val then pop min — more efficient than push + pop."""
        if self.heap and self.heap[0] < val:
            val, self.heap[0] = self.heap[0], val
            self._heapify_down(0)
        return val

    def is_empty(self) -> bool:
        return len(self.heap) == 0

    def __len__(self) -> int:
        return len(self.heap)

    def __repr__(self) -> str:
        return f"MinHeap({self.heap})"


# ── Usage ──────────────────────────────────────────────────────────
if __name__ == "__main__":
    h = MinHeap([9, 4, 7, 1, 8, 3, 6])
    print(h)            # MinHeap([1, 4, 3, 9, 8, 7, 6])
    print(h.peek())     # 1

    h.push(2)
    print(h.peek())     # 1 (2 is not the new min)

    print(h.pop())      # 1
    print(h.pop())      # 2
    print(h.pop())      # 3

    # Build from list
    h2 = MinHeap([5, 2, 8, 1, 9])
    while not h2.is_empty():
        print(h2.pop(), end=" ")  # 1 2 5 8 9 (sorted!)

💻LeetCode Problems

#215
Kth Largest Element in an Array
Medium Min-Heap Quickselect

Insight: Maintain a min-heap of exactly size k. If the heap grows beyond k, pop the minimum. At the end, the root of the heap is the kth largest element — because we kept only the k largest values and the smallest among them (root) is the kth.

Why this works: A min-heap of size k always contains the k largest elements seen so far. When you push a new element and the size exceeds k, you pop the minimum (the "worst" of the top k). After processing all elements, heap[0] is the kth largest.

Time: O(n log k) — n pushes, each O(log k). Space: O(k).

python 
import heapq

class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        # Maintain a min-heap of exactly size k
        heap = []
        for num in nums:
            heapq.heappush(heap, num)
            if len(heap) > k:
                heapq.heappop(heap)  # discard the smallest of current top-k
        # Root of the min-heap is the kth largest
        return heap[0]

# One-liner using heapq.nlargest:
# return heapq.nlargest(k, nums)[-1]  # O(n log k), same complexity
#347
Top K Frequent Elements
Medium Heap Counter Bucket Sort

Insight: Count frequencies with a hash map, then use a min-heap of size k to track the k most frequent elements — the same "maintain top-k" pattern from LC 215, but keyed on frequency instead of value.

Time: O(n log k). Space: O(n) for the frequency map.

python 
import heapq
from collections import Counter

class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        # Step 1: count frequencies
        freq = Counter(nums)  # {num: count, ...}

        # Step 2: heap of (frequency, num) — min-heap by frequency
        heap = []
        for num, count in freq.items():
            heapq.heappush(heap, (count, num))
            if len(heap) > k:
                heapq.heappop(heap)  # remove least frequent

        return [num for count, num in heap]

# Alternative: heapq.nlargest — more Pythonic, same complexity
# return [num for num, _ in Counter(nums).most_common(k)]
#23
Merge k Sorted Lists
Hard Min-Heap Linked List

Insight: A min-heap allows us to always extract the globally smallest element across all k lists in O(log k) time. Initialize the heap with the head node of each list. After extracting the min, push its next node (if any) to maintain k candidates.

Time: O(N log k) where N = total nodes, k = number of lists. Space: O(k) for the heap.

python 
import heapq
from typing import Optional, List

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        dummy = ListNode(0)
        cur = dummy

        # Heap stores tuples: (value, list_index, node)
        # list_index breaks ties when values are equal — avoids comparing ListNodes
        heap = []
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(heap, (node.val, i, node))

        while heap:
            val, i, node = heapq.heappop(heap)  # extract global minimum
            cur.next = node
            cur = cur.next
            if node.next:
                # Push the next node from the same list
                heapq.heappush(heap, (node.next.val, i, node.next))

        return dummy.next

# Walkthrough for lists = [[1,4,5],[1,3,4],[2,6]]:
# Initial heap: [(1,0,node1), (1,1,node1'), (2,2,node2)]
# Pop (1,0): output 1, push (4,0,node4) → heap: [(1,1), (2,2), (4,0)]
# Pop (1,1): output 1, push (3,1,node3) → heap: [(2,2), (3,1), (4,0)]
# ...continues until heap empty
# Output: 1→1→2→3→4→4→5→6
#295
Find Median from Data Stream
Hard Two Heaps Design

Insight: Maintain two heaps that partition the data stream at the median. A max-heap (lo) holds the lower half; a min-heap (hi) holds the upper half. Invariant: len(lo) == len(hi) or len(lo) == len(hi) + 1.

Why two heaps? The median is either the max of the lower half, the min of the upper half, or their average. Both are accessible in O(1) from heap roots.

Time: O(log n) addNum, O(1) findMedian. Space: O(n).

python 
import heapq

class MedianFinder:
    """
    lo: max-heap (lower half)  — Python: store negated values
    hi: min-heap (upper half)
    Invariant: len(lo) >= len(hi), and len(lo) - len(hi) <= 1
               lo.max <= hi.min  (all elements in lo are <= all in hi)
    """
    def __init__(self):
        self.lo = []   # max-heap (negate for Python's min-heap)
        self.hi = []   # min-heap

    def addNum(self, num: int) -> None:
        # Step 1: push to lo (max-heap via negation)
        heapq.heappush(self.lo, -num)

        # Step 2: ensure lo.max <= hi.min (cross-heap ordering)
        if self.hi and -self.lo[0] > self.hi[0]:
            heapq.heappush(self.hi, -heapq.heappop(self.lo))

        # Step 3: balance sizes — lo should have at most 1 more than hi
        if len(self.lo) > len(self.hi) + 1:
            heapq.heappush(self.hi, -heapq.heappop(self.lo))
        elif len(self.hi) > len(self.lo):
            heapq.heappush(self.lo, -heapq.heappop(self.hi))

    def findMedian(self) -> float:
        if len(self.lo) > len(self.hi):
            return -self.lo[0]              # odd total: lo has extra element
        return (-self.lo[0] + self.hi[0]) / 2.0  # even total: average the two midpoints

# Example:
# addNum(1) → lo=[-1], hi=[]       median = 1.0
# addNum(2) → lo=[-1], hi=[2]      median = 1.5
# addNum(3) → lo=[-2,-1], hi=[3]   median = 2.0
# addNum(4) → lo=[-2,-1], hi=[3,4] median = 2.5
#1046
Last Stone Weight
Easy Max-Heap Simulation

Insight: Each step, smash the two heaviest stones. This is a textbook max-heap problem. Since Python only has a min-heap, negate all values to simulate a max-heap. After smashing, push back the difference (if non-zero).

Time: O(n log n). Space: O(n).

python 
import heapq

class Solution:
    def lastStoneWeight(self, stones: list[int]) -> int:
        # Negate all values to simulate a max-heap
        heap = [-s for s in stones]
        heapq.heapify(heap)  # O(n)

        while len(heap) > 1:
            y = -heapq.heappop(heap)   # heaviest stone (negate back)
            x = -heapq.heappop(heap)   # second heaviest

            if x != y:
                # Stone of weight (y - x) survives
                heapq.heappush(heap, -(y - x))
            # If x == y, both are destroyed — push nothing

        return -heap[0] if heap else 0

# Walkthrough for stones = [2,7,4,1,8,1]:
# heap (negated): [-8,-7,-4,-2,-1,-1]
# Pop 8,7 → push -(8-7)=-1 → [-4,-2,-1,-1,-1]
# Pop 4,2 → push -(4-2)=-2 → [-2,-1,-1,-1]
# Pop 2,1 → push -(2-1)=-1 → [-1,-1,-1]
# Pop 1,1 → equal, destroy both → [-1]
# Return 1

🌎Real-World Applications

🔷

Dijkstra's Algorithm

Min-heap stores (distance, node) pairs. Always expands the closest unvisited node first — guarantees shortest paths in O((V+E) log V).

🎯

A* Search

Priority queue ordered by f(n) = g(n) + h(n). Used in game AI, GPS routing, and robotics for optimal pathfinding with heuristics.

📋

Task Schedulers

OS process scheduling (lowest-priority-first or highest-first). CPU schedulers use priority queues to decide which process to run next.

🕐

Event-Driven Simulation

Events sorted by timestamp. Always process the next chronological event. Used in network simulators, game engines, and discrete event systems.

📃

Huffman Encoding

Always merge the two lowest-frequency symbols. Min-heap makes finding the two minimums efficient — enables optimal prefix-free compression.

📂

Database Query Optimization

Top-K queries, merge-sort passes in external sorting, and streaming median computation all rely on heap-based priority queues internally.

🆕Variations

Max-Heap

Root holds the maximum. In Python, negate all values when pushing/popping from heapq. Pattern: push -val, pop and negate to get original.

Indexed Priority Queue

Supports decrease-key and arbitrary deletion in O(log n). Maintains a position map (element → heap index). Required for efficient Dijkstra's on dense graphs.

Fibonacci Heap

Amortized O(1) insert and decrease-key, O(log n) extract-min. Theoretically optimal for Dijkstra's (O(E + V log V)) but complex to implement. Rarely used in practice.

Binomial Heap

Supports merging two heaps in O(log n). Composed of binomial trees. Each "order" is a perfectly structured sub-heap, enabling efficient merge operations.

d-ary Heap

Generalization: each node has d children instead of 2. A 4-ary heap has lower height (log_4 n) but more comparisons per level. Optimal d depends on cache behavior.

Two-Heap Technique

Combine a max-heap (lower half) + min-heap (upper half) to maintain a running median in O(log n) per insertion, O(1) query. See LC #295.

Python Max-Heap Pattern Python's heapq only provides a min-heap. For a max-heap, negate values on push and negate again on pop. For objects, implement __lt__ to reverse comparison. For tuples, push (-priority, data).
python 
import heapq

# ── Max-Heap with negation ────────────────────────────────────────
nums = [3, 1, 4, 1, 5, 9, 2, 6]
max_heap = [-x for x in nums]

from dataclasses import dataclass, field

@dataclass(order=True)
class PrioritizedItem:
    priority: int
    item: object = field(compare=False)

# For max-heap, negate priority
tasks = []
heapq.heappush(tasks, PrioritizedItem(-10, "urgent task"))
heapq.heappush(tasks, PrioritizedItem(-3,  "normal task"))
heapq.heappush(tasks, PrioritizedItem(-7,  "important task"))

top = heapq.heappop(tasks)  # PrioritizedItem(-10, 'urgent task')
print(top.item)             # "urgent task"
